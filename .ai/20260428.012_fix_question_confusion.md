# 20260428.012 - Исправление: LLM путал примеры с реальными вопросами

**Дата:** 2026-04-28  
**Автор:** AI Agent (Claude)  
**Проблема:** LLM отвечал на примеры из промптов вместо реального вопроса пользователя

## Описание проблемы

При запросе "что такое smart monitor" агент отвечал про "Какое ПО установлено на АРМ оператора КЦОИ":

```bash
python rag_lg_agent.py "что такое smart monitor"

# Ответ агента:
{
  "thought": "Пользователь спрашивает, какое ПО установлено на АРМ оператора КЦОИ. 
             Это вопрос типа 'Какое X установлено на Y'..."
}
```

❌ **Проблема:** LLM читал ПРИМЕРЫ в system промптах и принимал их за реальный вопрос пользователя!

### Причина

В промптах были **очень детальные примеры**:

```markdown
**Пример:** "Какое ПО установлено на АРМ оператора"
- Основные: "ПО", "АРМ оператора", "установлено"
- Синонимы: "программное обеспечение", "программные средства", "софт"
...
```

LLM видел эти примеры в system message и думал, что это реальная задача, **игнорируя** реальный вопрос в user message.

## Решение

### 1. Четкое выделение реального вопроса в user messages

Во ВСЕХ user.md файлах добавлено явное указание:

```markdown
🎯 **РЕАЛЬНЫЙ ВОПРОС ПОЛЬЗОВАТЕЛЯ (отвечай на ЭТОТ вопрос):**

{{ user_query }}

⚠️ Примеры в system message - это ТОЛЬКО примеры, НЕ реальные вопросы!
```

### 2. Явная пометка примеров как ПРИМЕРОВ

В system промптах все примеры помечены:

```markdown
**Пример workflow (ЭТО ТОЛЬКО ПРИМЕР, НЕ РЕАЛЬНАЯ ЗАДАЧА!):** "Какое ПО..."

ПРИМЕР для "Какое ПО установлено..." (ЭТО ТОЛЬКО ПРИМЕР!):
```

## Как это работает

### До изменений:

```
System message:
  "...Пример: Какое ПО установлено на АРМ оператора..."
  
User message:
  "Вопрос: что такое smart monitor"

LLM думает:
  "Пример выглядит как задача → буду отвечать на пример"
```

### После изменений:

```
System message:
  "...ПРИМЕР (ЭТО ТОЛЬКО ПРИМЕР!): Какое ПО установлено..."
  
User message:
  "🎯 РЕАЛЬНЫЙ ВОПРОС ПОЛЬЗОВАТЕЛЯ: что такое smart monitor
   ⚠️ Примеры в system - это ТОЛЬКО примеры!"

LLM понимает:
  "Примеры - для справки. Реальный вопрос - 'что такое smart monitor'"
```

## Визуальные маркеры

Использованы яркие эмодзи для привлечения внимания LLM:

- 🎯 - для реального вопроса (цель, target)
- ⚠️ - для предупреждений
- ❌ - для неправильных примеров
- ✅ - для правильных примеров

Эти маркеры помогают LLM визуально отличить:
- Что важно (реальный вопрос) 🎯
- Что игнорировать (примеры)
- Что правильно/неправильно ✅❌

## Преимущества

### 1. Точность ответов

**До:** Агент отвечал на случайные примеры из промптов  
**После:** Агент отвечает строго на вопрос пользователя

### 2. Предсказуемость

**До:** Непонятно на что агент ответит  
**После:** Агент всегда отвечает на реальный вопрос

### 3. Надежность

**До:** Могло сломаться при добавлении новых примеров  
**После:** Примеры четко отделены от реальных задач

### 4. Отладка

**До:** Сложно понять почему агент отвечает неправильно  
**После:** Четко видно что агент должен читать
