# 20260428.016 - Вынос форматирования промптов в .md файлы

**Дата:** 2026-04-28  
**Автор:** AI Agent (Claude)  
**Изменение:** Форматирование промптов (SYSTEM + TOOLS + MESSAGES + USER) перенесено из Python кода в .md файлы

## Описание изменения

Ранее формирование полного промпта происходило в Python коде:

```python
# Было (в rag_lg_agent.py):
system_message = prompts.get_plan_system_prompt(state)
user_message = prompts.get_plan_user_prompt(state)

result = structured_llm.invoke([
    {"role": "system", "content": system_message},
    {"role": "user", "content": user_message}
])
```

Теперь всё форматирование находится в .md файлах:

```python
# Стало:
full_prompt = prompts.get_plan_prompt(state)

result = structured_llm.invoke(full_prompt)
```

## Почему это лучше

### 1. Единое место для управления структурой промпта

**До:** Структура промпта размазана по коду Python:
```python
# В plan_node:
messages = [
    {"role": "system", "content": system_message},
    {"role": "user", "content": user_message}
]

# В action_node:
messages = [
    {"role": "system", "content": system_message}
]
messages.extend(state["messages"])  # История
messages.append({"role": "user", "content": user_message})
```

**После:** Вся структура в одном .md файле:
```markdown
[SYSTEM]
{% include 'plan/system.md' %}

[TOOLS]
{{ available_tools }}

[MESSAGES]
{{ messages }}

[USER]
{% include 'plan/user.md' %}
```

### 2. Легче менять структуру

Хотим добавить новую секцию [EXAMPLES]?

**До:** Нужно изменить код Python в каждом node  
**После:** Просто добавить секцию в prompt.md

### 3. Шаблонизация всей структуры

Теперь можно использовать Jinja2 для управления структурой:

```markdown
[SYSTEM]
{% include 'action/system.md' %}

[TOOLS]
{{ available_tools }}

{% if iteration > 1 %}
[MESSAGES - PREVIOUS ITERATIONS]
{% for msg in messages %}
[{{ msg.role|upper }}]
{{ msg.content }}
{% endfor %}
{% endif %}

[USER]
{% include 'action/user.md' %}
```

### 4. Проще тестировать

Можно просто отрендерить prompt.md и посмотреть полный результат:

```python
from prompts import get_action_prompt
print(get_action_prompt(state))
```

## Обновления в prompts.py

Добавлены 5 новых функций для полных промптов:

```python
def get_plan_prompt(state: dict) -> str:
    """Полный промпт для plan node из plan/prompt.md"""
    return _loader.render('plan/prompt.md', state)

def get_action_prompt(state: dict) -> str:
    """Полный промпт для action node из action/prompt.md"""
    return _loader.render('action/prompt.md', state)

def get_observation_prompt(state: dict) -> str:
    """Полный промпт для observation node из observation/prompt.md"""
    return _loader.render('observation/prompt.md', state)

def get_refine_prompt(state: dict) -> str:
    """Полный промпт для refine node из refine/prompt.md"""
    return _loader.render('refine/prompt.md', state)

def get_final_prompt(state: dict) -> str:
    """Полный промпт для final node из final/prompt.md"""
    return _loader.render('final/prompt.md', state)
```

Старые функции (`get_*_system_prompt`, `get_*_user_prompt`) **сохранены** для обратной совместимости.

## Преимущества для разработки

### 1. Экспериментирование

Легко попробовать разные структуры промптов:

```markdown
# Вариант A:
[SYSTEM]
...
[USER]
...

# Вариант B:
[CONTEXT]
...
[TOOLS]
...
[TASK]
...
```

Просто меняем prompt.md, не трогая код.

### 2. Версионирование

В Git видно изменение структуры промпта:

```diff
 [SYSTEM]
 {% include 'action/system.md' %}

+[EXAMPLES]
+{% include 'action/examples.md' %}
+
 [TOOLS]
 {{ available_tools }}
```

### 3. Документация

prompt.md файл - это **самодокументирующаяся** структура промпта.

### 4. Переиспользование

Можно создать общий шаблон для всех nodes:

```markdown
# prompts/base_prompt.md
[SYSTEM]
{% block system %}{% endblock %}

[TOOLS]
{{ available_tools }}

[MESSAGES]
{% for msg in messages %}
  ...
{% endfor %}

[USER]
{% block user %}{% endblock %}
```

И наследовать:

```markdown
# prompts/action/prompt.md
{% extends 'base_prompt.md' %}
{% block system %}{% include 'action/system.md' %}{% endblock %}
{% block user %}{% include 'action/user.md' %}{% endblock %}
```
