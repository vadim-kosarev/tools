# Исправление: обработка пустых чанков и защита от ошибок

**Дата:** 2026-02-22  
**Файлы:** `t_gigaam_simple.py`, `t_gigaam_blocks.py`, `t_gigaam.py`

## 🐛 ПРОБЛЕМА

Скрипт падал с ошибкой при попытке транскрибировать пустой (0 байт) чанк:

```
ValueError: both buffer length (0) and count (-1) must not be 0
```

**Причина:**
- Последний чанк был слишком коротким (9 секунд)
- FFmpeg создал пустой файл (0 байт)
- GigaAM не может обработать пустой аудиофайл
- Исключение прерывало обработку
- **Результат не сохранялся**, несмотря на успешную транскрипцию 19 из 20 чанков

---

## ✅ РЕШЕНИЕ

Добавлена защита во все скрипты транскрибации:

### 1. Проверка размера файла перед транскрипцией
### 2. Обработка ошибок транскрипции
### 3. Продолжение работы при ошибке чанка
### 4. Сохранение результата даже при ошибках

### 1. `t_gigaam_simple.py`

**Добавлено:**

```python
texts = []
for i, chunk in enumerate(chunking_result.chunks, 1):
    logger.info(f"→ Транскрипция чанка {i}/{len(chunking_result.chunks)}...")
    
    # ✅ Проверяем размер чанка
    try:
        chunk_size = chunk.file_path.stat().st_size
        if chunk_size == 0:
            logger.warning(f"  Чанк {i} пустой (0 байт), пропускаем")
            continue
    except OSError as e:
        logger.error(f"  Ошибка чтения файла чанка {i}: {e}")
        continue
    
    # ✅ Обработка ошибок транскрипции
    try:
        with torch.inference_mode():
            text = model.transcribe(str(chunk.file_path))
        
        chunk_text = text.strip()
        if chunk_text:
            chunk_text = chunk_text + " [...]"
            texts.append(chunk_text)
            logger.info(f"  {chunk_text[:80]}...")
        else:
            logger.debug(f"  Пропущен пустой чанк {i}")
            
    except Exception as e:
        # ✅ Продолжаем обработку при ошибке
        logger.error(f"  Ошибка транскрипции чанка {i}: {type(e).__name__} → {e}")
        logger.debug(f"  Чанк будет пропущен, продолжаем обработку...")
        continue

# ✅ Сохраняем результат даже если были ошибки
full_text = " ".join(texts)
```

### 2. `t_gigaam_blocks.py`

```python
def transcribe_single_chunk(model, chunk_path, chunk_start_sec):
    # ✅ Проверяем размер файла
    try:
        chunk_size = chunk_path.stat().st_size
        if chunk_size == 0:
            logger.warning(f"Чанк {chunk_path.name} пустой (0 байт), пропускаем")
            return ""
    except OSError as e:
        logger.error(f"Ошибка чтения файла {chunk_path.name}: {e}")
        return ""
    
    # ✅ Обработка ошибок транскрипции
    try:
        with torch.inference_mode():
            text = model.transcribe(str(chunk_path)).strip()
        
        if text:
            return text + " [...]"
        else:
            return text
            
    except Exception as e:
        # ✅ Возвращаем пустую строку при ошибке
        logger.error(f"Ошибка транскрипции чанка {chunk_path.name}: {type(e).__name__} → {e}")
        return ""
```

### 3. `t_gigaam.py`

**Добавлено в цикл обработки сегментов:**

```python
chunk_path = extract_chunk(Path(process_path), start, end, tmp_dir)

# ✅ Проверяем размер чанка
try:
    chunk_size = chunk_path.stat().st_size
    if chunk_size == 0:
        logger.warning(f"  Чанк для сегмента {start:.1f}–{end:.1f} пустой (0 байт), пропускаем")
        if chunk_path.exists():
            chunk_path.unlink()
        continue
except OSError as e:
    logger.error(f"  Ошибка чтения файла чанка: {e}")
    continue

# ✅ Транскрипция с обработкой ошибок (уже было)
try:
    with torch.inference_mode():
        text = model.transcribe(str(chunk_path)).strip()
    # ...
except Exception as e:
    logger.error(f"Ошибка на сегменте {start:.1f}–{end:.1f}: {e}")
finally:
    if chunk_path.exists():
        chunk_path.unlink()
```

## 🔧 ТИПЫ ОШИБОК

### 1. Пустой чанк (0 байт):
```
WARNING: Чанк 20 пустой (0 байт), пропускаем
```
**Причина:** FFmpeg не смог извлечь аудио (слишком короткий сегмент)

### 2. Ошибка чтения файла:
```
ERROR: Ошибка чтения файла чанка 5: [Errno 2] No such file or directory
```
**Причина:** Файл удалён или недоступен

### 3. Ошибка транскрипции:
```
ERROR: Ошибка транскрипции чанка 15: ValueError → both buffer length (0) and count (-1) must not be 0
```

**Причина:** GigaAM не может обработать файл
