# Обработка коротких файлов - создание пустого результата

**Дата:** 2026-02-22  
**Файлы:** `transcribe_utils.py`, `t_gigaam_simple.py`, `t_gigaam.py` (требует ручного исправления), `t_gigaam_blocks.py` (требует ручного исправления)

## 🎯 ЗАДАЧА

Для файлов с длительностью **< 2 секунд** или **неопределенной длительностью** создавать **пустой файл результата** вместо выброса исключения.

**Причина:** Это позволит правильно пометить файл как обработанный (в `processed_files`), избежав зацикливания.

---

## ✅ РЕАЛИЗАЦИЯ

### 1. `transcribe_utils.py` - функция `cut_audio_to_chunks()`

**Было:**
```python
total_sec = get_audio_duration_from_ffmpeg(input_path)
if total_sec == 0:
    raise RuntimeError(f"Не удалось определить длительность: {input_path}")
```

**Стало:**
```python
total_sec = get_audio_duration_from_ffmpeg(input_path)

# Для файлов с неопределённой или очень короткой длительностью возвращаем пустой результат
if total_sec == 0 or total_sec < 2.0:
    logger.warning(f"Длительность файла: {total_sec:.1f} сек - слишком мала или не определена")
    return AudioChunkingResult(chunks=[], total_duration_sec=total_sec)
```

**Эффект:** Не выбрасываем исключение, а возвращаем пустой результат с `chunks=[]`.

### 2. `t_gigaam_simple.py` - функция `transcribe_file()`

**Было:**
```python
chunking_result = cut_audio_to_chunks(...)

# Проверяем, что создано хотя бы один чанк
if len(chunking_result.chunks) == 0:
    error_msg = (...)
    logger.error(error_msg)
    raise ValueError(error_msg)  # ❌ Исключение!

texts = []
```

**Стало:**
```python
chunking_result = cut_audio_to_chunks(...)

# Проверяем длительность и количество чанков
# Для файлов < 2 сек или без чанков создаём пустой результат
if chunking_result.total_duration_sec < 2.0 or len(chunking_result.chunks) == 0:
    logger.warning(
        f"Файл слишком короткий ({chunking_result.total_duration_sec:.1f} сек) "
        f"или не удалось создать чанки. Создаём пустой файл результата."
    )
    full_text = ""
    
    # Очистка
    cleanup_chunk_files(chunking_result.chunks)
    if is_video and audio_path:
        cleanup_temp_file(audio_path)
    if tmp_dir.exists():
        try:
            tmp_dir.rmdir()
        except:
            pass
    
    # ✅ Сохраняем пустой файл!
    output_path = save_transcription_to_file(full_text, input_path, revision, suffix="simple")
    logger.info(f"Создан пустой файл результата → {output_path}")
    return

texts = []
```

**Эффект:** Создаётся пустой файл транскрипции, функция завершается успешно.

### 3. `t_gigaam.py` - функция `transcribe_with_speakers()` ⚠️ ТРЕБУЕТ РУЧНОГО ИСПРАВЛЕНИЯ

**Файл слит в одну строку, невозможно отредактировать автоматически.**

Нужно добавить проверку **ПЕРЕД диаризацией**:

```python
# После определения process_path
logger.info(f"Обнаружен аудиофайл: {input_path.suffix}")
process_path = file_path

# ✅ ДОБАВИТЬ ЭТУ ПРОВЕРКУ:
# Проверяем длительность файла
duration = get_audio_duration(Path(process_path))
if duration < 2.0 or duration == 0:
    logger.warning(
        f"Файл слишком короткий ({duration:.1f} сек) "
        f"или не удалось определить длительность. Создаём пустой файл результата."
    )
    full_text = ""
    
    # Очистка
    if converted_audio_path and converted_audio_path.exists():
        converted_audio_path.unlink()
    tmp_dir.rmdir() if tmp_dir.exists() else None
    
    # Сохраняем пустой файл
    out_path = input_path.parent / f"{input_path.stem}.gigaam-{revision}-speakers.txt"
    out_path.write_text(full_text, encoding="utf-8")
    original_mtime = input_path.stat().st_mtime
    os.utime(out_path, (original_mtime, original_mtime))
    logger.info(f"Создан пустой файл результата → {out_path}")
    return

logger.info(f"→ Загрузка модели GigaAM-v3...")
# ...остальной код...
```

### 4. `t_gigaam_blocks.py` ⚠️ ТРЕБУЕТ РУЧНОГО ИСПРАВЛЕНИЯ

model = load_gigaam_model(revision, device)

# ✅ ДОБАВИТЬ ЭТУ ПРОВЕРКУ:

    cleanup_chunk_files(chunking_result.chunks)
    if is_video and audio_path and audio_path.exists():

            pass
    
    # Сохраняем пустой файл

all_sentences = transcribe_all_chunks(model, chunking_result.chunks)

## 💡 ПОЧЕМУ ПУСТОЙ ФАЙЛ?

### Альтернативы:

1. ❌ **Выбросить исключение** → зацикливание (файл не помечен как обработанный)
2. ❌ **Не создавать файл** → зацикливание (status = NOT_ATTEMPTED)
3. ✅ **Создать пустой файл** → правильно:
   - Файл помечен как обработанный
   - `status = FAILED` (пустой файл)
   - `processed_files.add(file)` при запуске
   - Нет повторных попыток

## 🔧 ВЗАИМОДЕЙСТВИЕ С processed_files

### В t_directory.py:

```python
processed_files = set()

while True:
    media_files = collect_media_files(...)
    media_files = [f for f in media_files if f not in processed_files]
    
    file_path = media_files[0]
    
    # ✅ Добавляем ДО обработки
    processed_files.add(file_path)
    
    # Обработка
    success = process_single_file(file_path, ...)
    
    # Результат не важен - файл в processed_files
```

### Для короткого файла:

```
Итерация #1:
  - processed_files.add(recording_2sec.amr)
  - Обработка → Создан пустой файл → Успех
  - stats["success"] += 1

Итерация #2:
  - recording_2sec.amr в processed_files
  - Отфильтрован ✅
  - Не обрабатывается повторно
```

## ⚠️ СТАТУС ФАЙЛОВ

### При следующем запуске скрипта:

```python
status = get_transcription_status(Path("recording_2sec.amr"), "e2e_rnnt")
# Транскрипция: recording_2sec.gigaam-e2e_rnnt-simple.txt (0 байт)
# → status = TranscriptionStatus.FAILED

# В collect_media_files():
if skip_existing:
    if status == TranscriptionStatus.SUCCESS:
        # Пропускаем
    # FAILED и NOT_ATTEMPTED → обрабатываем
```

**НО!** В `process_directory()`:
```python
# Новая сессия → processed_files пуст
# Файл попадёт в обработку снова!
```

**РЕШЕНИЕ:** При старте скрипта файлы со статусом FAILED уже не будут обрабатываться благодаря `skip_existing` в `collect_media_files()`.

Или можно добавить инициализацию:
```python
# В начале process_directory()
processed_files = set()

# Добавляем файлы со статусом FAILED
initial_scan = collect_media_files(directory, skip_existing=False)
for file_path in initial_scan:
    status = get_transcription_status(file_path, revision)
    if status == TranscriptionStatus.FAILED:
        processed_files.add(file_path)
```

Но это противоречит простоте логики. Лучше полагаться на `skip_existing`.
