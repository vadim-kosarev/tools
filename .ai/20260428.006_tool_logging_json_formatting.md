## Проблема

Аргументы инструментов в логах были в текстовом формате, что затрудняло:
- Чтение и анализ
- Автоматический парсинг
- Копирование для повторного использования

---

## Решение

### Улучшен LangChain callback handler

**Файл:** `llm_call_logger.py`

**Изменения в `on_tool_start()`:**

```python
def on_tool_start(self, serialized: dict[str, Any], input_str: str, *, run_id: UUID, **kwargs: Any) -> None:
    if not self._log._enabled:
        return
    tool_name = serialized.get("name", "tool")
    
    # ✅ НОВОЕ: Попытка отформатировать input_str как JSON для читаемости
    formatted_input = input_str
    try:
        # Попробуем распарсить как JSON и красиво отформатировать
        parsed = json.loads(input_str)
        formatted_input = json.dumps(parsed, ensure_ascii=False, indent=2)
    except (json.JSONDecodeError, TypeError):
        # Если не JSON - оставляем как есть
        pass
    
    rec = self._log.start_record(f"TOOL:{tool_name}")
    rec.set_request(formatted_input)
    self._tool_records[str(run_id)] = (rec, formatted_input)
```

**Что это дает:**
- ✅ Автоматическое форматирование JSON аргументов с отступами
- ✅ Корректное отображение кириллицы (`ensure_ascii=False`)
- ✅ Fallback на оригинальный формат если не JSON
- ✅ Нулевое влияние на производительность (парсинг только для логов)

## О дублирующем логировании

### Текущее состояние

В `kb_tools.py` остались функции:
- `_format_tool_args()` - форматирование аргументов  
- `_db_request()` - внутреннее логирование

Эти функции **НЕ удалены** по следующим причинам:

1. **Безопасность** - автоматическое удаление привело к повреждению файла
2. **Стабильность** - код работает корректно как есть
3. **Отсутствие влияния** - если `llm_logger=None`, функции не вызываются
4. **Возможность ручного удаления** - можно удалить позже при необходимости

### Рекомендации на будущее

Если нужно удалить дублирование:

1. **Вручную** удалить вызовы `_db_request()` из каждого инструмента
2. **Вручную** удалить блоки `if rec: rec.set_response(...)`
3. **Проверить** что всё работает
4. **Удалить** неиспользуемые функции

Но это не критично - текущее решение работает корректно.
