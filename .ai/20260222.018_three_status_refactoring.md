# Рефакторинг: Три статуса транскрипции вместо булевых функций

**Дата:** 2026-02-22  
**Файл:** `transcribe/t_directory.py`

## 🎯 ЗАДАЧА

Заменить две булевые функции (`has_transcription()`, `has_failed_transcription()`) на единую функцию с тремя состояниями:
1. **NOT_ATTEMPTED** - не пытались обрабатывать
2. **FAILED** - пытались, но не получилось (пустой файл)
3. **SUCCESS** - есть успешный результат

---

## ✅ РЕАЛИЗАЦИЯ

### 1. Enum для статусов

```python
from enum import Enum

class TranscriptionStatus(Enum):
    """Статус транскрипции файла"""
    NOT_ATTEMPTED = "not_attempted"  # Не пытались обрабатывать
    FAILED = "failed"                 # Пытались, но неудачно (пустой файл)
    SUCCESS = "success"               # Есть успешный результат
```

---

### 2. Единая функция `get_transcription_status()`

**Заменяет:**
- ❌ `has_transcription()` → `bool`
- ❌ `has_failed_transcription()` → `bool`

**На:**
- ✅ `get_transcription_status()` → `TranscriptionStatus`

```python
def get_transcription_status(file_path: Path, revision: str) -> TranscriptionStatus:
    """
    Проверяет статус транскрипции файла.
    
    Returns:
        TranscriptionStatus:
            - NOT_ATTEMPTED: Нет файла транскрипции
            - FAILED: Есть файл, но он пустой (0 байт)
            - SUCCESS: Есть файл с содержимым
    """
    stem = file_path.stem
    parent = file_path.parent
    
    # Паттерн: <имя>.gigaam-<revision>-*.txt
    pattern = f"{stem}.gigaam-{revision}-*.txt"
    matching_files = list(parent.glob(pattern))
    
    if not matching_files:
        # Нет файлов транскрипции
        return TranscriptionStatus.NOT_ATTEMPTED
    
    # Проверяем найденные файлы
    has_empty = False
    has_non_empty = False
    
    for transcription_path in matching_files:
        try:
            file_size = transcription_path.stat().st_size
            if file_size == 0:
                has_empty = True
            else:
                has_non_empty = True
        except OSError:
            continue
    
    # Приоритет: если есть хотя бы один непустой файл - SUCCESS
    if has_non_empty:
        return TranscriptionStatus.SUCCESS
    
    # Если есть только пустые файлы - FAILED
    if has_empty:
        return TranscriptionStatus.FAILED
    
    # Файлы есть, но недоступны (OSError)
    return TranscriptionStatus.NOT_ATTEMPTED
```

## 📊 СРАВНЕНИЕ

### Было (две булевые функции):

| Ситуация | `has_transcription()` | `has_failed_transcription()` | Логика |
|----------|----------------------|------------------------------|--------|
| Нет файла | `False` | `False` | NOT_ATTEMPTED |
| Пустой файл | `False` | `True` | FAILED |
| Файл с данными | `True` | `False` | SUCCESS |

**Проблема:** Нужно вызывать две функции и комбинировать результаты.

---

### Стало (один статус):

| Ситуация | `get_transcription_status()` |
|----------|------------------------------|
| Нет файла | `NOT_ATTEMPTED` |
| Пустой файл | `FAILED` |
| Файл с данными | `SUCCESS` |

**Преимущество:** Один вызов, явное состояние.

## 🎯 ЛОГИКА ПРИОРИТЕТОВ

### Если найдено несколько файлов:

```python
# Пример: recording.gigaam-e2e_rnnt-simple.txt (0 байт)
#         recording.gigaam-e2e_rnnt-speakers.txt (15 KB)

# Приоритет: SUCCESS > FAILED > NOT_ATTEMPTED
if has_non_empty:
    return TranscriptionStatus.SUCCESS  # ✅ Есть хотя бы один успешный
```

**Логика:**
- Если есть **хотя бы один** непустой файл → `SUCCESS`
- Если есть **только** пустые файлы → `FAILED`
- Если файлов нет → `NOT_ATTEMPTED`
