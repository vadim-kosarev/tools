## Цель

Привести все функции `get_*_prompt()` в модуле `prompts.py` к единой сигнатуре, где все функции принимают только один параметр `state: dict`. Все необходимые переменные (system_prompt, user_query, iteration, MAX_ITERATIONS и т.д.) должны извлекаться из этого словаря `state`.

## Выполненные изменения

### 1. Обновление `prompts.py`

#### a) Обновлен docstring модуля
Документация теперь отражает единую сигнатуру для всех функций и содержит список ожидаемых ключей в `state`:

**Ожидаемые ключи в state:**
- `system_prompt: str` - базовый системный промпт
- `user_query: str` - вопрос пользователя
- `iteration: int` - текущая итерация
- `MAX_ITERATIONS: int` - максимальное количество итераций
- `step: int` - текущий шаг
- `plan: list` - план поиска
- `refinement_plan: list` - план уточнения (опционально)
- `observation: str` - результат наблюдения
- `all_tool_results: list` - все результаты инструментов
- `tools_json: str` - JSON с результатами инструментов (для observation_node)
- `total_tools: int` - общее количество выполненных инструментов (для final_node)
- `all_results_json: str` - JSON со всеми результатами (для final_node)

## Преимущества изменений

### 1. **Единообразие и согласованность**
Все функции теперь имеют одинаковую сигнатуру `(state: dict) -> str`, что упрощает понимание и использование API.

### 2. **Гибкость**
Легко добавлять новые параметры - просто добавляем их в state, не меняя сигнатуры функций.

### 3. **Упрощение вызовов**
Теперь не нужно помнить, какие именно параметры передавать каждой функции - все в одном state.

### 4. **Лучшая поддержка в IDE**
- Единый тип параметра улучшает автодополнение
- Легче искать использование с помощью Find Usages
- Проще рефакторить

### 5. **Документированность**
В docstring модуля четко описано, какие ключи ожидаются в state, что служит своего рода контрактом API.
