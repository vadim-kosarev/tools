# 20260428.010 - Структурированные рекомендации для уточняющих запросов

**Дата:** 2026-04-28  
**Автор:** AI Agent (Claude)  
**Проблема:** Агент делал хороший анализ в observation, но не использовал выводы для следующей итерации

## Описание проблемы

На этапе observation агент делал отличный анализ:
```
"Найдено достаточно данных для формирования ответа... Однако, для более 
детального ответа требуется дополнительная информация.

Документы 'Общее описание системы Приложение А-(ИИ-26).md', 
'Техническое задание на разработку ПО.md' являются основными источниками..."
```

❌ **Проблема:** Эта информация оставалась в виде свободного текста и НЕ использовалась для:
- Принятия решения о продолжении (refine node)
- Выбора конкретных инструментов (action node на 2-й итерации)
- Указания конкретных файлов/разделов для изучения

## Решение

Добавлено **структурированное поле `next_targets`** в observation для передачи конкретных рекомендаций.

### Workflow до изменений:

```
Итерация 1:
  action → multi_term_exact_search
  observation → "Найдены упоминания, но не хватает деталей. Документы: X, Y, Z"
  refine → needs_refinement=True (но нет конкретики ГДЕ искать)
  
Итерация 2:
  action → ??? (агент не знает какие разделы читать)
```

### Workflow после изменений:

```
Итерация 1:
  action → multi_term_exact_search
  observation → 
    "Найдены упоминания..." + 
    next_targets: [
      {source: "file.md", section: "Раздел", relevance: "high", 
       reason: "содержит детальный список ПО"}
    ]
  refine → needs_refinement=True (есть КОНКРЕТНЫЕ разделы для изучения)
  
Итерация 2:
  action → get_section_content(source_file="file.md", section="Раздел")
  observation → "Получен полный список: ..." + next_targets: []
  refine → needs_refinement=False (данных достаточно)
  
final → формируем ответ
```

## Внесенные изменения

### 1. Python код (rag_lg_agent.py)

**Добавлено поле `next_targets` в AgentObservation:**

```python
class AgentObservation(BaseModel):
    status: Literal["observation"] = Field(default="observation")
    step: int
    thought: str
    observation: str
    next_targets: list[RecommendedSection] = Field(
        default_factory=list,
        description="Рекомендованные разделы для детального изучения"
    )
```

**Использует существующую модель RecommendedSection:**
```python
class RecommendedSection(BaseModel):
    source: str  # Файл
    section: str  # Раздел
    relevance: Literal["high", "medium"]  # Насколько важно
    reason: str  # Почему важно
```

### 2. Промпт observation/system.md

**Добавлена инструкция:**

```markdown
🎯 **ОПРЕДЕЛЕНИЕ РАЗДЕЛОВ ДЛЯ ДЕТАЛЬНОГО ИЗУЧЕНИЯ**:
Если результаты содержат упоминания разделов/файлов, но без детальной информации:
- Выпиши КОНКРЕТНЫЕ файлы и разделы, которые нужно прочитать полностью
- Укажи WHY каждый раздел релевантен
- Это поможет на следующей итерации использовать get_section_content
```

**Обновлен формат JSON:**

```json
{
  "status": "observation",
  "step": 3,
  "thought": "...",
  "observation": "...",
  "next_targets": [
    {
      "source": "Общее описание системы Приложение А-(ИИ-26).md",
      "section": "Сводная таблица устанавливаемого ПО",
      "relevance": "high",
      "reason": "содержит детальное описание установленного ПО"
    }
  ]
}
```

**Пустой массив `next_targets: []`** означает что данных достаточно для финального ответа.

### 3. Промпт refine/system.md

**Добавлена инструкция использовать next_targets:**

```markdown
🎯 **ПРИОРИТЕТ: Используй next_targets из observation!**
Если в предыдущем observation были указаны конкретные разделы:
- Это КОНКРЕТНЫЕ рекомендации куда смотреть дальше
- Используй get_section_content для получения ПОЛНОГО текста этих разделов
- Включи эти разделы в refinement_plan
```

### 4. Промпт action/system.md (для итерации 2+)

**Добавлен блок для уточняющих запросов:**

```markdown
🎯 **Используй TARGETED TOOLS для уточнения:**

**ПРИОРИТЕТ: Если в observation были указаны next_targets:**
- Используй get_section_content для получения ПОЛНОГО содержимого этих разделов
- ИЛИ exact_search_in_file_section для поиска терминов в этих разделах
- Параметры source_file и section берутся из next_targets
```

## Преимущества

### 1. Структурированная передача информации

**До:** "Документы X, Y, Z являются основными источниками" (свободный текст)  
**После:** Конкретный список файлов/разделов с relevance и reason

### 2. Автоматический выбор инструментов

Refine и action nodes **автоматически** используют next_targets для:
- Принятия решения о продолжении
- Выбора правильных инструментов (get_section_content, exact_search_in_file_section)
- Указания правильных параметров (source_file, section)

### 3. Меньше галлюцинаций

Агент не придумывает разделы "из головы" - использует **конкретные разделы из метаданных** результатов поиска.

### 4. Прозрачность

В логах четко видно:
- Какие разделы агент планирует изучить
- Почему он считает их релевантными
- Насколько они важны (high/medium)
