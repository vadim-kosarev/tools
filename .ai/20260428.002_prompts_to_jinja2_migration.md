# Отчет: Миграция промптов в .md файлы с Jinja2

**Дата:** 2026-04-28  
**Тип изменений:** Архитектурный рефакторинг  
**Затронутые компоненты:** RAG/prompts system

---

## Цель

Вынести все промпты из Python-кода в отдельные `.md` файлы с использованием **Jinja2** шаблонизатора для:
1. Красивого отображения Markdown разметки в редакторе
2. Видимых placeholders `{{ variable }}`
3. Возможности включения одного промпта в другой `{% include %}`
4. Автоматической замены placeholders на данные из state

### 2. Все промпты используют Jinja2 синтаксис

#### Переменные
```jinja2
Вопрос пользователя: {{ user_query }}
Итерация: {{ iteration }}/{{ MAX_ITERATIONS }}
```

#### Включение файлов
```jinja2
{% include 'system_prompt_base.md' %}

ТЕКУЩИЙ ЭТАП: plan
...
```

#### Условия
```jinja2
План {% if refinement_plan %}уточнения{% else %}поиска{% endif %}:
```

#### Циклы
```jinja2
{% for step in plan %}
{{ loop.index }}. {{ step }}
{% endfor %}
```

### 3. Создан PromptLoader (prompt_loader.py)

Класс для загрузки и рендеринга Jinja2 шаблонов:

```python
from prompt_loader import get_loader

loader = get_loader()

# Универсальный метод
text = loader.render('plan/system.md', state)

# Удобные методы для каждого типа промпта
text = loader.render_plan_system(state)
text = loader.render_plan_user(state)
text = loader.render_action_system(state)
# ... и т.д.
```

**Особенности:**
- Использует `FileSystemLoader` для загрузки из `RAG/prompts/`
- Поддерживает все возможности Jinja2 (условия, циклы, фильтры)
- `trim_blocks=True` и `lstrip_blocks=True` для чистого вывода
- Singleton pattern через `get_loader()` для эффективности

### 4. Обновлен prompts.py

Теперь это тонкая обертка над PromptLoader для обратной совместимости:

```python
from prompt_loader import get_loader

_loader = get_loader()

def get_plan_system_prompt(state: dict) -> str:
    return _loader.render_plan_system(state)
```

**Преимущества:**
- Сохранена единая сигнатура `(state: dict)` для всех функций
- API осталось прежним - код в `rag_lg_agent.py` работает без изменений
- Функции стали однострочными - весь код в `.md` файлах

### 5. Обновлен rag_lg_agent.py

Минимальные изменения в `_load_system_prompt()`:

```python
def _load_system_prompt() -> str:
    # Подготовка списка инструментов
    tools_formatted = _build_tools_json(compact=True)
    # ... форматирование ...
    
    # Создаем state для рендеринга
    state = {'available_tools': tools_formatted}
    
    # Jinja2 автоматически заменит {{ available_tools }}
    prompt = prompts.get_system_prompt_base(state)
    
    return prompt
```

**Изменилось:**
- Вместо ручной замены `.replace()` используется Jinja2
- Placeholder `{{ available_tools }}` в шаблоне заменяется автоматически

## Преимущества новой системы

### 1. ✅ Читаемость

**Было** (Python код):
```python
def get_action_system_prompt(state: dict) -> str:
    system_prompt = state.get('system_prompt', '')
    iteration = state.get('iteration', 1)
    MAX_ITERATIONS = state.get('MAX_ITERATIONS', 10)
    if state.get('refinement_plan'):
        plan_text = "\n".join(f"{i+1}. {s}" for i, s in enumerate(state['refinement_plan']))
    else:
        plan_text = "\n".join(f"{i+1}. {s}" for i, s in enumerate(state.get('plan', [])))
    return f"""{system_prompt}

ТЕКУЩИЙ ЭТАП: action (итерация {iteration}/{MAX_ITERATIONS})
План: {plan_text}
..."""
```

**Стало** (.md файл):
```jinja2
{% include 'system_prompt_base.md' %}

ТЕКУЩИЙ ЭТАП: action (итерация {{ iteration }}/{{ MAX_ITERATIONS }})

План:
{% for step in plan %}
{{ loop.index }}. {{ step }}
{% endfor %}
```

### 2. ✅ Markdown подсветка в IDE

- Заголовки, списки, код-блоки подсвечиваются корректно
- Placeholders `{{ variable }}` видны сразу
- Preview доступен в любом редакторе

### 3. ✅ Переиспользование промптов


Базовый промпт включается во все промпты без дублирования кода.
