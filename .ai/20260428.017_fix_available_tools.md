# 20260428.017 - Исправление: available_tools не передавался в state

**Дата:** 2026-04-28  
**Автор:** AI Agent (Claude)  
**Проблема:** В первом запросе секция [TOOLS] была пустой, хотя инструменты были определены

## Описание проблемы

При анализе лога `002_llm_plan_request.log` обнаружено:

```
[TOOLS]


[MESSAGES]
```

Секция **[TOOLS] пустая!** Должен быть JSON со списком инструментов.

### Корневая причина

В функциях `plan_node`, `action_node`, `observation_node`, `refine_node`, `final_node`:

```python
# Подготовка state для промптов
state['system_prompt'] = _SYSTEM_PROMPT
state['MAX_ITERATIONS'] = MAX_ITERATIONS
# ❌ available_tools НЕ добавлялся!

system_message = prompts.get_plan_system_prompt(state)
user_message = prompts.get_plan_user_prompt(state)
```

При рендеринге Jinja2 шаблона `plan/prompt.md`:

```markdown
[TOOLS]
{{ available_tools }}
```

`available_tools` был `None` или пустым, поэтому ничего не отображалось.

## Решение

### 1. Создана функция `_get_available_tools()`

Выделена логика генерации форматированного JSON инструментов:

```python
def _get_available_tools() -> str:
    """
    Возвращает форматированный JSON со списком доступных инструментов.
    
    Используется для передачи в state['available_tools'] перед генерацией промптов.
    """
    tools_json_compact = _build_tools_json(compact=True)
    # Форматируем JSON для читаемости
    try:
        tools_list = json.loads(tools_json_compact)
        # Красиво форматируем: каждый инструмент на новой строке
        tools_formatted = "[\n  " + ",\n  ".join(
            json.dumps(t, ensure_ascii=False, separators=(',', ':'))
            for t in tools_list
        ) + "\n]"
    except Exception:
        tools_formatted = tools_json_compact
    
    return tools_formatted
```

### 2. Обновлена `_load_system_prompt()`

Упрощена для использования новой функции:

```python
def _load_system_prompt() -> str:
    """Загружает базовый системный промпт."""
    state = {
        'available_tools': _get_available_tools()  # ← использует новую функцию
    }
    
    prompt = prompts.get_system_prompt_base(state)
    return prompt
```

## Почему это критично

### Без available_tools:

❌ Агент НЕ видит список доступных инструментов
❌ Не знает какие параметры принимают инструменты  
❌ Не может правильно спланировать действия
❌ Может вызвать несуществующие инструменты или с неправильными параметрами

### С available_tools:

✅ Агент видит полный список инструментов (15 штук)
✅ Знает точные имена параметров (substring, source_file, terms, и т.д.)
✅ Может правильно выбрать нужный инструмент
✅ Формирует корректные JSON с вызовами
