# Отчёт: Исправление organize.py - проблема с номерами телефонов

## Дата выполнения
2026-05-08

## Проблема
Скрипт `organize.py` пропускал большинство CallRec файлов с выводом "Skipped" без объяснения причины.

**Примеры пропущенных файлов:**
```
INFO: Skipped [Natalia Kosareva]_[+79200053440]_2026-05-06_14-44-10.wav
INFO: Skipped [+7 985 655-44-08]_[+79856554408]_2026-05-07_15-06-49.wav
INFO: Skipped [Diman Sorokin]_[+79043934099]_2026-05-07_18-12-20.wav
```

## Корень проблемы
Регулярное выражение `r'(\d{4})[\.-_](\d{2})[\.-_](\d{2})'` находило **даты в номерах телефонов**, а не в временных меткам файлов!

**Пример интерпретации:**
- Файл: `[+7 495 809-31-78]_[+79856554408]_2026-05-07_15-06-49.wav`
- Регулярное выражение находило: `7495 809 31` → год=`7495`, месяц=`80`, день=`93`
- Проверка: год 7495 > 2026 (текущий год) → **SKIP!**

## Решение

### 1. Добавлен детальный вывод диагностики
- Сообщает текущую рабочую директорию
- Выводит точную причину пропуска каждого файла (год, месяц, дата)
- Логирует успешное перемещение файлов

### 2. Переработано регулярное выражение
**Было:**
```python
date_pattern = re.compile(r'(\d{4})[\.-_](\d{2})[\.-_](\d{2})')
```

**Стало:**
```python
# Использует backreference \2 для гарантирования одного и того же разделителя
# Paттерн ищет: _YYYY-MM-DD_HH-MM-SS (или с дефисами/точками)
# Это гарантирует, что мы ищем именно дату+время, а не фрагменты из номер телефона
date_pattern = re.compile(r'_(\d{4})([-._])(\d{2})\2(\d{2})_\d{2}\2\d{2}\2\d{2}')
```

**Как это работает:**
- `_` - подчеркивание перед датой (гарантирует, что это разделитель между телефоном и датой)
- `(\d{4})` - год (группа 1)
- `([-._])` - разделитель (группа 2, может быть -, . или _)
- `(\d{2})` - месяц (группа 3)
- `\2` - повтор того же разделителя (backreference)
- `(\d{2})` - день (группа 4)
- `_\d{2}\2\d{2}\2\d{2}` - время (часы, минуты, секунды с тем же разделителем)

### 3. Обновлена логика извлечения групп
```python
year, _, month, day = m.groups()  # _ это разделитель, его пропускаем
```

## Дополнительные улучшения (обновление 2026-05-08)

### Поддержка SmartRecorder формата
Первоначально регулярное выражение требовало время в имени файла, что подходило для CallRec, но не для SmartRecorder.

**SmartRecorder формат:** `recording_YYYY.MM.DD-NN.wav` (без времени)

**Решение:** Время сделано необязательным в регулярном выражении:
```python
# До: требовало время
date_pattern = re.compile(r'_(\d{4})([-._])(\d{2})\2(\d{2})_\d{2}\2\d{2}\2\d{2}')

# После: время опциональное (для SmartRecorder)
date_pattern = re.compile(r'_(\d{4})([-._])(\d{2})\2(\d{2})(?:_\d{2}\2\d{2}\2\d{2})?')
```

**Результат:** Скрипт теперь поддерживает оба формата:
- ✅ CallRec: `[+7 985 655-44-08]_[+79856554408]_2026-05-07_15-06-49.wav`
- ✅ SmartRecorder: `recording_2026.05.04-60.wav`
