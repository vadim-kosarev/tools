# Динамическое пересканирование директории в t_directory.py

**Дата:** 2026-02-22  
**Файл:** `transcribe/t_directory.py`

## 🎯 ЗАДАЧА

Добавить динамическое пересканирование директории после каждого обработанного файла. Это необходимо, чтобы всегда обрабатывать самый свежий файл, даже если во время обработки появились новые файлы.

## 💡 ПРОБЛЕМА

### Было:
```python
# 1. Сканируем директорию ОДИН РАЗ в начале
media_files = collect_media_files(directory, skip_existing, revision)
# → Получаем список: [file1.amr, file2.amr, file3.amr]

# 2. Обрабатываем по порядку
for file in media_files:
    process(file)
    # Во время обработки file1 может появиться file4.amr (более свежий)
    # Но мы продолжим обработку file2, потому что список уже построен!
```

**Проблема:** Новые файлы, появившиеся во время работы, будут обработаны не в порядке свежести.

### Сценарий (CallRec):
```
09:00 - Запускаем скрипт
        Список: [call1.amr (08:30), call2.amr (08:45)]
09:00 - Начинаем обработку call1.amr
09:05 - Заканчиваем call1.amr
09:05 - Появился call3.amr (09:04) - САМЫЙ СВЕЖИЙ!
09:05 - Но скрипт обрабатывает call2.amr (08:45) - СТАРЫЙ!
        ❌ Неправильный порядок приоритетов
```

---

## ✅ РЕШЕНИЕ

Динамическое пересканирование после каждого файла:

```python
stats = {"total": 0, "success": 0, "failed": 0}

# Цикл с динамическим пересканированием
while True:
    # ✅ Пересканируем директорию КАЖДЫЙ РАЗ
    media_files = collect_media_files(directory, skip_existing, revision)
    
    if not media_files:
        break  # Нет больше файлов
    
    # ✅ Берём ПЕРВЫЙ файл (самый свежий из отсортированных)
    file_path = media_files[0]
    stats["total"] += 1
    
    logger.info(f"[{stats['total']}] Обработка (осталось: {len(media_files)})...")
    
    # Обрабатываем файл
    success = process_single_file(file_path, revision, device, t_gigaam_script)
    
    if success:
        stats["success"] += 1
    else:
        stats["failed"] += 1
    
    # ✅ Цикл повторяется → пересканирование → новый список
```

## 🎯 ПРЕИМУЩЕСТВА

### ✅ Всегда актуальный приоритет:
- Самые свежие файлы обрабатываются первыми
- Даже если они появились во время работы скрипта
- Важно для CallRec и других систем реального времени

### ✅ Адаптивность:
- Скрипт "видит" новые файлы сразу после их появления
- Не нужно перезапускать скрипт
- Автоматическая обработка новых записей

### ✅ Правильные приоритеты:
- Последние звонки обрабатываются в первую очередь
- Старые записи обрабатываются позже
- Логичное поведение для мониторинга

### ✅ Информативность:
- Показывает количество оставшихся файлов
- Динамически обновляется при появлении новых
- Видно реальное состояние директории

## 🔧 ТЕХНИЧЕСКИЕ ДЕТАЛИ

### Алгоритм:

1. **Пересканирование:**
   ```python
   media_files = collect_media_files(directory, skip_existing, revision)
   ```
   - Поиск всех медиафайлов в директории
   - Фильтрация обработанных (если `skip_existing=True`)
   - Сортировка по времени модификации (новые → старые)

2. **Выбор файла:**
   ```python
   file_path = media_files[0]  # Самый свежий
   ```
   - Берём первый файл из отсортированного списка
   - Это всегда самый свежий необработанный файл

3. **Обработка:**
   ```python
   success = process_single_file(file_path, ...)
   ```

4. **Повторение:**
   - Цикл `while True` возвращается к шагу 1
   - Новое пересканирование
   - Новый список файлов

## ⚙️ ВЛИЯНИЕ НА ПРОИЗВОДИТЕЛЬНОСТЬ

### Overhead пересканирования:

**Операции:**
- `Path.rglob('*')` - рекурсивный поиск файлов
- Фильтрация по расширению
- Проверка существования транскрипции
- Сортировка по `st_mtime`

**Время:**
- 100 файлов: ~10-50 мс
- 1000 файлов: ~50-200 мс
- 10000 файлов: ~200-1000 мс

**Вывод:** Незначительно по сравнению со временем транскрипции (минуты/файл)

## 🎯 ИТОГОВАЯ СТАТИСТИКА

### Счётчик `stats["total"]`:

**Раньше:**
```python
stats = {"total": len(media_files), ...}  # Фиксированное число в начале
```

**Теперь:**
```python
stats = {"total": 0, ...}
stats["total"] += 1  # Увеличивается с каждым файлом
```

**Результат:** `total` показывает **количество обработанных файлов**, а не изначальное количество в списке.

### Пример вывода:

```
================================================================================
ИТОГОВАЯ СТАТИСТИКА:
  Всего файлов: 7
  Успешно: 7
  Ошибок: 0
  Процент успеха: 100.0%
================================================================================
```

**Примечание:** Если во время работы появилось 2 новых файла, и все обработаны, то `total = 7` (5 изначальных + 2 новых).
