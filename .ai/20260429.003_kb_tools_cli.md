## Краткое описание

Добавлен CLI (Command Line Interface) интерфейс для модуля `kb_tools.py`. Теперь инструменты можно вызывать напрямую из командной строки без написания Python скриптов.

**Обновление:** Упрощен интерфейс, добавлена команда `help` для детальной справки.

## Команды

### 1. Без параметров или `help` - показывает общий help

```bash
python kb_tools.py
python kb_tools.py help
```

**Вывод:**
- Список команд
- Примеры использования
- Формат параметров

### 2. Команда `list` - список инструментов

```bash
python kb_tools.py list
```

**Вывод:**
- Название инструмента
- Краткий список параметров (обязательные и [опциональные])

**Пример вывода:**
```
📦 find_abbreviation_expansion
   Параметры: abbreviation, [max_results]

📦 exact_search
   Параметры: substring, [limit], [chunk_type], [source], [section]
```

### 3. Команда `help <tool>` - детальная справка

```bash
python kb_tools.py help find_abbreviation_expansion
```

**Вывод:**
- Полное описание инструмента
- Детальная информация о каждом параметре:
  - Тип данных
  - Обязательность (✓ обязательный / ○ опциональный)
  - Значение по умолчанию
  - Полное описание параметра
- Пример использования

**Пример вывода:**
```
================================================================================
Инструмент: find_abbreviation_expansion
================================================================================

Описание:
  Find expansions (расшифровки) of abbreviations in knowledge base...

Параметры:

  abbreviation=str
    ✓ обязательный
    Abbreviation in CAPITAL LETTERS to find expansion for...

  max_results=int
    ○ опциональный (default=50)
    Max matches to return
```

### 4. Команда `run` - запуск инструмента

```bash
python kb_tools.py run <tool_name> param1=value1 param2=value2
```

**Вывод:**
- JSON результат в stdout
- Сообщения об инициализации в stderr

## Формат параметров: `param=value`

**Преимущества:**
- ✅ Простой парсинг
- ✅ Работает с кириллицей без экранирования
- ✅ Автоматическая конвертация чисел
- ✅ Поддержка JSON для сложных структур

**Примеры:**

```bash
# Строка
python kb_tools.py run exact_search substring=КЦОИ

# Число
python kb_tools.py run exact_search substring=КЦОИ limit=10

# JSON массив (для multi_term_exact_search)
python kb_tools.py run multi_term_exact_search terms='["term1","term2","term3"]'
```

## Реализация

### 1. Функция `_cli_list_tools` (упрощенная)

Выводит краткий список инструментов с именами параметров:

```python
def _cli_list_tools(tools: list[BaseTool]) -> None:
    """Выводит краткий список инструментов с именами параметров"""
    for tool in tools:
        print(f"📦 {tool.name}")
        # Собираем имена параметров
        param_names = []
        for field_name, field_info in schema.model_fields.items():
            if field_info.is_required():
                param_names.append(field_name)
            else:
                param_names.append(f"[{field_name}]")
        print(f"   Параметры: {', '.join(param_names)}")
```

**Формат вывода:**
- Обязательные параметры: `param_name`
- Опциональные параметры: `[param_name]`

### 2. Функция `_cli_help_tool` (новая)

Выводит детальную справку по конкретному инструменту:

```python
def _cli_help_tool(tools: list[BaseTool], tool_name: str) -> None:
    """Выводит детальную справку по инструменту"""
    tool = next((t for t in tools if t.name == tool_name), None)
    
    print(f"Инструмент: {tool.name}")
    print(f"Описание: {tool.description}")
    print("Параметры:")
    
    for field_name, field_info in schema.model_fields.items():
        required = "✓ обязательный" if field_info.is_required() else f"○ опциональный (default={field_info.default})"
        print(f"  {field_name}={type}")
        print(f"    {required}")
        print(f"    {field_info.description}")
```

**Формат вывода:**
- ✓ обязательный
- ○ опциональный (default=значение)

### 3. Функция `_cli_run_tool`

Запускает инструмент и выводит JSON результат:

```python
def _cli_run_tool(tools: list[BaseTool], tool_name: str, args: dict) -> None:
    """Запускает инструмент с заданными аргументами"""
    tool = next((t for t in tools if t.name == tool_name), None)
    result = tool.invoke(args)
    # Вывод JSON
    if hasattr(result, 'model_dump'):
        print(json.dumps(result.model_dump(), ensure_ascii=False, indent=2))
```

### 4. Функция `main`

Основная CLI логика с тремя командами:

```python
def main():
    """CLI интерфейс для kb_tools"""
    parser = argparse.ArgumentParser(...)
    subparsers = parser.add_subparsers(dest='command')
    
    # Команда list
    list_parser = subparsers.add_parser('list', help='Список инструментов')
    
    # Команда help
    help_parser = subparsers.add_parser('help', help='Справка по инструменту')
    help_parser.add_argument('tool_name', nargs='?', help='Название инструмента')
    
    # Команда run
    run_parser = subparsers.add_parser('run', help='Запустить инструмент')
    run_parser.add_argument('tool_name')
    run_parser.add_argument('params', nargs='*')
    
    # Парсинг аргументов
    if len(sys.argv) == 1:
        parser.print_help()  # Без параметров - help
        sys.exit(0)
    
    args = parser.parse_args()
    
    # Инициализация tools (только для нужных команд)
    tools = None
    if args.command in ['list', 'run'] or (args.command == 'help' and args.tool_name):
        vectorstore = build_vectorstore()
        tools = create_kb_tools(vectorstore, knowledge_dir)
    
    # Выполнение команд
    if args.command == 'list':
        _cli_list_tools(tools)
    
    elif args.command == 'help':
        if args.tool_name:
            _cli_help_tool(tools, args.tool_name)
        else:
            parser.print_help()
    
    elif args.command == 'run':
        # Парсинг параметров формата param=value
        tool_args = {}
        for param in args.params:
            if '=' in param:
                key, value = param.split('=', 1)
                # Автоконвертация чисел и JSON
                ...
        _cli_run_tool(tools, args.tool_name, tool_args)
```

**Ключевые особенности:**
- Три команды: `list`, `help`, `run`
- `help` без параметров → общий help
- `help <tool>` → детальная справка
- Инициализация vectorstore только когда нужно

## Технические детали

### Автоматическая конвертация типов

```python
# Строка → число
"10" → 10
"3.14" → 3.14

# Строка → JSON
'["a","b"]' → ["a", "b"]
'{"key":"value"}' → {"key": "value"}

# Остальное → строка
"КЦОИ" → "КЦОИ"
"что такое RAG" → "что такое RAG"
```

### Вывод stdout vs stderr

- **stdout** - только JSON результат (для pipe в другие программы)
- **stderr** - логи, сообщения об инициализации, ошибки

**Пример использования в pipe:**
```powershell
python kb_tools.py run exact_search substring=КЦОИ 2>$null | ConvertFrom-Json
```

### Обработка ошибок

```python
try:
    result = tool.invoke(args)
except Exception as e:
    print(f"Ошибка выполнения: {e}", file=sys.stderr)
    traceback.print_exc()
    sys.exit(1)
```

## Ограничения

1. **Инициализация**
   - Требуется доступ к ClickHouse
   - Требуется settings.py с конфигурацией
   - ~1-2 секунды на старт

2. **Сложные параметры**
   - Списки нужно передавать как JSON строки
   - Пробелы в значениях нужно экранировать

3. **Вывод**
   - JSON всегда выводится с отступами (не минифицированный)
   - Кириллица может отображаться как unicode-escape в некоторых терминалах
