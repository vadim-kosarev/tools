# Защита от зацикливания на файлах с ошибками

**Дата:** 2026-02-22  
**Файл:** `transcribe/t_directory.py`

## 🐛 ПРОБЛЕМА

При динамическом пересканировании директории скрипт может зациклиться на файле с ошибкой:

```
09:00 - Сканирование: [broken.amr, file2.amr]
09:00 - Обработка broken.amr → ОШИБКА ❌
09:01 - Сканирование: [broken.amr, file2.amr]  ← broken.amr снова в списке!
09:01 - Обработка broken.amr → ОШИБКА ❌       ← Зацикливание!
09:02 - Сканирование: [broken.amr, file2.amr]
09:02 - Обработка broken.amr → ОШИБКА ❌       ← Бесконечный цикл!
...
```

**Проблема:** Файл с ошибкой не создаёт транскрипцию → остаётся в списке необработанных → обрабатывается снова → зацикливание.

---

## ✅ РЕШЕНИЕ

Добавлен механизм запоминания файлов с ошибками:

```python
# Множество путей к файлам с ошибками
failed_files = set()

while True:
    # Пересканируем директорию
    media_files = collect_media_files(directory, skip_existing, revision)
    
    # ✅ Фильтруем файлы с ошибками
    media_files = [f for f in media_files if f not in failed_files]
    
    if not media_files:
        break
    
    file_path = media_files[0]
    success = process_single_file(file_path, ...)
    
    if success:
        stats["success"] += 1
    else:
        stats["failed"] += 1
        # ✅ Запоминаем файл с ошибкой
        failed_files.add(file_path)
        logger.debug(f"Файл добавлен в список проваленных: {file_path.name}")
```

## 🎯 ЛОГИКА РАБОТЫ

### 1. Инициализация множества проваленных файлов:

```python
failed_files = set()  # Пустое множество в начале
```

**Почему `set`?**
- Быстрая проверка принадлежности: O(1)
- Автоматическая уникальность (нет дубликатов)
- Эффективность при большом количестве файлов

---

### 2. Фильтрация списка файлов:

```python
media_files = collect_media_files(directory, skip_existing, revision)
# Убираем файлы с ошибками из списка
media_files = [f for f in media_files if f not in failed_files]
```

**Эффект:**
- Файлы с ошибками не попадут в обработку повторно
- Скрипт продолжит работу с другими файлами
- Нет зацикливания

---

### 3. Запоминание файла с ошибкой:

```python
if success:
    stats["success"] += 1
else:
    stats["failed"] += 1
    failed_files.add(file_path)  # Добавляем в множество
    logger.debug(f"Файл добавлен в список проваленных: {file_path.name}")
```

---

### 4. Отчёт о проваленных файлах:

```python
if failed_files:
    logger.warning(
        f"Пропущено файлов с ошибками: {len(failed_files)}\n"
        f"Эти файлы не были обработаны повторно, чтобы избежать зацикливания"
    )
```

## 🔧 ТЕХНИЧЕСКИЕ ДЕТАЛИ

### Структура данных `failed_files`:

```python
failed_files = set()  # Пустое множество

# Добавление элемента
failed_files.add(Path("broken.amr"))

# Проверка принадлежности
if Path("broken.amr") in failed_files:
    print("Файл в списке проваленных")

# Размер множества
len(failed_files)  # Количество проваленных файлов
```

### Производительность:

| Операция | Сложность | Время (1000 файлов) |
|----------|-----------|---------------------|
| `add()` | O(1) | < 0.001 мс |
| `in` | O(1) | < 0.001 мс |
| list comprehension | O(n) | ~0.1 мс |

**Вывод:** Overhead минимален, защита работает эффективно.

## 🎯 ПРЕИМУЩЕСТВА

### ✅ Защита от зацикливания:
- Файл с ошибкой обрабатывается только один раз
- Скрипт не застревает на проблемных файлах
- Продолжает работу с остальными файлами

### ✅ Информативность:
- Логирование добавления в список проваленных (DEBUG)
- Предупреждение о количестве пропущенных файлов
- Видно, какие файлы не обработались

### ✅ Правильная статистика:
- `total` включает все попытки обработки
- `failed` включает файлы с ошибками
- `success` включает успешные обработки
- Процент успеха рассчитывается корректно

### ✅ Автоматическое восстановление:
- Если файл исправить вручную и удалить пустую транскрипцию
- При следующем запуске скрипта файл будет обработан
- Множество `failed_files` существует только в рамках одного запуска

## 🔄 ЖИЗНЕННЫЙ ЦИКЛ

### Время жизни `failed_files`:

1. **Создание:** В начале `process_directory()`
   ```python
   failed_files = set()
   ```

2. **Наполнение:** При ошибках обработки
   ```python
   if not success:
       failed_files.add(file_path)
   ```

3. **Использование:** При каждом пересканировании
   ```python
   media_files = [f for f in media_files if f not in failed_files]
   ```

4. **Уничтожение:** При завершении `process_directory()`
   - Множество уничтожается
   - При новом запуске создаётся заново

**Важно:** `failed_files` живёт только в рамках одного запуска скрипта!
