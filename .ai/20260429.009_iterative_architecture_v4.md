## Краткое описание

Возвращен итеративный функционал в RAG-агент. Теперь агент выполняет план поэтапно с проверкой выполнения каждого шага и принятием решения о продолжении или завершении.

---

## Архитектурные изменения

### Было (v3 - линейная)

```
START → planner → tool_selector → tool_executor → analyzer → refiner → final → END
```

- **1 проход** - всегда выполняется полностью без возврата
- **Refiner** - без LLM, всегда переходит к final
- **MAX_ITERATIONS = 1**

### Стало (v4 - итеративная)

```
START → planner → tool_selector → tool_executor → analyzer → refiner
                       ↑                                        ↓
                       +-------------[продолжить]---------------+
                                                                ↓
                                                           [завершено]
                                                                ↓
                                                             final → END
```

- **До 5 итераций** - выполнение плана по шагам
- **Refiner с LLM** - анализирует выполнение и принимает решение
- **MAX_ITERATIONS = 5**

## Детали реализации

### 1. Обновлен AgentState

Добавлены новые поля для отслеживания прогресса:

```python
class AgentState(TypedDict, total=False):
    user_query: str
    plan: list[str]
    current_step_index: int      # ← НОВОЕ: 0-based индекс текущего шага
    iteration: int               # ← НОВОЕ: 1-based номер итерации
    tool_instructions: list[dict]
    context: list[dict]
    history: list[dict]
    next_node: str               # ← ИЗМЕНЕНО: "tool_selector" или "final"
    final_answer: str
```

### 2. Добавлена Pydantic модель AgentRefiner

```python
class AgentRefiner(BaseModel):
    status: Literal["refiner"] = Field(default="refiner")
    step: int
    thought: str
    current_step_completed: bool
    all_plan_completed: bool
    next_action: Literal["continue", "final"]
    reason: str
```

### 3. Переписан refiner_node

**Было:**
- Без LLM
- Всегда next_node = "final"
- Простая логика

**Стало:**
- **С LLM** - анализирует выполнение шага
- **Проверяет**:
  - Выполнен ли текущий шаг
  - Остались ли шаги в плане
  - Собрано ли достаточно данных
  - Не превышен ли лимит итераций
- **Принимает решение**:
  - `continue` → инкрементирует current_step_index, переход к tool_selector
  - `final` → переход к формированию ответа

### 4. Обновлен planner_node

Инициализирует новые поля:

```python
state["plan"] = result.plan
state["current_step_index"] = 0  # ← НОВОЕ
state["iteration"] = 1           # ← НОВОЕ
```

### 5. Обновлен build_graph

Добавлен условный роутинг:

```python
def route_from_refiner(state: AgentState) -> str:
    return state.get("next_node", "final")

workflow.add_conditional_edges(
    "refiner",
    route_from_refiner,
    {
        "tool_selector": "tool_selector",  # Цикл
        "final": "final"                   # Выход
    }
)
```

### 6. Создана папка prompts_v2/refiner/

Новые промпты для refiner узла:

- **system.md** - инструкции по анализу выполнения шага и принятию решения
- **user.md** - контекст текущего шага, observation, прогресс
- **retry.md** - промпт для повторной попытки при ошибке

### 7. Обновлен prompt_loader.py

Добавлены методы:

```python
def render_refiner_system(self, state: Dict[str, Any]) -> str
def render_refiner_user(self, state: Dict[str, Any]) -> str
def render_refiner_retry(self, state: Dict[str, Any], extra: Dict[str, Any] = None) -> str
```

## Логика работы refiner

### Критерии продолжения (continue)

```python
if (current_step_completed and has_more_steps) and iteration < MAX_ITERATIONS:
    next_action = "continue"
    current_step_index += 1  # Переход к следующему шагу
    iteration += 1
```

### Критерии завершения (final)

```python
if all_plan_completed or iteration >= MAX_ITERATIONS or sufficient_data:
    next_action = "final"
```

### Пример работы

**План из 3 шагов:**
1. Найти упоминания СУБД (semantic_search + exact_search)
2. Найти IP-адреса серверов (regex_search)
3. Сопоставить СУБД с IP (multi_term_exact_search)

**Выполнение:**

```
Итерация 1: planner → tool_selector → tool_executor → analyzer → refiner
            refiner: "Шаг 1 выполнен, есть шаг 2" → continue → tool_selector

Итерация 2: tool_selector → tool_executor → analyzer → refiner
            refiner: "Шаг 2 выполнен, есть шаг 3" → continue → tool_selector

Итерация 3: tool_selector → tool_executor → analyzer → refiner
            refiner: "Шаг 3 выполнен, все шаги завершены" → final

final → END
```

## Потенциальные проблемы и решения

### 1. Зацикливание

**Проблема:** Refiner может продолжать циклы без необходимости

**Решение:**
- Жесткий лимит MAX_ITERATIONS = 5
- LLM явно проверяет достаточность данных
- Логирование каждого решения

### 2. Избыточные итерации

**Проблема:** Может делать больше итераций чем нужно

**Решение:**
- Refiner анализирует достаточность данных
- Может завершиться раньше если all_plan_completed=true

### 3. Производительность

**Проблема:** Больше вызовов LLM = медленнее

**Решение:**
- Для простых вопросов завершается на 1-й итерации
- Можно вернуться к v3 если нужна скорость (закомментировать conditional_edges)
