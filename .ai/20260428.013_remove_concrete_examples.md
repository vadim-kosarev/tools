# 20260428.013 - Удаление конкретных примеров из промптов

**Дата:** 2026-04-28  
**Автор:** AI Agent (Claude)  
**Проблема:** Даже с явными маркерами LLM продолжал путать конкретные примеры с реальными вопросами

## Описание проблемы

После добавления маркеров 🎯 и предупреждений, LLM все равно отвечал на примеры:

```bash
python rag_lg_agent.py "что такое smart monitor"

# Ответ:
"Пользователь спрашивает о ПО, установленном на АРМ оператора КЦОИ..."
```

### Корневая причина

В промптах были **очень детальные конкретные примеры** с длинными списками терминов:

```json
"terms": [
  "ПО", "программное обеспечение", "программные средства", "софт",
  "АРМ оператора", "АРМ", "рабочее место", "рабочая станция",
  "установлено", "развернуто", "настроено",
  // ... еще 10+ терминов ...
]
```

LLM воспринимал эти **детальные списки** как реальную задачу, игнорируя реальный вопрос пользователя.

## Решение

**УДАЛЕНЫ все конкретные примеры**, заменены на **абстрактные плейсхолдеры**.

## Почему это работает

### Психология LLM

LLM обрабатывает промпт так:

1. **Читает system message** (большой, с примерами)
2. **Читает user message** (короткий, реальный вопрос)
3. **Ищет паттерны** между ними

Когда в system message были **детальные конкретные примеры**, LLM видел:

```
System: ... "ПО", "установлено", "АРМ оператора" ...
User: "что такое smart monitor"

LLM думает: 
"В system много про ПО и АРМ → это важная тема
 User вопрос короткий → может он имел в виду про ПО на АРМ?"
```

С **абстрактными плейсхолдерами** LLM видит:

```
System: ... "термин1", "термин2", "синоним1" ...
User: "что такое smart monitor"

LLM думает:
"System показывает ФОРМАТ, как работать с терминами
 User спрашивает про 'smart monitor' → работаю с ЭТИМИ терминами"
```

## Принцип "чем меньше, тем лучше"

### До:
- 20+ конкретных терминов в примерах
- Детальные пошаговые примеры с конкретными значениями
- Конкретные названия файлов ("Общее описание системы...")

### После:
- 6 абстрактных плейсхолдеров
- Общие инструкции без конкретики
- Общие названия ("filename.md", "Section Name")

**Результат:** LLM имеет меньше "соблазна" зацепиться за конкретные примеры.
