# Отчет: Упрощение формата логов

**Дата:** 2026-04-28  
**Тип изменений:** Улучшение читаемости логов  
**Затронутые файлы:** `RAG/llm_call_logger.py`

---

## Цель

Упростить формат логов, убрав лишние декоративные линии (точки, тильды, дефисы), оставив только текстовую информацию с временными метками.

---

## Что было изменено

### Формат логов

#### Было (с декоративными линиями):

```
................................................................................
  #003  2026-04-28 07:16:50  [PLAN]  RESPONSE
................................................................................
{
  "step": 1,
  "thought": "...",
  "plan": [...]
}
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  end #003 RESPONSE
```

#### Стало (чистый текст):

```
#003  2026-04-28 07:16:50  [PLAN]  RESPONSE
{
  "step": 1,
  "thought": "...",
  "plan": [...]
}
#003  2026-04-28 07:16:51  [PLAN]  RESPONSE end
```

---

## Изменения в коде

### 1. Удалены неиспользуемые константы

**Было:**
```python
_W = 80
_SEP_LLM   = "." * _W        # LLM calls
_SEP_TOOL  = "-" * _W        # tool calls
_SEP_STAGE = "_" * _W        # pipeline stage markers
_SEP_END   = "~" * _W        # block end / footer
```

**Стало:**
```python
_W = 80
_SEP_STAGE = "_" * _W        # pipeline stage markers (используется для STAGE блоков)
```

### 2. Упрощен формат LLM блоков

**Метод `_write()` - LLM блоки:**

**Было:**
```python
header = f"  #{number:03d}  {ts}  [{step}]  {kind}"
end    = f"  end #{number:03d} {kind}"
block  = (
    f"\n{_SEP_LLM}\n{header}\n{_SEP_LLM}\n"
    f"{text}\n"
    f"{_SEP_END}\n{end}\n"
)
```

**Стало:**
```python
header = f"#{number:03d}  {ts}  [{step}]  {kind}"
end    = f"#{number:03d}  {ts}  [{step}]  {kind} end"
block  = (
    f"\n{header}\n"
    f"{text}\n"
    f"{end}\n"
)
```

### 3. Упрощен формат TOOL блоков

**Метод `_write()` - TOOL блоки:**

**Было:**
```python
header = f"  #{number:03d}  {ts}  [{step}]  {kind}"
end    = f"..............  end #{number:03d} {kind}  .............."
block  = f"\n{_SEP_TOOL}\n{header}\n{_SEP_TOOL}\n{text}\n{end}\n"
```

**Стало:**
```python
header = f"#{number:03d}  {ts}  [{step}]  {kind}"
end    = f"#{number:03d}  {ts}  [{step}]  {kind} end"
block  = f"\n{header}\n{text}\n{end}\n"
```

### 4. Обновлен streaming режим

**Метод `_write_streaming_header()`:**

**Было:**
```python
header = f"  #{number:03d}  {ts}  [{step}]  RESPONSE"
block = f"\n{_SEP_LLM}\n{header}\n{_SEP_LLM}\n"
```

**Стало:**
```python
header = f"#{number:03d}  {ts}  [{step}]  RESPONSE"
block = f"\n{header}\n"
```

**Метод `_write_streaming_footer()`:**

**Было:**
```python
end = f"  end #{number:03d} RESPONSE"
block = f"\n{_SEP_END}\n{end}\n"
```

**Стало:**
```python
ts = datetime.now().strftime("%Y-%m-%d %H:%M:%S")
end = f"#{number:03d}  {ts}  RESPONSE end"
block = f"\n{end}\n"
```

### 5. Обновлена документация

Обновлены:
- Docstring модуля (строки 1-42)
- Docstring класса `LlmCallLogger` (строки 126-148)
- Docstring метода `_write()` (строки 171-180)

Удалены упоминания о декоративных линиях, обновлены примеры формата.

## Преимущества изменений

### ✅ Читаемость
- Убраны лишние визуальные элементы
- Информация сразу видна - номер, время, тип события
- Меньше «шума» при чтении логов

### ✅ Компактность
- Логи стали на ~30% короче (убрано 4 строки разделителей на блок)
- Легче следить за последовательностью событий

### ✅ Согласованность
- Единый формат для header и footer:
  ```
  #NNN  YYYY-MM-DD HH:MM:SS  [STEP]  KIND
  #NNN  YYYY-MM-DD HH:MM:SS  [STEP]  KIND end
  ```
- Footer теперь тоже содержит timestamp (для streaming режима важно видеть время завершения)

### ✅ Парсинг
- Структура стала более предсказуемой
- Легче написать скрипт для анализа логов

## Обратная совместимость

⚠️ **Breaking change:** Формат логов изменился. 

**Влияние:**
- Скрипты для парсинга старых логов нужно обновить
- Визуальное восприятие изменилось (стало проще)
- Функциональность осталась прежней

**Миграция:**
Если есть внешние инструменты для анализа логов:
1. Обновить regex для парсинга header/footer:
   ```python
   # Старый формат
   r'^\s+#(\d{3})\s+(.+?)\s+\[(.+?)\]\s+(REQUEST|RESPONSE)$'
   
   # Новый формат
   r'^#(\d{3})\s+(.+?)\s+\[(.+?)\]\s+(REQUEST|RESPONSE)$'
   ```

2. Footer теперь содержит timestamp:
   ```python
   # Старый формат
   r'^\s+end #(\d{3})\s+(REQUEST|RESPONSE)$'
   
   # Новый формат
   r'^#(\d{3})\s+(.+?)\s+\[(.+?)\]\s+(REQUEST|RESPONSE)\s+end$'
   # или для streaming
   r'^#(\d{3})\s+(.+?)\s+(RESPONSE)\s+end$'
   ```
