# Отчет: JSON формат для аргументов инструментов в логах

**Дата:** 2026-04-28  
**Тип изменений:** Улучшение формата логирования  
**Затронутые файлы:** `RAG/kb_tools.py`

---

## Цель

Изменить формат логирования аргументов инструментов (tools) с текстового представления на JSON формат для:
- Лучшей читаемости
- Возможности автоматического парсинга
- Единообразия с другими частями логов
- Простоты копирования параметров для повторного вызова

---

## Что было изменено

### Формат логов аргументов

#### Было (текстовый формат):

```
#008  2026-04-28 07:47:22  [DB:exact_search]  REQUEST
substring='установлены следующие'

chunk_type=''
#008  2026-04-28 07:47:22  [DB:exact_search]  REQUEST end
```

#### Стало (JSON формат):

```
#008  2026-04-28 07:47:22  [DB:exact_search]  REQUEST
{
  "substring": "установлены следующие",
  "limit": 30,
  "chunk_type": ""
}
#008  2026-04-28 07:47:22  [DB:exact_search]  REQUEST end
```

## Изменения в коде

### 1. Добавлена helper функция `_format_tool_args`

```python
def _format_tool_args(**kwargs) -> str:
    """Format tool arguments as JSON for logging.
    
    Args:
        **kwargs: Tool arguments to format
        
    Returns:
        JSON formatted string with arguments
    """
    return json.dumps(kwargs, ensure_ascii=False, indent=2)
```

Эта функция:
- Принимает произвольные keyword аргументы
- Форматирует их в JSON с отступами
- Использует `ensure_ascii=False` для корректного отображения русских символов

### 2. Обновлены все 16 вызовов `_db_request`

Все инструменты теперь используют `_format_tool_args` для формирования JSON:

#### exact_search

**Было:**
```python
rec = _db_request(
    "DB:exact_search",
    f"substring={substring!r}\nlimit={limit}\nchunk_type={chunk_type!r}{filter_str}",
)
```

**Стало:**
```python
args_dict = {
    "substring": substring,
    "limit": limit,
    "chunk_type": chunk_type
}
if source:
    args_dict["source"] = source
if section:
    args_dict["section"] = section

rec = _db_request("DB:exact_search", _format_tool_args(**args_dict))
```

## Преимущества изменений

### ✅ Читаемость
- Аргументы четко структурированы
- Типы данных видны сразу (строки в кавычках, числа без кавычек)
- Отступы делают структуру наглядной

### ✅ Парсинг
- Логи легко парсятся стандартными JSON парсерами
- Можно автоматически извлекать параметры вызовов
- Простая интеграция с инструментами анализа логов

### ✅ Копирование параметров
- Можно скопировать JSON и использовать напрямую в коде
- Не нужно преобразовывать из текстового формата

### ✅ Единообразие
- Формат согласован с остальными частями системы
- tool RESPONSE уже использует JSON (SearchChunksResult, etc.)
- Теперь REQUEST тоже в JSON

### ✅ Optional параметры
- Четко видно, какие параметры переданы, а какие нет
- Для инструментов без параметров (list_sources) возвращается `{}`

## Обратная совместимость

⚠️ **Breaking change для парсеров логов:**

Если есть скрипты, которые парсят логи tool вызовов, их нужно обновить:

**Старый формат (regex):**
```python
# Было
pattern = r"substring='([^']+)'\nlimit=(\d+)"
```

**Новый формат (JSON):**
```python
# Стало
import json
import re

# Извлекаем JSON из лога
match = re.search(r'REQUEST\n(.+?)\n#\d+', log, re.DOTALL)
if match:
    args = json.loads(match.group(1))
    substring = args['substring']
    limit = args['limit']
```
