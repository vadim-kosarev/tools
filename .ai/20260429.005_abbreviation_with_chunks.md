# Добавление чанков в результат find_abbreviation_expansion

**Дата:** 2026-04-29  
**Номер:** 20260429.005  
**Модуль:** RAG/kb_tools.py  
**Тип изменения:** Улучшение функционала

---

## Краткое описание

Изменен инструмент `find_abbreviation_expansion` для возврата не просто списка строк расшифровок, а **для каждой расшифровки указывается чанк** (существующая Pydantic структура `ChunkResult`), в котором она была найдена.

---

## Мотивация

Раньше инструмент возвращал только список строк:
```json
{
  "expansions": ["Корпоративный Центр Обработки Информации"]
}
```

Теперь для каждой расшифровки возвращается полная информация о чанке, где она найдена:
- Содержимое чанка
- Метаданные (source, section, line_start, etc.)
- Возможность использовать для дальнейшей обработки

Это позволяет:
- Видеть контекст расшифровки
- Использовать metadata для навигации
- Применять get_neighbor_chunks для расширения контекста
- Интегрироваться с другими инструментами

---

## Изменения

### 1. Новая структура `AbbreviationExpansionItem`

```python
class AbbreviationExpansionItem(BaseModel):
    """Одна найденная расшифровка аббревиатуры с чанком"""
    expansion: str = Field(description="Текст расшифровки")
    chunk: ChunkResult = Field(description="Чанк в котором найдена расшифровка")
```

### 2. Обновлена структура `AbbreviationExpansionResult`

**Было:**
```python
class AbbreviationExpansionResult(BaseModel):
    abbreviation: str
    expansions: list[str]  # Просто список строк
    total_found: int
    pattern_used: str
```

**Стало:**
```python
class AbbreviationExpansionResult(BaseModel):
    abbreviation: str
    expansions: list[AbbreviationExpansionItem]  # Теперь с чанками!
    total_found: int
    pattern_used: str
```

### 3. Обновлена логика `find_abbreviation_expansion`

**Новый алгоритм:**

1. **Найти расшифровки через regex** (как раньше)
   - Генерируем паттерн из аббревиатуры
   - Ищем совпадения в исходных .md файлах
   - Получаем (expansion, source_file, line_number)

2. **Создать словарь expansions_map**
   ```python
   expansions_map = {}  # expansion -> [(file, line_number), ...]
   ```

3. **Для каждой уникальной расшифровки найти чанк в vectorstore**
   ```python
   docs = vectorstore.clone().exact_search(
       expansion,
       limit=5,
       chunk_type="",
       source=source_file,
       section=None
   )
   ```

4. **Выбрать наиболее подходящий чанк**
   - Ищем чанк, содержащий расшифровку
   - Выбираем ближайший по line_start к regex match
   - Если не нашли, берем первый

5. **Вернуть `AbbreviationExpansionItem` с expansion и chunk**

## Преимущества

1. **Полная контекстная информация**
   - Не просто расшифровка, а весь чанк
   - Можно увидеть в каком контексте используется
   - Метаданные для навигации

2. **Интеграция с другими инструментами**
   - Можно использовать `get_neighbor_chunks` с line_start
   - Можно использовать `get_section_content` с source + section
   - Единая структура ChunkResult как в других инструментах

3. **Релевантность**
   - Алгоритм выбирает наиболее подходящий чанк по расстоянию от regex match
   - Проверяет наличие расшифровки в содержимом чанка

4. **Обратная совместимость структур**
   - Используется стандартная структура ChunkResult
   - Агент может работать с результатами как с обычными чанками

## Технические детали

### Алгоритм поиска чанка

```python
# 1. Ищем в vectorstore по точному совпадению + фильтр по файлу
docs = vectorstore.clone().exact_search(
    expansion,
    limit=5,
    chunk_type="",
    source=source_file,
    section=None
)

# 2. Выбираем ближайший чанк по line_start
best_doc = None
min_distance = float('inf')

for doc in docs:
    if expansion.lower() in doc.page_content.lower():
        line_start = doc.metadata.get('line_start', 0)
        distance = abs(line_start - line_num)  # line_num из regex match
        if distance < min_distance:
            min_distance = distance
            best_doc = doc

# 3. Конвертируем в ChunkResult
chunk = ChunkResult(
    content=best_doc.page_content,
    metadata=ChunkMetadata(...),
    score=None
)
```

### Дедупликация

Расшифровки дедуплицируются на уровне текста:
- Одинаковые расшифровки группируются
- Для каждой уникальной берется первое местоположение
- Количество `max_results` применяется к уникальным расшифровкам

**Стало:**
```python

```

**Миграция для агента:**
```python

    print(expansion)

    print(item.expansion)
    print(f"  Файл: {item.chunk.metadata.source}")
    print(f"  Секция: {item.chunk.metadata.section}")
```
