# Пустые транскрипции как индикатор ошибки

**Дата:** 2026-02-22  
**Файл:** `transcribe/t_directory.py`

## 🎯 ИЗМЕНЕНИЕ ЛОГИКИ

Теперь файлы с **пустыми транскрипциями (0 байт)** считаются **ошибкой обработки** и не обрабатываются повторно.

---

## 💡 ПРОБЛЕМА

### Было:
```python
# Пустой файл транскрипции (0 байт) означал:
# - Файл "обработан" (has_transcription = False, т.к. размер 0)
# - Будет обрабатываться снова
# - Может зациклиться если ошибка повторяется
```

**Сценарий:**
1. Обработка файла завершилась с ошибкой
2. Создан пустой файл транскрипции (0 байт)
3. `has_transcription()` возвращает `False` (размер = 0)
4. Файл обрабатывается снова
5. Снова ошибка → снова пустой файл
6. **Зацикливание!**

---

## ✅ РЕШЕНИЕ

### Новая логика:

```
Пустой файл транскрипции (0 байт) = Ошибка обработки
```

**Реализация:**

### 1. Функция `has_failed_transcription()`

Проверяет наличие пустых транскрипций:

```python
def has_failed_transcription(file_path: Path, revision: str) -> bool:
    """
    Проверяет, есть ли пустой файл транскрипции (нулевой размер).
    Пустой файл указывает на ошибку обработки.
    
    Returns:
        True если найден файл транскрипции с нулевым размером
    """
    stem = file_path.stem
    parent = file_path.parent
    
    # Паттерн: <имя>.gigaam-<revision>-*.txt
    pattern = f"{stem}.gigaam-{revision}-*.txt"
    matching_files = list(parent.glob(pattern))
    
    for transcription_path in matching_files:
        try:
            file_size = transcription_path.stat().st_size
            if file_size == 0:
                # Найден пустой файл - это ошибка!
                logger.debug(f"Обнаружен пустой файл: {transcription_path.name}")
                return True
        except OSError:
            continue
    
    return False
```

---

### 2. Инициализация `failed_files` с пустыми транскрипциями

При запуске скрипта сканируем директорию и находим все файлы с пустыми транскрипциями:

```python
failed_files = set()

# Инициализация: ищем пустые транскрипции
logger.info("Проверка наличия пустых транскрипций...")
initial_scan = collect_media_files(directory, skip_existing=False, revision=revision)

empty_transcriptions_count = 0
for file_path in initial_scan:
    if has_failed_transcription(file_path, revision):
        # ✅ Добавляем в failed_files
        failed_files.add(file_path)
        empty_transcriptions_count += 1
        logger.debug(f"Файл с пустой транскрипцией: {file_path.name}")

if empty_transcriptions_count > 0:
    logger.warning(
        f"Обнаружено файлов с пустыми транскрипциями: {empty_transcriptions_count}\n"
        f"Эти файлы считаются ошибочными и будут пропущены"
    )
```

## 🎯 ЛОГИКА РАБОТЫ

### 1. При запуске скрипта:

```python
# Шаг 1: Сканируем все медиафайлы
media_files = collect_media_files(directory, skip_existing=False)

# Шаг 2: Проверяем каждый файл на пустые транскрипции
for file in media_files:
    if has_failed_transcription(file):
        failed_files.add(file)  # Помечаем как ошибочный
```

### 2. При обработке:

```python
# Шаг 3: Пересканируем директорию
media_files = collect_media_files(directory, skip_existing=True)

# Шаг 4: Фильтруем файлы с ошибками
media_files = [f for f in media_files if f not in failed_files]

# Шаг 5: Обрабатываем только "чистые" файлы
```

### 3. При новой ошибке:

```python
# Шаг 6: Обработка завершилась с ошибкой
if not success:
    failed_files.add(file_path)  # Добавляем в список ошибочных
```

## 🔧 ТЕХНИЧЕСКИЕ ДЕТАЛИ

### Алгоритм поиска пустых транскрипций:

1. **Glob паттерн:**
   ```python
   pattern = f"{file_stem}.gigaam-{revision}-*.txt"
   # Пример: "recording.gigaam-e2e_rnnt-*.txt"
   ```

2. **Проверка размера:**
   ```python
   for transcription_path in matching_files:
       file_size = transcription_path.stat().st_size
       if file_size == 0:
           return True  # Найден пустой файл
   ```

3. **Обработка ошибок:**
   ```python
   try:
       file_size = transcription_path.stat().st_size
   except OSError:
       continue  # Игнорируем недоступные файлы
   ```
