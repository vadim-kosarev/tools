# 20260428.008 - Добавление confidence (score) в результаты semantic_search

**Дата:** 2026-04-28  
**Автор:** AI Agent (Claude)  
**Проблема:** В результатах semantic_search отсутствовало поле confidence/score

## Описание проблемы

В логах результатов semantic_search отсутствовала информация о релевантности каждого найденного чанка:

```json
{
  "content": "На АРМ эксплуатационного персонала...",
  "metadata": {
    "source": "file.md",
    "section": "Section"
  }
  // ❌ Нет score!
}
```

Пользователь просил добавить **confidence** (оценку релевантности) к каждому результату.

## Внесенные изменения

### 1. Класс ChunkResult - добавлено опциональное поле score

**Файл:** `RAG/kb_tools.py` (строка ~64)

```python
class ChunkResult(BaseModel):
    """Результат поиска чанков"""
    content: str = Field(description="Содержимое чанка")
    metadata: ChunkMetadata = Field(description="Метаданные чанка")
    score: Optional[float] = Field(
        default=None, 
        description="Оценка релевантности (distance для semantic search, coverage для multi-term)"
    )
```

**Почему опциональное:**
- Не все инструменты возвращают score (exact_search, find_sections_by_term)
- semantic_search и multi_term_exact_search теперь могут заполнять score

### 2. Обновлена функция конвертации Document → ChunkResult

**Файл:** `RAG/kb_tools.py` (строка ~246)

```python
def _doc_to_chunk_result(doc: Document, score: Optional[float] = None) -> ChunkResult:
    """Конвертер Document → ChunkResult"""
    return ChunkResult(
        content=doc.page_content,
        metadata=_doc_to_chunk_metadata(doc.metadata),
        score=score  # ← Новый параметр
    )
```

### 3. Добавлена функция для конвертации docs с scores

**Файл:** `RAG/kb_tools.py` (строка ~258)

```python
def _docs_with_scores_to_chunk_results(docs_scores: list[tuple[Document, float]]) -> list[ChunkResult]:
    """Конвертер списка (Document, score) → список ChunkResult с scores"""
    return [_doc_to_chunk_result(doc, score) for doc, score in docs_scores]
```

### 4. Обновлен semantic_search для использования scores

**Файл:** `RAG/kb_tools.py` (строка ~688)

**Было:**
```python
docs = vectorstore.clone().similarity_search(
    query, k=top_k, chunk_type=chunk_type, source=source, section=section
)
chunks = _deduplicate_chunks(_docs_to_chunk_results(docs))
```

**Стало:**
```python
docs_scores = vectorstore.clone().similarity_search_with_score(
    query, k=top_k, chunk_type=chunk_type, source=source, section=section
)
chunks = _deduplicate_chunks(_docs_with_scores_to_chunk_results(docs_scores))
```

**Изменения:**
- ✅ Используется `similarity_search_with_score` вместо `similarity_search`
- ✅ Получаем пары (Document, float) вместо просто Document
- ✅ Конвертируем через новую функцию `_docs_with_scores_to_chunk_results`

### 5. Обновлен docstring semantic_search

Добавлено описание возвращаемого поля score:

```
Each result includes score (cosine distance) indicating relevance (lower = more similar).
```

## Результат

### Пример результата semantic_search с score:

```json
{
  "content": "На АРМ эксплуатационного персонала СОИБ КЦОИ должно быть установлено\nследующее программное обеспечение:",
  "metadata": {
    "source": "Общее описание системы-(ИИ-26).md",
    "section": "Сводная таблица устанавливаемого ПО",
    "chunk_type": "",
    "line_start": 15166,
    "line_end": 15167,
    "chunk_index": 4,
    "table_headers": null
  },
  "score": 0.31132590770721436  // ← Новое поле!
}
```

### Интерпретация score:

- **Score** = cosine distance (меньше = лучше)
- **0.0 - 0.3** - очень релевантный результат
- **0.3 - 0.5** - релевантный результат
- **0.5 - 0.7** - частично релевантный
- **> 0.7** - слабая релевантность
