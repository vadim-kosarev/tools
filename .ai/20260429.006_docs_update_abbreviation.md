## Ключевые обновления в документации

### Новая структура результата

Документация теперь показывает:

**Модели:**
```python
class AbbreviationExpansionItem(BaseModel):
    expansion: str      # Текст расшифровки
    chunk: ChunkResult  # Чанк с метаданными
```

**Пример JSON:**
```json
{
  "expansions": [
    {
      "expansion": "Корпоративный Центр...",
      "chunk": {
        "content": "...",
        "metadata": {
          "source": "file.md",
          "section": "Раздел",
          "line_start": 42
        }
      }
    }
  ]
}
```

### Обновлен Python API

**Было:**
```python
for expansion in result.expansions:
    print(expansion)
```

**Стало:**
```python
for item in result.expansions:
    print(item.expansion)
    print(f"Файл: {item.chunk.metadata.source}")
    print(f"Секция: {item.chunk.metadata.section}")
```

### Добавлен раздел Breaking Changes

Четко указано что изменился формат и как мигрировать:
- `result.expansions[0]` (строка) → `result.expansions[0].expansion` (строка)
- Доступен `result.expansions[0].chunk` (ChunkResult)

### Преимущества для агента

Документация объясняет что агент теперь может:
- Показать контекст использования термина
- Использовать `get_neighbor_chunks` для расширения
- Указать точный источник (файл, секция, строка)
