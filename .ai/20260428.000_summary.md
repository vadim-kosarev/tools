### 3. Расширенная стратегия определения терминов (009)
**Файл:** `.ai/20260428.009_extended_term_extraction.md`

**Что сделано:**
- Внедрена 5-шаговая стратегия определения терминов:
  1. Основные термины из вопроса
  2. Синонимы каждого термина
  3. Связанные слова
  4. Синонимы связанных слов
  5. Словосочетания из запроса
- Увеличено количество терминов с 3-5 до 15-20

**Измененные файлы:**
- `prompts/system_prompt_base.md` - добавлена детальная стратегия
- `prompts/action/system.md` - инструкции для action node
- `prompts/plan/system.md` - инструкции для планирования

**Пример:**
```
Вопрос: "Какое ПО установлено на АРМ оператора"

Термины (20):
- Основные: ПО, АРМ оператора, установлено
- Синонимы: программное обеспечение, программные средства, софт, АРМ, рабочее место
- Связанные: программа, приложения, компонент
- Синонимы связанных: утилита, модуль
- Словосочетания: "ПО установлено", "установлено на АРМ", "ПО на АРМ"
```

### 4. Структурированные рекомендации (next_targets) (010)
**Файл:** `.ai/20260428.010_observation_next_targets.md`

**Что сделано:**
- Добавлено поле `next_targets` в AgentObservation
- Агент теперь явно указывает какие разделы изучить дальше
- Автоматическое использование рекомендаций на следующей итерации

**Измененные файлы:**
- `rag_lg_agent.py` - добавлено поле next_targets в модель
- `prompts/observation/system.md` - инструкции по заполнению
- `prompts/refine/system.md` - приоритет на использование next_targets
- `prompts/action/system.md` - инструкции для уточняющих запросов

**Workflow:**
```
Итерация 1:
  observation → next_targets: [{source: "file.md", section: "Раздел", relevance: "high"}]
  refine → needs_refinement=True (есть конкретные разделы)
  
Итерация 2:
  action → get_section_content(source_file="file.md", section="Раздел")
  observation → next_targets: [] (данных достаточно)
  refine → needs_refinement=False
```
