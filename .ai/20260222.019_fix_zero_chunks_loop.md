# Исправление: зацикливание на файлах с 0 чанками

**Дата:** 2026-02-22  
**Файлы:** `t_gigaam_simple.py`, `t_gigaam_blocks.py` (требует исправления)

## 🐛 ПРОБЛЕМА

Файлы слишком короткие (< 5 секунд) создают **0 чанков**, но пустой файл транскрипции всё равно сохраняется → зацикливание.

### Лог ошибки:

```
2026-02-22 15:28:27,508 - INFO - Длительность аудио: 2.0 сек
2026-02-22 15:28:27,509 - INFO - Создано 0 чанков
2026-02-22 15:28:27,510 - INFO - Результат сохранён: ...gigaam-e2e_rnnt-simple.txt
2026-02-22 15:28:27,510 - INFO - Готово → ...gigaam-e2e_rnnt-simple.txt
```

### Проблема:
1. Файл 2 секунды → 0 чанков (минимум 5 сек для чанка)
2. `full_text = " ".join(texts)` → пустая строка
3. Файл транскрипции создаётся пустым (0 байт)
4. При следующем сканировании: `status = FAILED` → в `failed_files`
5. ✅ Больше не обрабатывается

**НО** если запустить напрямую `t_gigaam_simple.py файл.amr`:
- Создаётся пустой файл
- Скрипт завершается успешно (нет ошибки!)
- При запуске через `t_directory.py` → зацикливание

---

## ✅ РЕШЕНИЕ

Добавить проверку на **0 чанков** → выбросить исключение.

### В `t_gigaam_simple.py`:

```python
chunking_result = cut_audio_to_chunks(process_path, chunk_sec=chunk_sec, overlap_sec=OVERLAP_SEC)

# ✅ Проверяем, что создано хотя бы один чанк
if len(chunking_result.chunks) == 0:
    error_msg = (
        f"Файл слишком короткий ({chunking_result.total_duration_sec:.1f} сек), "
        f"не удалось создать чанки для обработки"
    )
    logger.error(error_msg)
    raise ValueError(error_msg)

texts = []
# ...остальной код...
```

## 🎯 ЛОГИКА

### Почему ValueError?

```python
if len(chunking_result.chunks) == 0:
    raise ValueError("Файл слишком короткий...")
```

**Причина:** Это **реальная ошибка** - файл не может быть обработан.

**Альтернативы:**
- ❌ Создать пустой файл → зацикливание
- ❌ Записать "Файл слишком короткий" → не настоящая транскрипция
- ✅ Выбросить исключение → корректная обработка ошибки

## 🔧 КАК РАБОТАЕТ В t_directory.py

### Обработка исключений:

```python
def process_single_file(file_path, ...):
    try:
        # Запускаем скрипт транскрибации
        result = subprocess.run([...])
        
        if result.returncode != 0:
            return False  # Ошибка
        
        # Проверяем результат
        status = get_transcription_status(file_path, revision)
        return status == TranscriptionStatus.SUCCESS
        
    except Exception as e:
        logger.error(f"Ошибка: {e}")
        return False
```

### Цикл обработки:

```python
while True:
    media_files = collect_media_files(...)
    media_files = [f for f in media_files if f not in failed_files]
    
    file_path = media_files[0]
    success = process_single_file(file_path, ...)
    
    if not success:
        failed_files.add(file_path)  # ✅ Запоминаем
```
