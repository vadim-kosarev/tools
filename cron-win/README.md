# Задача
- скрипт cron-win.ps1 для Powershell заводит scheduled tasks в windows по реестру, расположенному в файлах crontab.txt
- начальный файл - C:\crontab.txt
- формат строк в crontab.txt:
```
Path: C:\dev\tools\cron-win
Path: C:\dev\vsnifer
* * * * * powershell -File run.ps1
* * * * * powershell -File "run with spaces.ps1"
```
- Если строка содержит cron-выражение - делать задачу в Windows Task Scheduler, которая будет запускать указанный скрипт по расписанию из директории расположения файла crontab.txt
- Если строка содержит путь к директории с префиксом "Path: Путь-к-директории" - идти в эту директорию и обрабатывать crontab.txt, который лежит там
- При создании задачи:
  - задача запускается по расписанию, указанному в cron-выражении
  - каталог для задач в Windows Task Scheduler - "CronWin"
  - имя задачи - по полному пути к директории, в котором лежит crontab.txt, с заменой ":" на "_" и "\" на "_" + "_" + имя скрипта, который нужно запускать
  - задачи могут добавляться и удаляться из crontab.txt, скрипт должен синхронизировать задачи в Windows Task Scheduler с содержимым crontab.txt
  - директория выполнения задачи - директория, в которой лежит crontab.txt
  - если в описании задачи указан полный путь до исполняемого файла - в задаче в Task Sheduler использовать его как имя процесса, иначе нужно запускать cmd.exe /C и указанный скрипт как аргумент, чтобы он запускался в контексте cmd.exe, что позволит использовать переменные окружения и относительные пути
- При удалении задачи из crontab.txt - удалять соответствующую задачу из Windows Task Scheduler
- При добавлении новой задачи в crontab.txt - добавлять соответствующую задачу в Windows Task Scheduler
- При изменении расписания или команды в crontab.txt - обновлять соответствующую задачу в Windows Task Scheduler
- Если строчку не удалось обработать - ничего не делать, выводить ошибку в консоль и продолжать обработку остальных строк
- Лог-файл исполнения - C:\Temp\cron-win.log, ротация лог-файла - 7 дней, сохранять только 5 последних логов

# Использование

## Требования
- Windows 10 / Windows Server 2016+
- PowerShell 5.1+
- Запуск **от имени администратора** (требуется для управления Task Scheduler)

## Запуск
```powershell
# Запустить от имени администратора:
powershell -ExecutionPolicy Bypass -File "cron-win.ps1"
```

## Структура crontab.txt

```
# Комментарии начинаются с #

# Директива пути - обработать crontab.txt в указанной директории
Path: C:\dev\myproject
Path: "C:\path with spaces\project"

# Cron-записи: поля <минута> <час> <день> <месяц> <день_недели> <команда>
* * * * * powershell -File run.ps1
*/5 * * * * powershell -File "run with spaces.ps1"
0 3 * * * C:\scripts\backup.bat
0 9 * * 1 powershell -File weekly-report.ps1
```

## Имена задач в Task Scheduler

Задачи создаются в папке **CronWin** в Windows Task Scheduler.

Имя задачи формируется как:
`<путь_к_директории_с_crontab>_<имя_скрипта>`

где `:` и `\` в пути заменены на `_`. Например:
- Директория: `C:\dev\myproject`, скрипт: `run.ps1`
- Имя задачи: `C__dev_myproject_run.ps1`

## Лог-файл

`C:\Temp\cron-win.log` — ротация раз в 7 дней, хранится 5 последних ротированных файлов.

## Поддерживаемые cron-паттерны

| Паттерн          | Описание                        |
|------------------|---------------------------------|
| `* * * * *`      | Каждую минуту                   |
| `*/N * * * *`    | Каждые N минут                  |
| `M */N * * *`    | Каждые N часов в минуту M       |
| `M H * * *`      | Ежедневно в HH:MM               |
| `M H * * DOW`    | Еженедельно (0=Sun, 6=Sat)      |

Остальные паттерны обрабатываются как fallback: каждую минуту (с предупреждением в лог).
