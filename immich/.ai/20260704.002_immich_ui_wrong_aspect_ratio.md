# Immich v3: неправильное соотношение сторон в просмотрщике

**Дата:** 2026-07-04

## Симптом

Часть фото из внешней библиотеки (папка `O-Photos/2023/2023.05`) отображалась
в просмотрщике Immich v3 с неправильными пропорциями — растянутые по ширине
(портретные) или сплюснутые (ландшафтные). Боксы лиц тоже смещались.

## Диагностика

В Immich v3 появились новые колонки `width` и `height` в таблице `asset`.
Именно они используются для вычисления размера контейнера просмотрщика.

При миграции с v2 эти поля не были заполнены — остались `NULL`.  
Когда `width IS NULL OR height IS NULL`, Immich рендерит контейнер как квадрат (1:1),
и изображение растягивается под него.

Проверка через Playwright + DevTools:
```
naturalWidth: 2160, naturalHeight: 1440   ← правильно (landscape)
displayWidth: 1270, displayHeight: 1270   ← квадрат! (баг)
```

Через API: `GET /api/assets/{id}` возвращал `"width": null, "height": null`.

При этом `asset_exif.exifImageWidth/Height` были заполнены корректно —
но просмотрщик v3 их не использует.

## Масштаб

```sql
SELECT COUNT(*) FROM asset WHERE width IS NULL OR height IS NULL;
-- 7639 (7159 IMAGE + 480 VIDEO)
```

## Исправление

### Конкретные 5 проблемных фото (моментальный фикс)

```sql
UPDATE asset SET width = 3266, height = 4898 WHERE id = 'd23ed911-6713-4000-b67f-07ca75c5d838'; -- Lav086.jpg
UPDATE asset SET width = 4898, height = 3265 WHERE id = 'e12dc3e9-4cff-43e8-8883-4ea3ad2ffab8'; -- Lav089.jpg
UPDATE asset SET width = 4899, height = 3266 WHERE id = '489b65dc-207a-45c6-9c9b-025d8784a1d0'; -- Lav088.jpg
UPDATE asset SET width = 2899, height = 4349 WHERE id = '9eedc4a4-92b9-4946-a30e-fa762e1c32db'; -- Lav091.jpg
UPDATE asset SET width = 3265, height = 4898 WHERE id = '2234152b-5600-4f3a-a38d-af876114e9b0'; -- Lav093.jpg
```

### Все 7639 активов (bulk fix)

Через UI: **Admin → Jobs → Extract Metadata → All** (запущено 04.07.2026).  
Или через API:
```bash
curl -X PUT http://brightsky:2283/api/jobs/metadataExtraction \
  -H 'Content-Type: application/json' \
  -d '{"command":"start","force":true}'
```

Immich перечитывает оригинальные файлы, заполняет `asset_exif` и `asset.width/height`.
Очередь: ~87k файлов, ожидаемое время — несколько часов.
