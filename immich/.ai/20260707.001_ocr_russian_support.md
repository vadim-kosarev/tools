# Immich OCR: ошибка "Unknown model combination" и поддержка русского текста

**Дата:** 2026-07-07
**immich-server:** release (v3.0.1)
**immich-machine-learning:** был `pr-18272-cuda`, стал `${IMMICH_VERSION:-release}-cuda`

## Проблема

В логах `immich_machine_learning` при обработке OCR-джобы сыпались ошибки:

```
ValueError: Unknown model combination: None, detection, ocr
```

В логах `immich_server`:

```
Unable to run job handler (Ocr): Error: Machine learning request
'{"ocr":{"detection":{"modelName":"PP-OCRv5_mobile", ...}}}' failed for all URLs
```

## Диагностика

`immich-server` работал на теге `release` (реально v3.0.1) — в этой версии OCR уже полноценная функция.
`immich-machine-learning` при этом был закреплён в `docker-compose.prod.yml` на теге
`ghcr.io/immich-app/immich-machine-learning:pr-18272-cuda` — это preview-сборка старого PR,
собранная до того, как в `machine-learning` реализовали сами классы OCR
(`models/__init__.py` внутри контейнера не содержал `case` для `ModelTask.OCR`,
а `ModelSource` не знал про `PADDLE` вовсе — только `INSIGHTFACE/MCLIP/OPENCLIP`).

Сервер посылал task `ocr`, ML-сервис такой комбинации не знал → падение на каждом ассете.

## Решение

1. В `immich/docker/docker-compose.prod.yml` заменить фиксированный тег ML-образа на версию,
   синхронизированную с сервером:

   ```yaml
   image: ghcr.io/immich-app/immich-machine-learning:${IMMICH_VERSION:-release}-cuda
   ```

2. `docker compose pull immich-machine-learning && docker compose up -d immich-machine-learning`.
   После пересоздания в образе есть `ModelSource.PADDLE` и реализация детектора/распознавателя OCR.

3. Для русского языка в Admin UI (Settings → Machine Learning → OCR → Model) выбрать вариант
   **"PP-OCRv5_mobile (Russian, Belarussian, Ukranian and English)"** — это сохраняется в
   `system-config` как `modelName: "ESLAV__PP-OCRv5_mobile"`.

   Механизм: `TextRecognizer` в `immich_ml/models/ocr/recognition.py` парсит язык из имени модели
   (`model_name.split("__")[0]` → `LangRec[...]`, здесь `LangRec.ESLAV` — восточнославянская группа).
   Без `__`-префикса используется `LangRec.CH` (китайский) по умолчанию — отсюда и было бы
   плохое распознавание кириллицы, даже если бы старый образ поддерживал OCR.

## Проверка

После пересоздания контейнера модели скачались и загрузились без ошибок:

```
Downloading detection model 'PP-OCRv5_mobile' → ch_PP-OCRv5_det_mobile.onnx (4.60MB)
Downloading recognition model 'ESLAV__PP-OCRv5_mobile' → eslav_PP-OCRv5_rec_mobile.onnx (7.55MB)
Setting execution providers to ['CUDAExecutionProvider', 'CPUExecutionProvider']
```

Запущена полная OCR-джоба (Admin → Jobs → OCR → Run all) на всю библиотеку (~89k ассетов),
обрабатывается в фоне (`concurrency: 1`).

## На заметку

На хосте одна GPU (RTX 3060 Ti, 8 ГБ) делится между `immich-machine-learning`, `immich-server`
(nvenc), `frigate` и `rgzz.process_anpr` — VRAM почти всегда занята под завязку. Если начнутся
OOM при OCR/CLIP инференсе, вероятная причина — конкуренция за VRAM с `frigate`.
