## ️ Среда разработки

### Основные параметры
- **IDE:** JetBrains IDEA
- **Shell:** PowerShell (Windows) - команды для run_in_terminal должны быть PowerShell-совместимыми. ОСЬ - Windows.

### PowerShell команды (важно!)
При использовании `run_in_terminal` tool:
- ✅ Разделитель команд: `;` (НЕ `&&`)
- ✅ Чтение файлов: `type` (НЕ `cat`)
- ✅ Использовать только PowerShell-совместимые команды
- ✅ Команды 'head' и 'tail' НЕ использовать, вместо них использовать:
  - `Get-Content -TotalCount n` для чтения первых n строк файла
  - `Get-Content -Tail n` для чтения последних n строк
- ✅ **ОБЯЗАТЕЛЬНО** добавлять `[Console]::Out.Flush()` в конце команды для гарантированного вывода результата

**Пример:**
```powershell
cd path; python script.py    # ✅ Правильно
cd path && python script.py  # ❌ Ошибка в PowerShell

# Для гарантированного вывода:
git commit -m "message" ; git log --oneline -1 ; [Console]::Out.Flush()  # ✅ Правильно
```

##  Работа с файлами

### Создание файлов
- ❌ **НЕ создавать** тестовые скрипты/данные без явного запроса
- ❌ **НЕ создавать** `*.md` файлы
- ❌ **НЕ дублировать** существующие файлы проекта
- ✅ При создании файлов учитывать структуру проекта в IDEA

### Тестовые скрипты
- ❌ **НЕ генерировать** тестовые скрипты без явного запроса
- ✅ Генерировать тестовые скрипты ТОЛЬКО если пользователь явно запросил

### Отчеты о работе
- ❌ **НЕ генерировать** отчеты о работе без явного запроса
- ✅ Генерировать отчеты ТОЛЬКО если пользователь явно попросил

#### Если отчет запрошен явно:
1. **Папка:** `.ai/`
2. **Формат имени:** `YYYYMMDD.NN_<название>.md`
   - `YYYYMMDD` - текущая дата
   - `NNN` - порядковый номер (трехзначный, с ведущими нулями)
3. **Порядковый номер:**
   - Читается из файла `.ai/version.txt`
   - Инкрементируется при создании нового отчета
   - Сохраняется обратно в `.ai/version.txt`

**Пример:**
```
.ai/20260215.01_api_integration_report.md
.ai/20260215.02_database_migration_report.md
.ai/version.txt  # содержит: 02
```
