## Подключение правил из субмодуля claude

**Если субмодуля нет:**
```powershell
git submodule add https://github.com/vadim-kosarev/claude.git claude
git submodule update --init --recursive
```

**Используемые правила:**
- `claude/CLAUDE.base.md` - базовые правила (язык, среда разработки, файлы)
- `claude/CLAUDE.python.md` - Python-специфичные правила

## Для создания .md-файлов

Важно - при создании очередного .md-файла нужно иметь нумерацию в имени, чтобы сохранять порядок. Префикс - 001., 002., et... 
Создаваемые файлы .md в папке длжны быть с этим префиксом, чтобы сохранять порядок и структуру документации. Например:
- `001_описание_проекта.md`
- `002_инструкции_для_агента.md`
- `003_тестирование.md`
Важно - все создаваемые агентом (не человеком) файлы .md, сгенеренные во время работы должны быть в папке docs чтобы не засорять корневой каталог и сохранять структуру.
Все файлы .md, которые используются в коде, должны быть там, где их создал пользователь - не нужно их насильно переименовывать и пермемещать вместо с остальными. они требудт специального ухода и внимания. Они часть функционала.
Новых файлов .md не должно создаваться много - если нет нужды в новом файле, то можно дополнять существующий. Но если нужно создать новый, то обязательно с нумерацией и в папке docs.  Проверка - на один запрос пользователя новых файлов не больше одного. А то и ноль. В приоритете - создание отчета по большой работе (большой прирост или изменение функционала) в папке .ai
