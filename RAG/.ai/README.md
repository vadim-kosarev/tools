## 🎯 СУТЬ ЗАДАЧ (кратко)

### Задача 1: Проблема чанкинга (v3a-v3c, 2026-04-26)

**Что было:**
```
Вопрос: "Какое ПО на АРМ СОИБ?"
→ Exact search нашел 50 чанков
→ Но LLM ответил: "информация не найдена" ❌
```

**Почему:**
- Exact search нашел заголовок: "установлены следующие программные средства"
- Но сам список находился в других чанках
- `get_neighbor_chunks()` НЕ включал якорный чанк
- LLM видел заголовок без списка

**Что сделали:**
- ✅ Исправили `get_neighbor_chunks` - теперь включает якорь
- ✅ Обновили system prompts для ОБОИХ агентов
- ✅ Добавили автоисправление параметров
- ✅ Исправили AttributeError в get_chunks_by_index
- ✅ Создали тесты и документацию

**Результат:**
```
Вопрос: "Какое ПО на АРМ СОИБ?"
→ Ответ: "Консоль Kaspersky, Агент Kaspersky, Endpoint Security" ✅
```

### Задача 2: Обязательное указание разделов (v6, 2026-04-27)

**Что требовалось:**
```
В ответе агента должна быть информация:
- разделы, в которых точно есть ответ или части ответа
- разделы, в которых также может быть ответ или части ответа
```

**Что сделали:**
- ✅ Обновили `system_prompt.md` (LangGraph агент) - добавлены поля в final_answer
- ✅ Обновили `rag_lc_agent.py` (ReAct агент) - добавлены блоки в ответ
- ✅ Добавлена секция "🎯 Обязательное отслеживание разделов"
- ✅ Примеры для обоих форматов (JSON и текст)

**Результат:**
```json
{
  "final_answer": {
    "sections_with_answer": [
      {"source": "file.md", "section": "...", "confidence": "high", "reason": "..."}
    ],
    "sections_maybe_relevant": [
      {"source": "file2.md", "section": "...", "confidence": "medium", "reason": "..."}
    ]
  }
}
```

**Или в текстовом формате:**
```
**📍 РАЗДЕЛЫ С НАЙДЕННОЙ ИНФОРМАЦИЕЙ:**
- [file.md] — Раздел > Подраздел
  Причина: содержит определение / список

**🔍 РАЗДЕЛЫ ДЛЯ ДОПОЛНИТЕЛЬНОГО ИЗУЧЕНИЯ:**
- [file2.md] — Другой раздел
  Причина: может содержать дополнительную информацию
```
