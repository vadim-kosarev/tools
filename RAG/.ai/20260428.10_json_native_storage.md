# Хранение JSON данных в нативном формате (dict/list)

**Дата:** 2026-04-28  
**Версия:** 10  
**Файлы:** `rag_lg_agent.py`, `llm_call_logger.py`, `pydantic_utils.py`

---

## 🎯 Проблема

До изменений:
- ❌ Pydantic объекты → Markdown текст → JSON строка → обрезанный текст
- ❌ Множественные преобразования теряли структуру данных
- ❌ Обрезка по символам: `[:2000]`, `[:1500]`, `[:60]`
- ❌ LLM получал неполную информацию в искаженном формате

---

## ✅ Решение

**Стратегия:** Хранить JSON как объекты (dict/list) до самого конца, сериализовать только для LLM запросов.

---

## 📝 Внесенные изменения

### 1. **action_node: Сохранение результатов как dict**

**Было:**
```python
# Конвертация в строку
if hasattr(result_raw, "model_dump_json"):
    result_str = pydantic_to_markdown(result_raw)  # → TEXT
else:
    result_str = str(result_raw)  # → TEXT

tool_results.append({
    "tool": tool_name,
    "input": tool_input,
    "result": result_str  # STRING
})
```

**Стало:**
```python
# Сохраняем как dict, НЕ как строку
if hasattr(result_raw, "model_dump"):
    result_dict = result_raw.model_dump()  # → dict
elif isinstance(result_raw, (dict, list)):
    result_dict = result_raw  # → dict/list как есть
else:
    result_dict = {"value": str(result_raw)}  # → wrap в dict

tool_results.append({
    "tool": tool_name,
    "input": tool_input,
    "result": result_dict  # ← DICT/LIST, не строка!
})
```

**Эффект:**
- ✅ Структура данных сохранена (поля, типы, вложенность)
- ✅ Нет потери информации при преобразовании
- ✅ JSON остается JSON, не текстом

### 2. **messages: Хранение tool results как dict**

**Было:**
```python
state["messages"].append({
    "role": "tool",
    "name": tr["tool"],
    "tool_call_id": f"call_{iteration}_{idx}",
    "content": json.dumps(tr, ensure_ascii=False, indent=2)  # → JSON STRING

**Стало:**
```python
state["messages"].append({
    "role": "tool",
    "name": tr["tool"],
    "tool_call_id": f"call_{iteration}_{idx}",
    "content": tr  # ← Храним dict как есть, НЕ json.dumps!

```

**Эффект:**
- ✅ Нет двойной сериализации
- ✅ Доступ к полям напрямую
- ✅ LangChain корректно обрабатывает dict в content

### 3. **observation_node: JSON вместо текста**

**Было:**
```python
tools_summary = "\n\n".join(
    f"### {tr['tool']}\n"
    f"Параметры: {json.dumps(tr['input'], ensure_ascii=False)}\n"
    f"Результат:\n```\n{str(tr['result'])[:2000]}\n```"  # ← ОБРЕЗКА!
    for tr in current_results
)

user_message = f"""Вопрос: {state['user_query']}

Результаты:
{tools_summary}"""
```

**Стало:**
```python
# Формируем JSON с результатами (БЕЗ ОБРЕЗКИ!)
tools_data = []
for tr in current_results:
    tool_entry = {
        "tool": tr['tool'],
        "input": tr['input'],
        "result": tr['result']  # Уже dict
    }
    tools_data.append(tool_entry)

tools_json = json.dumps(tools_data, ensure_ascii=False, indent=2)

user_message = f"""Вопрос: {state['user_query']}

Результаты в формате JSON:
```json
{tools_json}
```

ВАЖНО: Данные в структурированном формате, НЕ обрезаны."""
```

**Эффект:**
- ✅ Убрана обрезка `[:2000]`
- ✅ LLM видит полную структуру JSON
- ✅ Компактнее (нет Markdown разметки)

### 4. **final_node: JSON вместо текста**

**Было:**
```python
all_results_summary = "\n\n".join(
    f"### {tr['tool']}\n{str(tr['result'])[:1500]}"  # ← ОБРЕЗКА!
    for tr in state.get("all_tool_results", [])

ВСЕ результаты ({total_tools}):
{all_results_summary}"""
```

**Стало:**
```python

all_results_data = []
for tr in state.get("all_tool_results", []):
    result_entry = {
        "tool": tr['tool'],
        "input": tr['input'],
        "result": tr['result']  # Уже dict
    }
    all_results_data.append(result_entry)

all_results_json = json.dumps(all_results_data, ensure_ascii=False, indent=2)

user_message = f"""...
ВСЕ результаты ({total_tools}) в формате JSON:
```json
{all_results_json}
```

ВАЖНО: Результаты НЕ обрезаны - полная информация."""
```

**Эффект:**
- ✅ Убрана обрезка `[:1500]`
- ✅ Финальный ответ основан на полных данных

### 5. **llm_call_logger: Обработка dict в content**

**Было:**
```python
if isinstance(msg.content, str):
    content = msg.content
else:
    content = json.dumps(msg.content, ensure_ascii=False, default=str)
```

**Стало:**
```python
if isinstance(msg.content, str):
    content = msg.content
elif isinstance(msg.content, (dict, list)):
    # Dict/list → JSON с отступами для читаемости
    content = json.dumps(msg.content, ensure_ascii=False, indent=2, default=str)
else:
    content = json.dumps(msg.content, ensure_ascii=False, default=str)
```

**Эффект:**
- ✅ Логи содержат красиво отформатированный JSON
- ✅ Легко читать структуру данных

**Эффект:**

## ✅ Преимущества

1. **Структура данных сохранена**
   - LLM видит поля, типы, вложенность
   - Можно обращаться к полям по именам

2. **Нет потери информации**
   - Убраны все обрезки: `[:2000]`, `[:1500]`, `[:60]`
   - Весь контент доступен

3. **Компактность**
   - JSON без Markdown разметки → меньше токенов
   - Нет дублирования (заголовки, отступы)

4. **Читаемость логов**
   - JSON с indent=2 для людей
   - Валидный JSON для автоматической обработки

5. **Единый подход**
   - Один формат на всех этапах
   - Меньше преобразований = меньше ошибок

---

## 🔍 Места хранения данных

| Место | Тип | Формат |
|-------|-----|--------|
| `tool_results` | list[dict] | `{"tool": str, "input": dict, "result": dict}` |
| `state["all_tool_results"]` | list[dict] | То же |
| `state["messages"]` | list[dict] | `{"role": "tool", "content": dict}` |
| Логи | str | JSON с indent=2 |
| LLM запрос | str | JSON с indent=2 внутри Markdown блока |

## 🚀 Результат

**ДО:**
```
Pydantic → Markdown → json.dumps() → [:2000] → TEXT
         └─ Потеря структуры
                   └─ Двойная сериализация
                             └─ Обрезка данных
```

**ПОСЛЕ:**
```
Pydantic → dict → dict → dict → JSON (для LLM)
         └─ Структура сохранена
               └─ Нет двойной сериализации
                     └─ Полные данные
```
