# JSON Formatting in Logs - Обрамление JSON тегами

**Дата:** 2026-04-28  
**Автор:** AI Agent  
**Статус:** ✅ Завершено

## Проблема

В логах запросов к LLM (файл `logs/017_llm_final_request.log`) JSON-данные в истории сообщений не были обрамлены тегами для синтаксической подсветки. Это затрудняло чтение и анализ логов, особенно для:

- TOOL_RESULT сообщений с результатами выполнения инструментов
- ASSISTANT сообщений с tool_calls
- Любых dict/list данных в content

### Пример проблемы

```
[#5 TOOL_RESULT: exact_search]
{'tool': 'exact_search', 'input': {...}, 'result': {...}}
```

Вместо:

```
[#5 TOOL_RESULT: exact_search]
```json
{
  "tool": "exact_search",
  "input": {...},
  "result": {...}
}
```
```

## Решение

Внесены изменения в `llm_call_logger.py` для автоматического обрамления JSON-контента тегами ` ```json ... ``` ` во всех местах форматирования сообщений.

### Изменённые функции

#### 1. `_fmt_message_list()` - строки 399-409

**Что изменено:**
- При форматировании `msg.content` типа dict/list добавлено обрамление ` ```json ... ``` `

**Было:**
```python
elif isinstance(msg.content, (dict, list)):
    content = json.dumps(msg.content, ensure_ascii=False, indent=2, default=str)
```

**Стало:**
```python
elif isinstance(msg.content, (dict, list)):
    json_content = json.dumps(msg.content, ensure_ascii=False, indent=2, default=str)
    content = f"```json\n{json_content}\n```"
```

#### 2. `_fmt_message_list()` - TOOL_CALLS в ASSISTANT сообщениях (строки 435-443)

**Что изменено:**
- Изменён формат tool_calls с компактного на отформатированный с отступами
- Добавлено обрамление ` ```json ... ``` `

**Было:**
```python
tc_json = json.dumps(tool_calls, ensure_ascii=False, default=str, separators=(',', ':'))
parts.append(f"\n[#{msg_num} ASSISTANT]\n{content}\n\n[TOOL_CALLS]\n{tc_json}")
```

**Стало:**
```python
tc_json = json.dumps(tool_calls, ensure_ascii=False, default=str, indent=2)
parts.append(f"\n[#{msg_num} ASSISTANT]\n{content}\n\n[TOOL_CALLS]\n```json\n{tc_json}\n```")
```

#### 3. `_fmt_llm_result()` - строки 454-473

**Что изменено:**
- Аналогичное обрамление для tool_calls в ответах LLM

**Было:**
```python
tc_json = json.dumps(tool_calls, ensure_ascii=False, default=str, separators=(',', ':'))
parts.append(f"{content}\n\n[TOOL_CALLS]\n{tc_json}")
```

**Стало:**
```python
tc_json = json.dumps(tool_calls, ensure_ascii=False, default=str, indent=2)
parts.append(f"{content}\n\n[TOOL_CALLS]\n```json\n{tc_json}\n```")
```

#### 4. `on_tool_start()` - строки 589-607

**Что изменено:**
- Обрамление JSON-параметров инструмента при логировании запроса

**Было:**
```python
parsed = json.loads(input_str)
formatted_input = json.dumps(parsed, ensure_ascii=False, indent=2)
```

**Стало:**
```python
parsed = json.loads(input_str)
json_content = json.dumps(parsed, ensure_ascii=False, indent=2)
formatted_input = f"```json\n{json_content}\n```"
```

#### 5. `on_tool_end()` - строки 609-632

**Что изменено:**
- Обрамление JSON-результата выполнения инструмента

**Было:**
```python
response_text = f"[args]\n{input_str}\n{'-' * 40}\n{str(output)}"
```

**Стало:**
```python
output_str = str(output)
if isinstance(output, (dict, list)):
    try:
        json_content = json.dumps(output, ensure_ascii=False, indent=2)
        output_str = f"```json\n{json_content}\n```"
    except (TypeError, ValueError):
        pass
response_text = f"[args]\n{input_str}\n{'-' * 40}\n{output_str}"
```

## Результат

✅ **Все JSON-блоки в логах теперь обрамлены тегами ` ```json ... ``` `**

Это обеспечивает:
- 🎨 Синтаксическую подсветку при просмотре в Markdown-редакторах
- 📖 Лучшую читаемость структурированных данных
- 🔍 Упрощённый поиск и анализ JSON-блоков в логах
- ✨ Консистентный формат во всех типах сообщений

## ДОПОЛНЕНИЕ: Удаление текстовых префиксов из tool responses

### Проблема №2

После первого этапа выяснилось, что в `kb_tools.py` все tool responses содержали дополнительные текстовые префиксы перед JSON:

```
Найдено 10 чанков

{
  "query": "smart monitor",
  ...
}
```

Вместо чистого JSON:

```json
{
  "query": "smart monitor",
  ...
}
```

### Решение №2

Удалены все текстовые префиксы из `rec.set_response()` во всех инструментах:

**Изменённые инструменты (10 шт):**
1. `semantic_search` - убран `"Найдено {len(chunks)} чанков\n\n"`
2. `exact_search` - убран `"Найдено {len(chunks)} чанков\n\n"`
3. `exact_search_in_file` - убран `"Найдено {len(chunks)} чанков в {source_file}\n\n"`
4. `exact_search_in_file_section` - убран `"Найдено {len(chunks)} чанков в [{source_file}] -> {section}\n\n"`
5. `multi_term_exact_search` - убран `"Найдено {result.total_chunks} чанков\n\n"`
6. `find_sections_by_term` - убран `"Найдено {result.total_found} разделов\n\n"`
7. `find_relevant_sections` - убран `"Найдено {result.total_found}, возвращено топ-{result.returned_count}\n\n"`
8. `read_table` - убран `"Найдено {result.total_rows} строк\n\n"`
9. `regex_search` - убран `"total_matches={result.total_matches}\n\n"`
10. `get_neighbor_chunks` - убран `"Найдено {total_chunks} чанков (якорь: ...)\n\n"`

**Результат:**
Теперь все они возвращают только `result.model_dump_json(indent=2)` - чистый JSON без префиксов.

## Итоговый результат

✅ **Два уровня форматирования JSON:**

1. **Логирование в llm_call_logger.py** - все JSON-блоки обрамлены тегами ` ```json ... ``` ` для:
   - dict/list content в messages
   - TOOL_CALLS в ASSISTANT
   - tool_calls в ответах LLM
   - Tool input
   - Tool output

2. **Tool responses в kb_tools.py** - убраны все текстовые префиксы, остался только чистый JSON для всех 10 инструментов

🎯 **Эффект:**
- Чистый JSON в response от tools (когда LLM его получает напрямую)
- Красиво форматированный JSON с тегами в логах (когда мы просматриваем логи)
