## 🐛 Проблема 1: ValidationError для find_sections_by_term

### Ошибка:
```
1 validation error for FindSectionsByTermInput
substring
  Field required [type=missing, input_value={'term': 'СОИБ КЦОИ'}, input_type=dict]
```

### Причина:
LLM передает параметр `term` вместо требуемого `substring` для инструмента `find_sections_by_term`.

### Решение:

#### 1. Добавлена функция автоисправления в `rag_lg_agent.py`:
```python
def _fix_tool_args(tool_name: str, tool_input: dict) -> dict:
    """
    Автоматическое исправление некорректных параметров от LLM.
    """
    fixed = tool_input.copy()
    
    # find_sections_by_term: 'term' → 'substring'
    if tool_name == "find_sections_by_term":
        if "term" in fixed and "substring" not in fixed:
            fixed["substring"] = fixed.pop("term")
        if "query" in fixed and "substring" not in fixed:
            fixed["substring"] = fixed.pop("query")
    
    # get_section_content: 'source' → 'source_file'
    if tool_name == "get_section_content":
        if "source" in fixed and "source_file" not in fixed:
            fixed["source_file"] = fixed.pop("source")
    
    # exact_search_in_file: 'source' → 'source_file'
    if tool_name in ["exact_search_in_file", "exact_search_in_file_section"]:
        if "source" in fixed and "source_file" not in fixed:
            fixed["source_file"] = fixed.pop("source")
    
    return fixed
```

#### 2. Интеграция в `action_node`:
```python
for tc in state["tool_calls"]:
    tool_name = tc["tool"]
    tool_input = tc["input"]
    
    # Автоисправление некорректных параметров от LLM
    tool_input = _fix_tool_args(tool_name, tool_input)  # ← ДОБАВЛЕНО
    
    if tool_name in tools_map:
        ...
```

#### 3. Обновлен `system_prompt.md`:
- Добавлена таблица с ключевыми параметрами каждого инструмента
- Добавлены примеры правильных и неправильных параметров
- Выделены часто ошибочные параметры: `substring` (НЕ term!), `source_file` (НЕ source!)

## 🐛 Проблема 2: AttributeError в get_chunks_by_index

### Ошибка:
```
AttributeError: 'ClickHouseVectorStore' object has no attribute 'table'
```

### Причина:
В функции `get_chunks_by_index` (kb_tools.py:1517) использовалось:
- `vs_clone.table` вместо `vs_clone._cfg.table`
- `vs_clone.client` вместо `vs_clone._client`

### Решение:

#### Исправлен SQL-запрос:
```python
# Было:
FROM {vs_clone.table}
result_rows = vs_clone.client.query(query, params)

# Стало:
db, tbl = vs_clone._cfg.database, vs_clone._cfg.table
FROM {db}.{tbl} FINAL
result = vs_clone._client.query(query, params)
```

#### Исправлена обработка результатов:
```python
# Было:
for row in result_rows.result_rows:
    content = row[0]
    meta = row[1] if isinstance(row[1], dict) else {}
    ...

# Стало:
for row in result.result_rows:
    content, src, sec, ct, th, ls, le, ci = row
    meta = {
        'source': src,
        'section': sec,
        'chunk_type': ct,
        'line_start': int(ls),
        'line_end': int(le),
        'chunk_index': int(ci),
    }
    if th:
        meta['table_headers'] = th
```
