**Дата:** 2026-04-27

## 🎯 ЗАДАЧА

Добавить в системный промпт обоих агентов (LangGraph и ReAct) требование обязательно указывать в ответе:
- **Разделы, в которых точно есть ответ или части ответа**
- **Разделы, в которых также может быть ответ или части ответа**

---

## 📋 КОНТЕКСТ

### Зачем нужно

После решения проблемы чанкинга (v3b) стало очевидно, что пользователь должен иметь:
1. Прозрачность источников информации
2. Возможность проверить и углубить изучение темы
3. Понимание, насколько полон ответ
4. Навигацию по документации для самостоятельного поиска

### Требования пользователя

> "надо добавить в системный промпт, что нужно в ответе обязательно иметь информацию
> разделы, в которых точно есть ответ или части ответа
> разделы, в которых также может быть ответ или части ответа"

## ✅ РЕАЛИЗАЦИЯ

### 1. Обновление `system_prompt.md` (LangGraph агент)

**Файл:** `RAG/system_prompt.md`

#### 1.1. Обновлена структура `final_answer`

**Было:**
```json
{
  "final_answer": {
    "summary": "краткий ответ",
    "details": "подробное объяснение",
    "data": [...],
    "sources": ["doc1", "doc2"],
    "confidence": 0.87
  }
}
```

**Стало:**
```json
{
  "final_answer": {
    "summary": "краткий ответ",
    "details": "подробное объяснение",
    "data": [...],
    "sources": ["doc1", "doc2"],
    "sections_with_answer": [
      {
        "source": "file.md",
        "section": "Раздел > Подраздел",
        "confidence": "high",
        "reason": "содержит прямой ответ на вопрос"
      }
    ],
    "sections_maybe_relevant": [
      {
        "source": "file2.md",
        "section": "Другой раздел",
        "confidence": "medium",
        "reason": "может содержать дополнительную информацию"
      }
    ],
    "confidence": 0.87
  }
}
```

**Добавлено:**
- `sections_with_answer` - массив разделов с ТОЧНОЙ информацией
- `sections_maybe_relevant` - массив разделов с ВОЗМОЖНОЙ информацией

Каждый раздел содержит:
- `source` - имя файла .md
- `section` - полный breadcrumb путь (H1 > H2 > H3)
- `confidence` - уровень уверенности ("high" / "medium" / "low")
- `reason` - объяснение релевантности раздела

#### 1.2. Добавлена секция "🎯 Обязательное отслеживание разделов"

**Где:** После раздела "Логика принятия решений"

**Содержит:**
- Правила для работы с инструментами - записывать все источники
- Правила для level of confidence (high/medium/low)
- Правила для формирования final_answer
- Примеры причин (reason) релевантности:
  - "содержит определение термина"
  - "перечисляет список компонентов"
  - "описывает конфигурацию системы"
  - "содержит таблицу с IP-адресами"
  - "может содержать дополнительные детали"
  - "упоминает связанные системы"

#### 1.3. Обновлен пример выполнения (Шаг 3)

**Добавлено в пример ответа:**
```json
{
  "status": "final",
  "step": 3,
  "final_answer": {

    "details": "...",
    "data": [...],
    "sources": ["Приложение_А.md", "Руководство_администратора.md"],
    "sections_with_answer": [
      {
        "source": "Приложение_А.md",
        "section": "Архитектура системы > Компоненты инфраструктуры > ППРК",
        "confidence": "high",
        "reason": "содержит определение и описание назначения ППРК"
      },
      {
        "source": "Руководство_администратора.md",
        "section": "Первичное развертывание > Настройка ППРК",
        "confidence": "high",
        "reason": "описывает процесс использования ППРК при развертывании"
      }
    ],
    "sections_maybe_relevant": [
      {
        "source": "Руководство_администратора.md",
        "section": "Архитектура > Распределенная конфигурация",
        "confidence": "medium",
        "reason": "может содержать дополнительную информацию о принципах распределения конфигурации"
      }
    ],
    "confidence": 0.87
  }
}
```

**Статистика изменений файла:**
- Добавлено строк: ~30
- Изменено разделов: 3
- Обновлено примеров: 1

### 2. Обновление `rag_lc_agent.py` (ReAct агент)

**Файл:** `RAG/rag_lc_agent.py`

#### 2.1. Добавлена секция в ПРАВИЛА ОТВЕТА

**Где:** После существующих правил ответа, перед закрывающими кавычками

**Содержимое:**
```python
━━━ ОБЯЗАТЕЛЬНОЕ ТРЕБОВАНИЕ: УКАЗАНИЕ РАЗДЕЛОВ С ИНФОРМАЦИЕЙ ━━━━━━━━━━━━

⚠️ В КОНЦЕ ОТВЕТА ОБЯЗАТЕЛЬНО добавь два блока:

**📍 РАЗДЕЛЫ С НАЙДЕННОЙ ИНФОРМАЦИЕЙ:**
Перечисли разделы, из которых взята информация для ответа:
- [file.md] — Раздел > Подраздел
  Причина: содержит определение / список / таблицу / конфигурацию
  
**🔍 РАЗДЕЛЫ ДЛЯ ДОПОЛНИТЕЛЬНОГО ИЗУЧЕНИЯ:**
Перечисли разделы, которые МОГУТ содержать дополнительную информацию:
- [file2.md] — Другой раздел
  Причина: упоминает связанные системы / может содержать детали

ВАЖНО:
- Отслеживай все разделы (source + section) из которых получена информация
- Различай уровень confidence:
  • ТОЧНО есть ответ - добавляй в первый блок
  • МОЖЕТ быть информация - добавляй во второй блок
- Для каждого раздела указывай причину релевантности

ПРИМЕР:
**📍 РАЗДЕЛЫ С НАЙДЕННОЙ ИНФОРМАЦИЕЙ:**
- [Приложение И.md] — Состав программных средств КОИБ z/OS
  Причина: содержит полный список ПО с версиями
- [Техническое описание.md] — АРМ администратора > Конфигурация
  Причина: описывает конфигурацию и список установленного ПО
  
**🔍 РАЗДЕЛЫ ДЛЯ ДОПОЛНИТЕЛЬНОГО ИЗУЧЕНИЯ:**
- [Руководство администратора.md] — Установка и настройка ПО
  Причина: может содержать дополнительные детали по установке
"""
```

**Ключевые моменты:**
1. Использует текстовый формат (не JSON) - естественно для ReAct агента
2. Emoji-маркеры для визуального разделения (📍 и 🔍)
3. Формат списка с отступами для причины
4. Примеры реальных разделов из документации

### Ручное тестирование

**Тест 1: LangGraph агент**
```bash
python rag_lg_agent.py "какое ПО установлено на АРМ СОИБ?"
```

**Ожидаемый результат:**
- final_answer содержит sections_with_answer

- Для каждого раздела указан source, section, confidence, reason

**Тест 2: ReAct агент**
```bash
python rag_lc_agent.py "какое ПО установлено на АРМ СОИБ?"
```

**Ожидаемый результат:**
- Ответ содержит блок "📍 РАЗДЕЛЫ С НАЙДЕННОЙ ИНФОРМАЦИЕЙ"
- Ответ содержит блок "🔍 РАЗДЕЛЫ ДЛЯ ДОПОЛНИТЕЛЬНОГО ИЗУЧЕНИЯ"
- Для каждого раздела указана причина релевантности

### Автоматическое тестирование

На данном этапе автоматические тесты не требуются, так как:
- Изменения затрагивают только промпты
- Валидация осуществляется самим LLM
- Структура ответа зависит от содержимого запроса

Рекомендуется провести ручное тестирование на разных типах вопросов:
- Определения ("что такое КЦОИ")
- Списки ("все СУБД")
- Конфигурации ("IP адреса серверов")

## 💡 РЕКОМЕНДАЦИИ

### Для пользователей

1. **Проверяйте разделы "с точной информацией"**
   - Это основные источники для ответа
   - Можно открыть и проверить полноту данных

2. **Изучайте разделы "для дополнительного изучения"**
   - Могут содержать важные детали
   - Расширяют контекст ответа

3. **Обращайте внимание на confidence**
   - high - уверенный источник
   - medium - возможный источник
   - low - сомнительный источник

### Для разработчиков

1. **При доработке агентов**
   - Всегда заполняйте оба поля (sections_with_answer и sections_maybe_relevant)
   - Используйте понятные reason для каждого раздела
   - Отличайте уровни confidence

2. **При добавлении новых инструментов**
   - Убедитесь, что инструменты возвращают source и section
   - Добавьте метаданные для оценки confidence
   - Документируйте формат возврата

3. **При обновлении промптов**
   - Сохраняйте требования об указании разделов
   - Добавляйте примеры для новых сценариев
   - Тестируйте на реальных запросах
