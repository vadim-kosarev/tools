# MCP HTTP сервер на официальном SDK (Streamable HTTP) + запуск через uv

Дата: 2026-06-06

## Проблема

Старый `kb_tools_mcp_http.py` вручную реализовывал MCP поверх **SSE** через FastAPI
(`GET /sse` + `POST /messages` с ручными очередями). Транспорт SSE устарел, реализация
была хрупкой. Скрипт запуска не работал: искал `.venv` только в папке `RAG`, а окружение
лежит на уровень выше; в `.venv` отсутствовали нужные пакеты.

## Что сделано

### 1. `kb_tools_mcp_http.py` — переписан на официальный MCP SDK (`mcp`)
- Транспорт **Streamable HTTP** (актуальный) через `StreamableHTTPSessionManager`.
- Low-level `mcp.server.lowlevel.Server` с обработчиками `list_tools` / `call_tool`.
- Инструменты берутся из `kb_tools.create_kb_tools()`; их JSON-схемы — из
  `args_schema.model_json_schema()` (16 инструментов публикуются автоматически).
- `call_tool` выполняет синхронный `tool.invoke()` в пуле потоков
  (`anyio.to_thread.run_sync`), чтобы не блокировать event loop.
- Stateless-режим; `TransportSecuritySettings(enable_dns_rebinding_protection=False)` —
  иначе при пустом allowlist (значение по умолчанию) сервер отклоняет все запросы.
- Эндпоинты: `POST /mcp` (протокол MCP), `GET /health` (статус + список инструментов).
- CLI: `--host`, `--port`, `--json-response` (env: `MCP_HTTP_HOST`, `MCP_HTTP_PORT`).

### 2. Зависимости
- `.venv` уже содержал `starlette`, `uvicorn`, `anyio`; не хватало только `mcp` — установлен.
- `requirements_mcp.txt`: теперь `mcp>=1.21`, `uvicorn[standard]`, `starlette`
  (убраны `fastapi`, `python-multipart` — больше не используются).

### 3. Запуск через uv
- `.venv` проекта наполнен — поэтому uv должен использовать его, а не создавать новый.
- Рабочая команда: `uv run --active --no-project kb_tools_mcp_http.py`
  (`--active` использует активированный venv, `--no-project` отключает поиск pyproject).
- `start_kb_tools_mcp_http.ps1` пересоздан: ищет `.venv` в текущей/родительской папке,
  активирует, при отсутствии `mcp` ставит `requirements_mcp.txt`, запускает через uv.

### 4. Документация
- `QUICKSTART_MCP.md`, `continue.config.example.yaml`: SSE `/sse` → Streamable HTTP `/mcp`.

## Проверка (выполнена)

- `GET /health` → `{"status":"ok","tools_count":16,...}`.
- End-to-end через клиент `mcp.client.streamable_http`: `initialize` → `serverInfo kb-tools`,
  `tools/list` → 16 инструментов, `tools/call exact_search {"substring":"СКДПУ"}` →
  `isError=false`, корректные данные из ClickHouse.
- Подтверждён запуск рабочего сервера командой `uv run --active`.

## Запуск

```powershell
# venv активирован
uv run --active --no-project kb_tools_mcp_http.py --host 0.0.0.0 --port 8000
# или
.\start_kb_tools_mcp_http.ps1

# статус
Invoke-RestMethod http://localhost:8000/health
```

MCP endpoint: `http://localhost:8000/mcp` (transport: `streamable-http`).
