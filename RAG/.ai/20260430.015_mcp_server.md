# MCP Server для Knowledge Base Tools

**Дата:** 2026-04-30  
**Тип работы:** Создание HTTP API сервера

## 📋 Задача

Обернуть 15 инструментов базы знаний в HTTP API, создав MCP (Model Context Protocol) сервер.

**Цель:**
- Предоставить REST API для всех инструментов
- Совместимость с MCP протоколом от Anthropic
- Простая интеграция с LLM (OpenAI, Claude, etc.)
- Документация и примеры использования

## ✅ Решение

### 1️⃣ Создан MCP Server (mcp_server.py)

**Файл:** `mcp_server.py` (350+ строк)

**FastAPI приложение с эндпоинтами:**
- `GET /health` - проверка работоспособности
- `GET /tools` - список всех инструментов
- `GET /tools/{tool_name}` - описание инструмента
- `POST /tools/{tool_name}` - вызов инструмента
- `POST /invoke` - универсальный вызов (MCP-совместимый)

**Особенности:**
- Автоматическая инициализация векторного хранилища
- Извлечение JSON схем из Pydantic моделей
- Сериализация результатов в JSON
- Обработка ошибок
- CORS middleware
- Swagger/ReDoc документация

**Архитектура:**
```python
FastAPI App
    ↓
init_tools() → создание vectorstore + kb_tools
    ↓
_tools_map: {tool_name: BaseTool}
_tools_schemas: {tool_name: ToolSchema}
    ↓
API endpoints → вызов tool.invoke(args)
    ↓
Сериализация результата → JSON response
```

## 🎯 Функциональность

### API Эндпоинты

#### GET /health
Проверка здоровья сервера:
```json
{
  "status": "ok",
  "version": "1.0.0",
  "tools_count": 15,
  "vectorstore_status": "ok"
}
```

#### GET /tools
Список всех инструментов с JSON схемами:
```json
{
  "tools": [
    {
      "name": "semantic_search",
      "description": "Semantic similarity search...",
      "parameters": {
        "type": "object",
        "properties": {...}
      }
    }
  ],
  "count": 15
}
```

#### POST /tools/{tool_name}
Вызов инструмента:
```bash
curl -X POST http://localhost:8000/tools/semantic_search \
  -H "Content-Type: application/json" \
  -d '{"query": "СУБД", "top_k": 5}'
```

Ответ:
```json
{
  "tool": "semantic_search",
  "success": true,
  "result": {
    "query": "СУБД",
    "chunks": [...],
    "total_found": 5
  },
  "error": null
}
```

#### POST /invoke
MCP-совместимый универсальный вызов:
```bash
curl -X POST http://localhost:8000/invoke \
  -H "Content-Type: application/json" \
  -d '{
    "tool": "semantic_search",
    "arguments": {"query": "СУБД", "top_k": 5}
  }'
```

### Доступные инструменты (15)

1. **semantic_search** - семантический поиск
2. **exact_search** - точный поиск
3. **exact_search_in_file** - точный поиск в файле
4. **exact_search_in_file_section** - точный поиск в разделе
5. **multi_term_exact_search** - мультитерминовый поиск
6. **find_sections_by_term** - поиск разделов по термину
7. **find_relevant_sections** - поиск релевантных разделов
8. **regex_search** - regex поиск
9. **find_abbreviation_expansion** - расшифровка аббревиатур
10. **read_table** - чтение таблицы
11. **get_section_content** - содержимое раздела
12. **list_sections** - список разделов
13. **get_neighbor_chunks** - соседние чанки
14. **get_chunks_by_index** - чанки по индексу
15. **list_sources** - список источников

### Интеграция с LLM

#### OpenAI Function Calling

```python
import openai
import requests

# Получаем список инструментов
tools_response = requests.get("http://localhost:8000/tools")
tools = tools_response.json()["tools"]

# Конвертируем в формат OpenAI
openai_tools = [{
    "type": "function",
    "function": {
        "name": tool["name"],
        "description": tool["description"],
        "parameters": tool["parameters"]
    }
} for tool in tools]

# Используем в chat completion
response = openai.chat.completions.create(
    model="gpt-4",
    messages=[{"role": "user", "content": "Найди информацию о СУБД"}],
    tools=openai_tools
)

# Вызываем выбранный инструмент
tool_call = response.choices[0].message.tool_calls[0]
result = requests.post(
    f"http://localhost:8000/tools/{tool_call.function.name}",
    json=json.loads(tool_call.function.arguments)
).json()
```

## 📊 Примеры запросов

### 1. Семантический поиск (PowerShell)

```powershell
$body = @{
    query = "СУБД базы данных"
    top_k = 5
} | ConvertTo-Json

Invoke-RestMethod -Method Post `
  -Uri "http://localhost:8000/tools/semantic_search" `
  -ContentType "application/json" `
  -Body $body
```

### 2. Точный поиск (curl)

```bash
curl -X POST http://localhost:8000/tools/exact_search \
  -H "Content-Type: application/json" \
  -d '{"substring": "PostgreSQL", "limit": 10}'
```

### 3. Regex поиск IP-адресов

```bash
curl -X POST http://localhost:8000/tools/regex_search \
  -H "Content-Type: application/json" \
  -d '{
    "pattern": "\\d+\\.\\d+\\.\\d+\\.\\d+",
    "max_results": 50
  }'
```

## 🔧 Технические детали

### Сериализация результатов

Функция `_serialize_result()` конвертирует результаты в JSON:
- Pydantic модели → `model_dump()`
- Словари, списки → рекурсивно
- Базовые типы → as is
- Сложные объекты → `str(obj)`

### Обработка ошибок

- Глобальный exception handler
- Все ошибки возвращаются в формате:
```json
{
  "tool": "tool_name",
  "success": false,
  "result": null,
  "error": "error message"
}
```

### Thread safety

Векторное хранилище инициализируется один раз при старте.
LangChain инструменты используют `.clone()` для thread safety.

## 🔐 Безопасность

### Рекомендации:

1. **API Key аутентификация** - добавить в middleware
2. **Rate limiting** - slowapi или Redis-based
3. **HTTPS** - только через reverse proxy (nginx)
4. **CORS** - ограничить allow_origins
5. **Input validation** - уже есть через Pydantic

### Пример API Key auth:

```python
from fastapi import Header, HTTPException

async def verify_api_key(x_api_key: str = Header(...)):
    if x_api_key != os.getenv("API_KEY"):
        raise HTTPException(status_code=403, detail="Invalid API Key")

@app.post("/tools/{tool_name}", dependencies=[Depends(verify_api_key)])
async def invoke_tool_by_path(...):
    ...
```
