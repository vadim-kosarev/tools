# Реализация решений проблемы чанкинга

## Дата: 2026-04-26

## 🎯 Цель

Исправить проблему, когда exact search находит заголовок списка, но LLM не видит сам список, т.к. он находится в соседних чанках.

**Пример проблемы:**
```
Вопрос: "Какое ПО установлено на АРМ эксплуатационного персонала СОИБ КЦОИ?"
Ответ LLM: "Документация не указывает конкретное ПО" ❌

Реальность: Информация ЕСТЬ в документации! ✅
- Консоль администрирования Kaspersky Security Center 14.2.0.26967
- Агент администрирования Kaspersky Security Center 14.2.0.26967
- Kaspersky Endpoint Security 12.1.0.506 для Windows
```

---

## ✅ Реализованные решения

### 1. Исправлен инструмент `get_neighbor_chunks` в `kb_tools.py`

**Проблема:**
- Функция возвращала только соседние чанки (ДО и ПОСЛЕ якоря)
- Сам якорный чанк НЕ включался в результат
- Комментарий в коде: `# Якорный чанк (line_start == anchor) пропускаем`

**Решение:**

#### A. Обновлена модель `NeighborChunksResult`:
```python
class NeighborChunksResult(BaseModel):
    """Результат получения соседних чанков"""
    anchor_line: int = Field(description="line_start якорного чанка")
    anchor_chunk: Optional["ChunkResult"] = Field(default=None, description="Сам якорный чанк (если include_anchor=True)")  # ← НОВОЕ ПОЛЕ
    chunks_before: list[ChunkResult] = Field(description="Чанки до якоря")
    chunks_after: list[ChunkResult] = Field(description="Чанки после якоря")
```

#### B. Добавлен параметр `include_anchor: bool = True`:
```python
def get_neighbor_chunks(
    source: str, 
    line_start: int, 
    before: int = 5, 
    after: int = 5,
    include_anchor: bool = True  # ← НОВЫЙ ПАРАМЕТР (по умолчанию True)
) -> NeighborChunksResult:
```

#### C. Якорный чанк получается отдельным SQL-запросом:
```python
if include_anchor:
    query = f"""
        SELECT source, section, chunk_type, table_headers, content,
               line_start, line_end, chunk_index
        FROM {vs_clone._cfg.database}.{vs_clone._cfg.table} FINAL
        WHERE source = %(src)s
          AND line_start = %(ls)s
        LIMIT 1
    """
    result = vs_clone._client.query(query, parameters={"src": source, "ls": line_start})
    # Конвертация в ChunkResult
    anchor_chunk = _doc_to_chunk_result(anchor_doc)
```

**Результат:**
- ✅ Якорь теперь **включается** в результат по умолчанию
- ✅ Обратная совместимость: можно отключить через `include_anchor=False`
- ✅ Тест показал: фраза "установлены следующие" + полный список ПО

### 2. Обновлены system prompts для агентов

**Проблема:**
- Агент мог найти заголовок через exact_search
- Но не понимал, что нужно расширить контекст для получения самого списка
- Отвечал "информация не найдена" при наличии информации в следующих чанках

**Решение:**

#### A. Обновлен `rag_lc_agent.py` (`_SYSTEM_PROMPT_TEMPLATE`):

Добавлено **ПРАВИЛО 10** в system prompt:

```markdown
ПРАВИЛО 10 — АВТОМАТИЧЕСКОЕ РАСШИРЕНИЕ КОНТЕКСТА ДЛЯ ЗАГОЛОВКОВ СПИСКОВ. ⚠️
  Проблема: exact_search часто находит ЗАГОЛОВОК списка/таблицы, но САМ СПИСОК
  находится в следующих чанках, которые не содержат поисковый термин.
  
  ПРИЗНАКИ ЗАГОЛОВКА В НАЙДЕННОМ ЧАНКЕ:
    - "установлены следующие"
    - "включает в себя:"
    - "состоит из:"
    ...
```

#### B. Обновлен `system_prompt.md` (для `rag_lg_agent.py`):

Добавлен раздел **"Стратегии поиска в документации"**:

```markdown
### ⚠️ КРИТИЧЕСКОЕ ПРАВИЛО: Автоматическое расширение контекста

**ПРОБЛЕМА:** 
`exact_search` часто находит **ЗАГОЛОВОК** списка/таблицы, но **САМ СПИСОК** находится в следующих чанках...

**ОБЯЗАТЕЛЬНОЕ ДЕЙСТВИЕ при обнаружении заголовка:**

**Option A (предпочтительно):**
- get_section_content(source_file, section) → Получить ПОЛНЫЙ раздел

**Option B (если раздел большой):**
- get_neighbor_chunks(source, line_start, after=15) → Якорь + 15 чанков
```

**Результат:**
- ✅ ReAct-агент (`rag_lc_agent.py`) теперь понимает, когда нужно расширять контекст
- ✅ LangGraph-агент (`rag_lg_agent.py`) получил те же инструкции через `system_prompt.md`
- ✅ Два варианта расширения: `get_section_content` (предпочтительно) или `get_neighbor_chunks`
- ✅ Примеры правильной и неправильной цепочки действий
