# Улучшенная система логирования с нумерацией узлов

**Дата:** 2026-04-30  
**Тип работы:** Улучшение системы логирования

## 📋 Задача

Изменить систему логирования для использования фиксированной нумерации узлов графа вместо последовательного counter.

**Проблема (было):**
- Файлы логов именовались последовательным счетчиком: `001_llm_request.log`, `002_llm_response.log`, и т.д.
- Сложно идентифицировать, какой узел создал файл
- Нет явной связи между логами и узлами графа
- Файлы state именовались: `001_agent_state_planner.json`

**Цель:**
- Фиксированная нумерация узлов: 001 - planner, 002 - tool_selector, 003 - tool_executor, 004 - analyzer, 005 - refiner, 006 - final
- Имена файлов: `{node_number}_{node_name}_{counter}_llm_request.log`
- Имена файлов state: `{node_number}_{node_name}_state.json` (или `_iter{N}_state.json` для итеративных узлов)

## ✅ Решение

### 1️⃣ Добавлены константы нумерации узлов

**Файл:** `rag_lg_agent.py`

```python
# Нумерация узлов графа для логирования
NODE_NUMBERS = {
    "planner": "001",
    "tool_selector": "002",
    "tool_executor": "003",
    "analyzer": "004",
    "refiner": "005",
    "final": "006",
}
```

### 2️⃣ Обновлена функция _save_agent_state

**Файл:** `rag_lg_agent.py`

**Новая логика именования:**
```python
def _save_agent_state(state: AgentState, node_name: str, llm_logger: LlmCallLogger) -> None:
    # Получаем номер узла
    node_number = NODE_NUMBERS.get(node_name, "999")
    
    # Для итеративных узлов добавляем номер итерации
    iteration = state.get("iteration", 0)
    if node_name in ("tool_selector", "tool_executor", "analyzer", "refiner") and iteration > 0:
        filename = f"{node_number}_{node_name}_iter{iteration}_state.json"
    else:
        filename = f"{node_number}_{node_name}_state.json"
```

### 3️⃣ Расширен LlmCallLogger

**Файл:** `llm_call_logger.py`

**Добавлены поля:**
```python
def __init__(
    self,
    ...
    node_numbers: dict[str, str] | None = None  # Маппинг node_name → номер
) -> None:
    ...
    self._node_numbers = node_numbers or {}
    self._current_node = None  # Текущий узел
    self._current_node_counter = 0  # Счетчик внутри узла
```

**Добавлены методы:**
```python
def set_current_node(self, node_name: str) -> None:
    """Устанавливает текущий узел для логирования."""
    with self._lock:
        self._current_node = node_name
        self._current_node_counter = 0  # Сбрасываем счетчик

def _next_node_number(self) -> tuple[str, int]:
    """Возвращает (node_prefix, counter) для текущего узла."""
    with self._lock:
        node_prefix = self._node_numbers.get(self._current_node, "999")
        self._current_node_counter += 1
        return node_prefix, self._current_node_counter
```

## 🔍 Технические детали

### Логика счетчиков:

1. **node_prefix** (001-006) - фиксированный префикс узла
2. **node_counter** (001, 002, 003...) - счетчик вызовов внутри узла
3. Счетчик сбрасывается при каждом `set_current_node()`

### Thread safety:

Все операции с счетчиками защищены `threading.Lock`:
```python
with self._lock:
    self._current_node = node_name
    self._current_node_counter = 0
```

### Поддержка итераций:

Для итеративных узлов (tool_selector, tool_executor, analyzer, refiner) в имени state файла добавляется номер итерации:
```python
if node_name in ("tool_selector", "tool_executor", "analyzer", "refiner") and iteration > 0:
    filename = f"{node_number}_{node_name}_iter{iteration}_state.json"
```
