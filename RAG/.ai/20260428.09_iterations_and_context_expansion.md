**Дата:** 2026-04-28  
**Версия:** 09  
**Файл:** `RAG/rag_lg_agent.py`

## Задача

1. Увеличить количество итераций в агенте до 5
2. Добавить автоматическое расширение контекста нужными данными из БД

---

## Внесенные изменения

### 1. Увеличение MAX_ITERATIONS до 5

**Было:**
```python
MAX_ITERATIONS = 3  # Максимум итераций уточнения
```

**Стало:**
```python
MAX_ITERATIONS = 5  # Максимум итераций уточнения
AUTO_EXPAND_CONTEXT = True  # Автоматическое расширение контекста через get_neighbor_chunks
```

**Эффект:**
- Агент теперь может делать до 5 итераций уточнения вместо 3
- Больше возможностей для глубокого поиска и уточнения данных
- Улучшенная полнота ответа на сложные вопросы

---

### 2. Автоматическое расширение контекста

Добавлена функция `_auto_expand_context()` которая:

#### Алгоритм работы:

1. **Анализ результатов инструментов**
   - Сканирует результаты всех выполненных инструментов поиска
   - Ищет паттерны типа `[file.md] — Section (line 123)`

2. **Извлечение метаданных**
   - Использует regex для извлечения `source` (имя файла) и `line_start` (номер строки)
   - Паттерн: `r'\[([^\]]+\.md)\][^\(]*\(line (\d+)\)'`

3. **Автоматический вызов get_neighbor_chunks**
   - Для каждого найденного чанка вызывает `get_neighbor_chunks`
   - Параметры: `before=3, after=3` (по 3 чанка до и после)
   - Максимум 5 расширений на итерацию (избегаем перегрузки)

4. **Добавление расширенного контекста**
   - Результаты get_neighbor_chunks добавляются в `all_tool_results`
   - Помечаются флагом `auto_expanded: True`
   - Логируются в отдельной секции `AUTO CONTEXT EXPANSION`

#### Преимущества:

✅ **Автоматическое обогащение контекста**
- Агент видит не только найденный чанк, но и окружающие чанки
- Улучшается понимание контекста каждого факта

✅ **Интеллектуальное ограничение**
- Максимум 5 расширений на итерацию (не перегружаем LLM)
- Максимум 2 расширения на результат инструмента
- Пропуск ошибок и уже расширенных контекстов

✅ **Прозрачность**
- Каждое расширение логируется
- Видно какие чанки расширены и сколько символов добавлено
- Маркер `auto_expanded: True` позволяет отличить от ручных запросов

---

### 3. Интеграция в action_node

**Место вызова:** После выполнения всех tool calls, перед добавлением в историю

```python
# Выполняем все tool calls
tool_results = [...]

# ... добавление tool_results в all_tool_results и messages

# Автоматическое расширение контекста
try:
    expansion_results = _auto_expand_context(
        tool_results,
        tools_map,
        llm_logger,
        iteration
    )
    state["all_tool_results"].extend(expansion_results)
except Exception as e:
    logger.warning(f"Ошибка при автоматическом расширении контекста: {e}")
```

**Особенности:**
- Вызов обернут в try-except - ошибки расширения не прерывают агента
- Расширения добавляются ПОСЛЕ основных результатов
- Используется тот же `tools_map` что и для основных инструментов

## Пример работы

### Запрос: "Найди все СУБД с IP-адресами"

**Итерация 1: Первичный поиск**
```
ACTION:
  - semantic_search("база данных СУБД")
  - exact_search("PostgreSQL")
  
RESULTS:
  [1] [servers.md] — Database Servers > PostgreSQL (line 150)
  Content: PostgreSQL установлен на сервере 10.0.0.1

AUTO CONTEXT EXPANSION:
  - servers.md:line 150 (±3/3 chunks)
    → Получены чанки 147-153: конфигурация, параметры запуска, бэкапы
```

**Observation:**
```
Найдено: PostgreSQL на 10.0.0.1
Контекст расширен: получена информация о конфигурации и порте 5432
Не найдено: MySQL, Oracle, MongoDB
```

**Refine:**
```
needs_refinement: true
refinement_plan:
  - Поиск MySQL, Oracle, MongoDB
  - Уточнить конфигурацию PostgreSQL (уже есть в контексте)
```

**Итерация 2: Уточняющий поиск**
```
ACTION:
  - multi_term_exact_search(["MySQL", "Oracle", "MongoDB"])
  - exact_search_in_file_section("порт", "servers.md", "PostgreSQL")

RESULTS (автоматически расширены контекстом)
```

## Включение/отключение

Автоматическое расширение контекста управляется константой:

```python
AUTO_EXPAND_CONTEXT = True  # Включено по умолчанию
```

## Влияние на производительность

### До изменений (3 итерации):
- Максимум запросов: 3 × (2-4 tools) = 6-12 инструментов
- Контекст: только найденные чанки

### После изменений (5 итераций + auto-expand):
- Максимум запросов: 5 × (2-4 tools + 5 expansions) = 10-45 инструментов
- Контекст: найденные чанки + соседние (±3)

**Оптимизация:**
- Расширение происходит только для чанков с line_start
- Лимит 5 расширений на итерацию
- Пропуск уже расширенных контекстов
- Try-except обработка ошибок
