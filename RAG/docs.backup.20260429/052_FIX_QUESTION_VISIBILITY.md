# Фикс: LLM не видит вопрос пользователя

## Проблема

LLM отвечала: `"thought":"Ожидание вопроса пользователя для формирования плана поиска"`

Вопрос передавался, но терялся в огромном системном промпте (600+ строк).

## Решение

### 1. Упрощен plan/system.md

**Было:** 
- Include system_prompt_base.md (600+ строк)
- Много деталей о стратегиях
- Вопрос упоминался в конце

**Стало:**
- Компактный промпт (~30 строк)
- Только суть: задача, инструменты, формат
- Четкая структура

```markdown
# RAG-агент: ЭТАП ПЛАНИРОВАНИЯ

Ты — RAG-агент для работы с технической документацией в ClickHouse.

## Задача на этом этапе
**Составить ПЛАН поиска** для ответа на вопрос пользователя.

## Доступные инструменты
{{ available_tools }}

## Типичные стратегии
**"Что такое X?"** → semantic_search + exact_search
**"Дай список X"** → find_sections_by_term + get_section_content  
**"Какое X на Y?"** → multi_term_exact_search (основные термины + синонимы)

## Формат ответа (ТОЛЬКО JSON!)
{
  "thought": "краткий анализ вопроса",
  "plan": ["шаг 1", "шаг 2", "шаг 3"]
}
```

### 2. Усилено выделение вопроса в plan/user.md

**Было:**
```markdown
================================================================================
🎯 ВОПРОС ПОЛЬЗОВАТЕЛЯ (ГЛАВНАЯ ЗАДАЧА):
================================================================================

{{ user_query }}

================================================================================
```

**Стало:**
```markdown

🎯🎯🎯 **ВОПРОС ПОЛЬЗОВАТЕЛЯ** 🎯🎯🎯

{{ user_query }}

🎯🎯🎯🎯🎯🎯🎯🎯🎯🎯🎯🎯🎯

Сформируй ПЛАН поиска для ответа на ЭТОТ вопрос.
```

## Измененные файлы

- ✅ `prompts/plan/system.md` — упрощен с 43 до ~30 строк
- ✅ `prompts/plan/user.md` — усилено визуальное выделение

## Философия изменений

### Меньше текста — больше фокуса

**Проблема длинных промптов:**
- LLM теряет важную информацию в массе текста
- Attention рассеивается
- Инструкции становятся фоном

**Решение:**
- Системный промпт: только СУТЬ задачи
- User промпт: ВОПРОС максимально заметен
- Детали: только необходимые

### JSON-first

Вместо объяснений "что делать" → сразу "вот формат ответа":

```json
{
  "thought": "...",
  "plan": ["..."]
}
```

LLM лучше понимает через примеры, чем через описания.

### Визуальные маркеры

`🎯🎯🎯` — работает как attention mechanism:
- Выделяется среди текста
- Легко найти глазами (и токенами LLM)
- Универсальный символ "цель/задача"

## Дальнейшие улучшения

Если проблема останется, можно:

1. **Переместить вопрос в начало system message:**
   ```markdown
   Вопрос пользователя: {{ user_query }}
   
   # RAG-агент: ЭТАП ПЛАНИРОВАНИЯ
   ...
   ```

2. **Повторить вопрос в нескольких местах:**
   - В system (начало)
   - В user (основное)
   - В конце (напоминание)

3. **Использовать XML-теги:**
   ```xml
   <user_question>
   {{ user_query }}
   </user_question>
   ```

4. **Few-shot примеры с вопросами:**
   ```markdown
   Пример 1:
   Вопрос: "что такое СОИБ?"
   Plan: {
     "thought": "нужно определение СОИБ",
     "plan": ["semantic_search", "exact_search"]
   }
   
   Твоя очередь:
   Вопрос: {{ user_query }}
   ```

Но сначала тестируем текущее решение — оно должно работать.

## Уроки

1. **Краткость > Подробность** для LLM промптов
2. **Визуальные маркеры** привлекают внимание
3. **Примеры > Объяснения** для формата ответа
4. **Повторение важного** помогает не потеряться
5. **Тестирование с --steps** экономит время отладки
