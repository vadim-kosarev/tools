# Single-Pass RAG Agent — Документация

**Файл:** `rag_single_pass_agent.py`  
**Дата:** 2026-04-26  
**Тип:** Линейный граф без итераций

---

## 📋 Обзор

**Single-pass агент** — упрощённая версия RAG агента, выполняющая поиск **за один проход** без итераций.

### Философия

Базируется на принципах `analytical_agent` из `system_prompt.md`:
- ✅ **plan** → **action** → **observation** → **final**
- ✅ Работа ТОЛЬКО с данными из tools
- ✅ Честное признание отсутствия данных
- ✅ Структурированные JSON ответы

---

## 🔄 Pipeline (граф)

```
START
  ↓
query_analyzer     (plan + action) — анализ вопроса, выбор инструментов
  ↓
tool_executor      (технический)  — параллельное выполнение 2-4 tools
  ↓
data_analyzer      (observation)  — извлечение данных из результатов
  ↓
answer_generator   (final)        — формирование ответа с confidence
  ↓
answer_evaluator   (метрики)      — оценка качества (relevance, completeness, accuracy)
  ↓
END
```

**Особенности:**
- 🚀 **Один проход** — нет итераций, быстрее работает
- 🔧 **Параллельные tools** — выполняет 2-4 инструмента одновременно
- 📊 **Автооценка** — встроенная оценка качества ответа
- 🎯 **Структурированность** — все узлы возвращают Pydantic модели

---

## 📦 Состояние агента (AgentState)

```python
class AgentState(TypedDict):
    user_query: str                    # Вопрос пользователя
    plan: list[str]                    # План поиска (3-5 шагов)
    tool_calls: list[dict]             # [{tool, args}] — что вызвать
    tool_results: list[dict]           # [{tool, result}] — результаты
    found_data: list[dict]             # Структурированные данные
    evidence: list[str]                # Цитаты из документов
    gaps: list[str]                    # Что не найдено
    final_answer: str                  # JSON с ответом
    evaluation: dict                   # Оценка качества
```

## 🎯 Узлы графа

### 1. query_analyzer (plan + action)

**Задача:** Проанализировать вопрос и выбрать 2-4 инструмента

**Модель:** `QueryPlan`
```python
{
  "thought": "краткое рассуждение",
  "plan": ["шаг 1", "шаг 2", "шаг 3"],
  "tool_calls": [
    {"tool": "semantic_search", "args": {"query": "...", "top_k": 10}},
    {"tool": "exact_search", "args": {"substring": "...", "limit": 30}}
  ]
}
```

**Принцип выбора:**
- Комбинируй semantic (концептуальный) + exact (точный)
- Для перечислений: `multi_term_exact_search`
- Для IP/портов: `regex_search`
- Для таблиц: `read_table`

### 2. tool_executor (технический)

**Задача:** Выполнить все инструменты параллельно

**Особенности:**
- Без LLM — прямое выполнение tools
- Обработка ошибок для каждого инструмента
- Логирование результатов

### 3. data_analyzer (observation)

**Задача:** Извлечь данные из результатов

**Модель:** `DataAnalysis`
```python
{
  "found_data": [
    {"entity": "PostgreSQL", "attribute": "ip", "value": "10.0.0.1",
     "source": "file.md", "context": "цитата"}
  ],
  "evidence": ["ТОЧНАЯ цитата 1", "ТОЧНАЯ цитата 2"],
  "gaps": ["что НЕ найдено"],
  "summary": "краткая сводка"
}
```

**Правила:**
- ❌ ЗАПРЕЩЕНО придумывать
- ✅ ТОЛЬКО из результатов tools
- ✅ Если ничего нет → `found_data: []`

### 4. answer_generator (final)

**Задача:** Сформировать финальный ответ

**Модель:** `FinalAnswer`
```python
{
  "summary": "краткий ответ (1-2 предложения)",
  "details": "подробное объяснение",
  "data": [...],  # из found_data
  "sources": ["file1.md", "file2.md"],
  "confidence": 0.85  # уверенность (0-1)
}
```

**Формула confidence:**
- Много данных + много источников → 0.8-0.95
- Мало данных → 0.4-0.6
- Нет данных → 0.1-0.3

### 5. answer_evaluator (метрики)

**Задача:** Оценить качество ответа

**Модель:** `AnswerEvaluation`
```python
{
  "relevance": 4,        # 1-5 (отвечает ли на вопрос)
  "completeness": 3,     # 1-5 (достаточно ли информации)
  "accuracy": 5,         # 1-5 (есть ли источники)
  "overall_score": 4.0,  # среднее
  "missing": ["что отсутствует"],
  "feedback": "краткий комментарий"
}
```

**Правила оценки:**
- Нет источников → снижаем accuracy
- Мало данных → снижаем completeness
- Не по теме → снижаем relevance

## 🚀 Использование

### Командная строка

```bash
# Простой запрос
python rag_single_pass_agent.py "найди все СУБД"

# С подробным выводом
python rag_single_pass_agent.py "найди все IP серверов" --verbose

# Интерактивный режим
python rag_single_pass_agent.py
```

### В коде

```python
from rag_single_pass_agent import run_query

state = run_query("какие СУБД используются?", verbose=True)

# Результат
answer = json.loads(state["final_answer"])
print(f"Summary: {answer['summary']}")
print(f"Confidence: {answer['confidence']}")

# Оценка
eval_data = state["evaluation"]
print(f"Overall score: {eval_data['overall_score']}/5")
```

**Когда использовать single-pass:**

**Когда использовать rag_lg_agent:**
- ✅ Сложные вопросы
- ✅ Нужна глубокая проработка
- ✅ Итеративное уточнение
- ✅ Максимальная полнота

## 🎓 Системные промпты

Все промпты следуют принципам `analytical_agent`:

### query_analyzer
```
Статусы: plan + action
Задача: анализ + выбор инструментов
Выход: QueryPlan (plan, tool_calls)
```

### data_analyzer
```
Статус: observation
Задача: извлечение данных из результатов
Выход: DataAnalysis (found_data, evidence, gaps)
```

### answer_generator
```
Статус: final
Задача: формирование ответа
Выход: FinalAnswer (summary, details, confidence)
```

### answer_evaluator
```
Задача: оценка качества
Выход: AnswerEvaluation (relevance, completeness, accuracy)
```

### Фаза 2: Добавление аналитики

После текущей версии планируется:
