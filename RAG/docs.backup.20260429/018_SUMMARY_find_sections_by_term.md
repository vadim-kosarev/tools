# ✅ ЗАВЕРШЕНО: Добавление инструмента find_sections_by_term

## Дата: 2026-04-26

---

## 📋 ЗАДАЧА

Создать инструмент для поиска по точным терминам, который возвращает список уникальных пар (source, section) где найдены совпадения, вместо самих чанков. Это позволит агенту быстро определить, какие разделы наиболее полезны для детального изучения.

**Требования:**
- Поиск по точному термину (case-insensitive)
- Возврат списка всех разделов в пределах указанного лимита чанков
- Группировка по source + section с подсчётом упоминаний
- Агрегированный обзор вместо детальных чанков

## ✅ ВЫПОЛНЕНО

### 1. Новый метод в ClickHouseVectorStore (clickhouse_store.py)

#### ✅ `exact_search_sections()`
```python
def exact_search_sections(
    substring: str,
    limit: int = 100,
    chunk_type: Optional[str] = None,
    source: Optional[str] = None,
) -> list[tuple[str, str, int]]
```

**Реализация:**
- Сканирует до `limit` чанков с термином
- Извлекает уникальные пары (source, section)
- Подсчитывает количество упоминаний в каждом разделе
- Возвращает список отсортированный по количеству упоминаний (DESC)

**SQL запрос:**
```sql
SELECT source, section, count() AS match_count
FROM (
    SELECT source, section
    FROM db.table FINAL
    WHERE positionCaseInsensitive(content, 'term') > 0
    LIMIT 100
)
GROUP BY source, section
ORDER BY match_count DESC, source, section
```

**Возвращает:**
```python
[
    ("file1.md", "Section A", 15),  # 15 упоминаний в разделе
    ("file1.md", "Section B", 8),   # 8 упоминаний в разделе
    ("file2.md", "Section C", 3),   # 3 упоминания в разделе
]
```

### 2. Новый инструмент в kb_tools.py

#### ✅ Pydantic схема `FindSectionsByTermInput`
```python
class FindSectionsByTermInput(BaseModel):
    substring: str         # термин для поиска
    limit: int = 100       # макс. чанков для сканирования (1-500)
    chunk_type: Optional[str] = None  # фильтр по типу чанка
    source: Optional[str] = None      # фильтр по файлу
```

#### ✅ @tool функция `find_sections_by_term()`

**Docstring:**
```
Find all unique sections (source + section) containing exact substring matches.

Best for: discovering which sections contain specific terms before diving deep.
Returns a list of (source, section) pairs with match counts, sorted by relevance.

This scans up to 'limit' chunks and returns ALL unique sections found within
that scan. For example, with limit=100 searching "PostgreSQL", you'll get
all sections that mention PostgreSQL within the first 100 matching chunks.
```

**Формат вывода:**
```
Найдено разделов с термином 'PostgreSQL': 22 из 7 файлов

[F] file1.md:
   • Section A (15 упоминаний)
   • Section B (8 упоминаний)

[F] file2.md:
   • Section C (3 упоминания)
```

### Сценарий 2: Приоритизация разделов
```python
# Найти все разделы с "WebLogic"

# → Начинает с разделов с максимальным количеством упоминаний
```

### Сценарий 3: Комбинация инструментов
```python
# Шаг 1: Найти разделы (быстрый обзор)
sections = find_sections_by_term("IP адрес", limit=100)

# → "Сетевые настройки" (8 упоминаний)

# Шаг 2: Детальный поиск в приоритетном разделе
details = exact_search("IP адрес", section="Карта сетевых потоков")
# → Конкретные чанки с IP-адресами
```
