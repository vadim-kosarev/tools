# rag_lg_agent.py - Single-Pass Агент

**Дата переписывания:** 2026-04-26  
**Тип:** Линейный single-pass агент следующий system_prompt.md

---

## ✅ Что сделано

### 1. Полностью переписан rag_lg_agent.py

**Старая архитектура (удалена):**
```
planner → tool_selector → tool_executor → analyzer → refiner → finalizer
   ↑                                                      |
   +------------------- (цикл до 6 итераций) -----------+
```

**Новая архитектура (single-pass):**
```
START → plan_node → action_node → observation_node → final_node → END
```

**Один проход без итераций!**

---

## 📋 Описание узлов

### 1. plan_node (status: "plan")
- Читает system_prompt.md
- Анализирует вопрос пользователя
- Формирует план поиска (3-5 шагов)
- Возвращает JSON: `{"status": "plan", "step": 1, "thought": "...", "plan": [...]}`

### 2. action_node (status: "action")
- Выбирает 2-4 инструмента для параллельного выполнения
- Возвращает JSON: `{"status": "action", "step": 2, "thought": "...", "action": [...]}`
- **Выполняет все инструменты сразу**
- Логирует каждый tool call и результат

### 3. observation_node (status: "observation")
- Анализирует результаты инструментов
- Извлекает ТОЛЬКО факты из результатов
- НЕ придумывает, НЕ домысливает
- Возвращает JSON: `{"status": "observation", "step": 3, "thought": "...", "observation": "..."}`

### 4. final_node (status: "final")
- Формирует итоговый ответ
- Структура: summary, details, data[], sources[], confidence
- Возвращает JSON: `{"status": "final", "step": 4, "thought": "...", "final_answer": {...}}`

---

## 🎯 Следование system_prompt.md

### Строгая схема JSON

Каждый узел возвращает JSON по схеме из `system_prompt.md`:
- `status`: plan | action | observation | final
- `step`: номер шага (1-4)
- `thought`: краткое рассуждение
- Специфичные поля для каждого статуса

### Принципы работы

✅ **Работа ТОЛЬКО с данными из tools**  
✅ **Честное признание пробелов**  
✅ **Никаких выдумок и домыслов**  
✅ **Структурированные Pydantic модели**

## 🎯 Преимущества новой архитектуры

### 1. Простота
- Линейный граф без циклов
- Легко отлаживать
- Предсказуемое время выполнения

### 2. Полное логирование
- Весь system prompt виден в логе
- Все user messages
- Все assistant responses (JSON)
- Все tool calls и results
- Полная история messages

### 3. Следование стандартам
- Строгое следование system_prompt.md
- Pydantic модели для всех ответов
- JSON schema validation

### 4. Производительность
- Параллельное выполнение 2-4 tools
- Нет повторных итераций
- Один проход
