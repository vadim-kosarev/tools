## 📚 ОПИСАННЫЕ ИНСТРУМЕНТЫ (14)

### Поисковые инструменты:

1. **`semantic_search`** - семантический поиск по эмбеддингам
   - Параметры: `query`, `top_k=10`, `source`, `section`
   - Возвращает: отформатированный список чанков

2. **`exact_search`** - точный поиск подстроки
   - Параметры: `substring`, `limit=30`, `chunk_type`, `source`, `section`
   - Возвращает: список чанков

3. **`exact_search_in_file`** - точный поиск в файле
   - Параметры: `substring`, `source_file`, `limit=30`, `chunk_type`
   - Возвращает: список чанков из файла

4. **`exact_search_in_file_section`** - точный поиск в разделе
   - Параметры: `substring`, `source_file`, `section`, `limit=30`, `chunk_type`
   - Возвращает: список чанков из раздела

5. **`multi_term_exact_search`** - поиск по нескольким терминам
   - Параметры: `terms`, `limit=30`, `chunk_type`, `source`, `section`
   - Возвращает: чанки сгруппированные по покрытию терминов

### Инструменты обзора разделов:

6. **`find_sections_by_term`** - какие разделы содержат термин
   - Параметры: `substring`, `limit=100`, `chunk_type`, `source`
   - Возвращает: список разделов с количеством упоминаний

7. **`find_relevant_sections`** ⭐ PRIMARY TOOL - двухэтапный поиск
   - Параметры: `query`, `exact_terms=[]`, `limit=50`, `source`
   - Возвращает: топ-N разделов с приоритизацией (🎯 NAME MATCH первые)

### Специализированные инструменты:

8. **`regex_search`** - regex-поиск в .md файлах
   - Параметры: `pattern`, `max_results=20`
   - Возвращает: совпадения с контекстом

9. **`read_table`** - чтение строк таблицы
   - Параметры: `section`, `source_file`, `limit=50`
   - Возвращает: строки в формате "Колонка: значение"

10. **`get_section_content`** - полный текст раздела
    - Параметры: `source_file`, `section`
    - Возвращает: полный текст раздела из .md

### Навигационные инструменты:

11. **`list_sections`** - список разделов в файле/БД
    - Параметры: `source_file` (Optional)
    - Возвращает: дерево разделов

12. **`list_all_sections`** - все уникальные разделы
    - Параметры: нет
    - Возвращает: 827 уникальных пар (source, section)

13. **`get_neighbor_chunks`** - контекст вокруг чанка
    - Параметры: `source`, `line_start`, `before=5`, `after=5`
    - Возвращает: чанки до и после якоря

14. **`list_sources`** - список файлов в БД
    - Параметры: нет
    - Возвращает: файлы с количеством чанков
