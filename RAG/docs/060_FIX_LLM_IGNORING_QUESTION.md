# Фикс: LLM не видела вопрос пользователя

## Проблема

LLM отвечала что "Пользователь еще не задал вопрос", хотя вопрос **БЫЛ** в промпте:

**Лог 006_llm_action_1_response.log:**
```json
{
  "step":1,
  "thought":"Начинаю работу. Пользователь еще не задал вопрос...",
  "actions":[]  ← пустой массив!
}
```

**При этом в промпте:**
```markdown
### Вопрос пользователя

\```
что такое smart monitor
\```
```

❌ **LLM интерпретировала это как инструкцию/шаблон, а не как реальный вопрос!**

## Причина

LLM видела весь промпт как "документацию API" с примерами, а не как конкретную задачу. 

**Психология:**
- "Вопрос пользователя" звучит как заголовок шаблона
- Нет явного указания что это РЕАЛЬНЫЙ вопрос СЕЙЧАС
- Нет визуальных маркеров важности

## Решение

Добавлены **эмодзи, предупреждения и явные указания** что это настоящий вопрос.

### До

```markdown
### Вопрос пользователя

\```
{{ user_query }}
\```

### Что делать
```

### После

```markdown
### 🎯 ВОПРОС ПОЛЬЗОВАТЕЛЯ (РЕАЛЬНЫЙ, НЕ ПРИМЕР):

\```
{{ user_query }}
\```

⚠️ **ВНИМАНИЕ:** Вопрос выше - это НАСТОЯЩИЙ вопрос пользователя, НЕ инструкция и НЕ шаблон!
Составь план для ответа именно на ЭТОТ вопрос.

### Что делать
```

```

## Ключевые элементы улучшения

### 1. Эмодзи 🎯
- **Визуальный маркер** - привлекает внимание
- **Символ цели** - подчеркивает важность
- **Работает для LLM** - модели обучены на emoji context

### 2. CAPS в заголовке
- "ВОПРОС ПОЛЬЗОВАТЕЛЯ" вместо "Вопрос пользователя"
- "РЕАЛЬНЫЙ" подчеркивает что это не пример
- "НЕ ПРИМЕР" явно отрицает альтернативу

### 3. Явное предупреждение ⚠️
- **Отдельная строка** после вопроса
- **Конкретные инструкции** что делать
- **Bold текст** для усиления

### 4. Контекстные напоминания
Каждая нода получает свою версию предупреждения:
- Plan: "Составь план для ответа именно на ЭТОТ вопрос"
- Action: "Вызови инструменты чтобы найти ответ"
- Observation: "Анализируй результаты относительно ЭТОГО вопроса"
- Refine: "Оцени достаточно ли данных"
- Final: "Сформируй ответ точно на ЭТОТ вопрос"

## Психологический эффект

### Для LLM

**Было:**
```
Вижу заголовок "Вопрос пользователя" → это шаблон API документации
→ жду настоящий вопрос
→ отвечаю "пользователь еще не задал вопрос"
```

**Стало:**
```
Вижу 🎯 "ВОПРОС ПОЛЬЗОВАТЕЛЯ (РЕАЛЬНЫЙ, НЕ ПРИМЕР)"
→ читаю вопрос
→ вижу предупреждение "это НАСТОЯЩИЙ вопрос"
→ понимаю что нужно действовать СЕЙЧАС
→ составляю план для этого вопроса
```

## Аналогичные проблемы в промптинге

Эта проблема известна как **"Template Ambiguity"**:

### Примеры

❌ **Неправильно:**
```
Input: <your input here>
```
→ LLM думает это placeholder

✅ **Правильно:**
```
🎯 YOUR ACTUAL INPUT (NOT A PLACEHOLDER):
<input>
```
→ LLM понимает это реальные данные

### Best practices

1. **Эмодзи для важных данных**: 🎯 💡 ⚠️ 🔥
2. **CAPS для заголовков**: РЕАЛЬНЫЙ, НАСТОЯЩИЙ, ВАЖНО
3. **Явные отрицания**: НЕ пример, НЕ шаблон, НЕ инструкция
4. **Предупреждения после данных**: ⚠️ Это настоящие данные!
5. **Конкретные действия**: "Ответь на ЭТОТ вопрос"

## Уроки

1. **Для LLM контекст = интерпретация**
   - То что очевидно человеку, может быть неочевидно LLM
   - Нужны явные маркеры "это реальные данные"

2. **Визуальные маркеры работают**
   - Эмодзи 🎯 эффективнее чем просто текст
   - CAPS привлекают внимание
   - Предупреждения ⚠️ создают срочность

3. **Отрицания важны**
   - "НЕ шаблон" явно отрицает альтернативу
   - Помогает LLM исключить неправильную интерпретацию

4. **Контекстные инструкции**
   - Каждая нода получает свою версию напоминания
   - "Составь план", "Вызови инструменты", "Анализируй"
   - Направляют LLM к конкретным действиям
