Рефакторим текущую реализацию
- гланове логировать в лог и не глотать Exception-s
- упрощаем реализацию для четкого понимания

На основе текущей реализации либо убедиться, либо переделать следующие аспекты.

Узлы графа:

AgentState (обязательные поля)
- запрос пользователя
- набор тулов
- контекст - данные для посторения финального ответа
- история (user_prompts, LLM replies, tool_execution с аргументами, сводна яинформация от tool_execution, сводная информация от refiner-а)
  Note: все хранится в виде Pydantic-объектов и сам AgetnState - Pydantic объект с четкой схемой.

planner
- на основе запроса строит план выполнения тулов - текстом
- вовзращает план на русском языке

tool_selector
- на основе плана от planner-а
- делает json-интсрукии для узла выполнения

tool_executor
- берет json от tool_selecotr, выполняет вызовы тулов
- возвращает результат вызова тулов

analyzer
- фильтрует результат от tool_executor-а
- добавляет в контекст все результаты от tool_executor-а

refiner
- определяет какой узел нужно выполнить дальше - в данном варианте - всегда final

final
- по текущему контексту строит ответ на запрос пользователя - делает вызов в LLM с передачей контекста
