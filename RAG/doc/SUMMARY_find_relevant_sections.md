## 📋 ЗАДАЧА

Реализовать двухэтапную стратегию поиска разделов:
1. **ЭТАП 1:** Поиск по **названию раздела** - точное совпадение с запросом пользователя
2. **ЭТАП 2:** Поиск по **содержимому** - разделы содержащие точные термины
3. **Объединение:** Приоритизация и объединение результатов

## ✅ ВЫПОЛНЕНО

### 1. Новый метод в ClickHouseVectorStore

#### ✅ `find_sections_by_name()` в `clickhouse_store.py`
```python
def find_sections_by_name(
    name_substring: str,
    source: Optional[str] = None,
) -> list[tuple[str, str, int]]
```

**Реализация:**
- Ищет подстроку в названии раздела (не в содержимом!)
- GROUP BY source, section для подсчёта чанков
- ORDER BY chunk_count DESC для приоритизации

**SQL:**
```sql
SELECT source, section, count() AS chunk_count
FROM table FINAL
WHERE positionCaseInsensitive(section, 'query') > 0
GROUP BY source, section
ORDER BY chunk_count DESC
```

### 2. Новый инструмент в kb_tools.py

#### ✅ Pydantic схема `FindRelevantSectionsInput`
```python
class FindRelevantSectionsInput(BaseModel):
    query: str              # фраза для поиска в названиях
    exact_terms: list[str]  # термины для поиска в содержимом
    limit: int = 100        # лимит сканирования на термин
    source: Optional[str]   # опциональный фильтр по файлу
```

#### ✅ @tool функция `find_relevant_sections()`

**Алгоритм:**

```
STAGE 1: Поиск по НАЗВАНИЮ раздела
    → find_sections_by_name(query)
    → разделы где query есть в названии
    → помечаются как [NAME MATCH] с 🎯

STAGE 2: Поиск по СОДЕРЖИМОМУ  
    → для каждого термина: exact_search_sections(term)
    → разделы где термины есть в содержимом
    → подсчёт упоминаний терминов

MERGE: Объединение результатов
    → дедупликация по (source, section)
    → приоритет: [NAME MATCH] > упоминания терминов
    → группировка по source файлам
```

**Формат вывода:**
```
Найдено релевантных разделов: 21 из 7 файлов
  • Запрос: 'PostgreSQL'
  • Точные термины: ['PostgreSQL', 'СУБД']

[F] file1.md:
   🎯 PostgreSQL Settings [NAME MATCH] (150 чанков)
   • Database Configuration (12 упоминаний терминов)

[F] file2.md:
   • Server Setup (8 упоминаний терминов)
```

## 🎯 СЦЕНАРИИ ИСПОЛЬЗОВАНИЯ

### Сценарий 1: Пользователь спросил про "PostgreSQL"

**Старый подход:**
```python
# Агент не знает, искать в названии или содержимом
exact_search("PostgreSQL")  # может пропустить разделы с названием "PostgreSQL"
```

**Новый подход (двухэтапный):**
```python
find_relevant_sections(
    query="PostgreSQL",           # ищем в названиях
    exact_terms=["PostgreSQL"],   # ищем в содержимом
    limit=100
)

# Результат:
# 🎯 PostgreSQL Configuration [NAME MATCH]  ← начинаем отсюда!
# • Database Setup (15 упоминаний)
# • Server Settings (8 упоминаний)
```

### Сценарий 2: Запрос "настройка сетевого оборудования"

```python
find_relevant_sections(
    query="сетевое оборудование",        # ищем разделы с таким названием
    exact_terms=["настройка", "сеть"],   # + ищем эти термины
    limit=100
)

# → Приоритет: разделы названные "сетевое оборудование"
# → Дополнительно: разделы с терминами о настройке
```

### Сценарий 3: Уточнённый поиск в файле

```python
find_relevant_sections(
    query="VLAN",
    exact_terms=["VLAN", "192.168"],
    source="Network_Config.md",
    limit=50
)

# → Только в Network_Config.md
# → Разделы с "VLAN" в названии + разделы с IP адресами
```

## 📊 СРАВНЕНИЕ СТРАТЕГИЙ

| Инструмент | Поиск где | Результат | Когда использовать |
|---|---|---|---|
| `exact_search` | Содержимое | Чанки | Нужны конкретные данные |
| `find_sections_by_term` | Содержимое | Список разделов | Быстрый обзор по термину |
| `find_relevant_sections` | Название + Содержимое | Список разделов | **PRIMARY TOOL** - полный обзор |
| `list_sections` | Все разделы | Структура БД | Навигация по БД |

### Преимущества `find_relevant_sections`:

1. **Максимальный охват** - находит разделы и по названию и по содержимому
2. **Приоритизация** - разделы с названием идут первыми (🎯 маркер)
3. **Умная дедупликация** - если раздел найден обоими способами, показывается один раз
4. **Подсчёт упоминаний** - видно, насколько релевантен каждый раздел

## 💡 СТРАТЕГИЯ ИСПОЛЬЗОВАНИЯ АГЕНТОМ

### Рекомендуемый workflow:

```
1. find_relevant_sections(query, terms)
   → Получить ВСЕ релевантные разделы

2. Приоритизировать:
   a) Разделы с [NAME MATCH] 🎯
   b) Разделы с максимумом упоминаний

3. Детальное изучение:
   - get_section_content(top_section)
   - exact_search(term, section=top_section)
```

### Пример последовательности:

```python
# Шаг 1: Обзор
result = find_relevant_sections(
    query="PostgreSQL конфигурация",
    exact_terms=["PostgreSQL", "настройка", "параметры"]
)
# → 🎯 PostgreSQL Configuration [NAME MATCH]
# → • Database Parameters (15 упоминаний)

# Шаг 2: Детали
content = get_section_content(
    source_file="database.md",
    section="PostgreSQL Configuration"
)
# → Полный текст раздела с конфигурацией
```
