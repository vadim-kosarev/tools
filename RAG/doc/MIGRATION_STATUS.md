# ✅ Готово: Предложение по переходу на структурированные результаты

## 📊 КРАТКАЯ СВОДКА

**Проблема:** Инструменты возвращают `str` (форматированный текст)  
**Решение:** Переход на Pydantic модели (структурированные объекты)  
**Статус:** ✅ МОДЕЛИ СОЗДАНЫ, ОЖИДАЕТСЯ РЕШЕНИЕ О МИГРАЦИИ

---

## ✅ ЧТО СДЕЛАНО

### 1. Созданы Pydantic модели (14 классов)

**Файл:** `kb_tools.py` (добавлено в начало после импортов)

#### Базовые модели:
- `ChunkMetadata` - метаданные чанка
- `ChunkResult` - результат поиска чанка
- `ScoredChunkResult` - чанк с оценкой
- `SectionInfo` - информация о разделе

#### Результаты поиска:
- `SearchChunksResult` - список чанков
- `SearchSectionsResult` - список разделов
- `MultiTermSearchResult` - мультитермовый поиск

#### Специализированные:
- `RegexMatch` + `RegexSearchResult` - regex поиск
- `TableRow` + `TableResult` - таблицы
- `SectionContent` - полный контент раздела

#### Навигация:
- `SectionTreeNode` + `SectionsTree` - дерево разделов
- `NeighborChunksResult` - соседние чанки
- `SourceInfo` + `SourcesList` - список файлов

## 🎯 СРАВНЕНИЕ

### СЕЙЧАС (строки):
```python
def semantic_search(...) -> str:
    return "[1] [file.md] — Section\nContent...\n[2]..."
```

### ПРЕДЛАГАЕТСЯ (структуры):
```python
def semantic_search(...) -> SearchChunksResult:
    return SearchChunksResult(
        query="PostgreSQL",
        chunks=[
            ChunkResult(
                content="Content...",
                metadata=ChunkMetadata(
                    source="file.md",
                    section="Section",
                    line_start=123,
                    ...
                )
            )
        ],
        total_found=10
    )
```
