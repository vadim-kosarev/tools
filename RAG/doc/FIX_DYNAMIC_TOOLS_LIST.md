# Исправление: Динамический список инструментов из реестра

**Дата:** 2026-04-26  
**Файл:** `rag_lg_agent.py`

---

## Проблема

В методах `plan_node` и `action_node` список доступных инструментов был **захардкожен** в тексте системного промпта:

```python
Доступные инструменты:
- semantic_search: семантический поиск по документации
- exact_search: точный поиск подстроки
- multi_term_exact_search: поиск по нескольким терминам
- regex_search: поиск по regex-паттерну
...
```

### Проблемы захардкоженного подхода:

1. ❌ Список может устареть при добавлении новых инструментов
2. ❌ При удалении инструментов нужно обновлять код в нескольких местах
3. ❌ Нарушается принцип DRY (Don't Repeat Yourself)
4. ❌ Нет единого источника истины для списка инструментов
5. ❌ Риск рассинхронизации с реальными инструментами

---

## Решение

Используется **реестр инструментов** из `kb_tools.py`:

### 1. Добавлен импорт

```python
from kb_tools import create_kb_tools, get_tool_registry
```

### 2. Создана функция форматирования

```python
def _format_tools_list() -> str:
    """Форматирует список доступных инструментов из реестра для промпта."""
    tool_registry = get_tool_registry()
    lines = ["Доступные инструменты:"]
    for tool_name, description in tool_registry.items():
        lines.append(f"- {tool_name}: {description}")
    return "\n".join(lines)
```

### 3. Обновлён plan_node

**Было:**
```python
system_message = f"""{_SYSTEM_PROMPT}

ТЕКУЩИЙ ЭТАП: plan

Доступные инструменты:
- semantic_search: семантический поиск по документации
- exact_search: точный поиск подстроки
...
```

**Стало:**
```python
system_message = f"""{_SYSTEM_PROMPT}

ТЕКУЩИЙ ЭТАП: plan

{_format_tools_list()}

Сформируй план поиска для ответа на вопрос пользователя."""
```

### 4. Обновлён action_node

Аналогично добавлен динамический список через `_format_tools_list()`.
