ТЕКУЩИЙ ЭТАП: observation (итерация {{ iteration }}/{{ MAX_ITERATIONS }})

Проанализируй результаты выполнения инструментов.
Извлеки ТОЛЬКО информацию из результатов инструментов
НЕ придумывай, НЕ домысливай.

Опиши ПОДРОБНО:
1. Что найдено (конкретные факты из результатов) - перечисли ВСЕ ключевые находки
2. Что НЕ найдено (какие пробелы остались) - укажи что искали, но не нашли
3. Достаточно ли данных для полного ответа
4. Какие связи и закономерности видны в найденных данных
5. **Какие КОНКРЕТНЫЕ разделы нужно изучить детально** (если данных недостаточно)

📝 **ВАЖНО - ДЕТАЛЬНОСТЬ АНАЛИЗА**:
- НЕ ограничивайся общими фразами типа "найдено несколько записей"
- КОНКРЕТИЗИРУЙ что именно найдено - названия, значения, источники
- Если нашёл частичную информацию - опиши её полностью
- Помоги понять ОБЩУЮ КАРТИНУ найденных данных
- Observation должна быть информативной - это база для финального ответа

🎯 **ОПРЕДЕЛЕНИЕ РАЗДЕЛОВ ДЛЯ ДЕТАЛЬНОГО ИЗУЧЕНИЯ**:
Если результаты содержат упоминания разделов/файлов, но без детальной информации:
- Выпиши КОНКРЕТНЫЕ файлы и разделы, которые нужно прочитать полностью
- Укажи WHY каждый раздел релевантен (какую информацию ожидаем там найти)
- Это поможет на следующей итерации использовать get_section_content / exact_search_in_file_section

⚠️ ОБЯЗАТЕЛЬНЫЙ ФОРМАТ ОТВЕТА для этапа observation:
```json
{
  "status": "observation",
  "step": {{ step + 1 }},
  "thought": "краткое рассуждение о результатах",
  "observation": "ПОДРОБНЫЙ анализ (несколько абзацев): 
  - перечисли ВСЕ ключевые факты что найдено
  - укажи конкретные значения, названия, источники
  - опиши что НЕ найдено и какие пробелы
  - оцени достаточность данных для ответа
  - отметь интересные связи и закономерности",
  "next_targets": [
    {
      "source": "Общее описание системы Приложение А-(ИИ-26).md",
      "section": "Конкретное название раздела",
      "relevance": "high",
      "reason": "содержит детальное описание установленного ПО"
    }
  ]
}
```

**Поле next_targets:**
- Заполняй ТОЛЬКО если данных недостаточно и нужно изучить конкретные разделы
- Указывай КОНКРЕТНЫЕ файлы и разделы (из метаданных результатов поиска)
- relevance: "high" - обязательно нужно прочитать, "medium" - может быть полезно
- reason: кратко опиши, какую информацию ожидаешь найти в этом разделе
- Пустой массив [] означает, что данных достаточно для финального ответа

**Пример workflow с next_targets:**
```
Итерация 1 - action: multi_term_exact_search
          ↓
Итерация 1 - observation: 
   "Найдены упоминания в разделе 'Сводная таблица устанавливаемого ПО', но не хватает деталей"
   next_targets: [{source: "file.md", section: "Сводная таблица устанавливаемого ПО", relevance: "high"}]
          ↓
Итерация 1 - refine: needs_refinement=True (есть конкретные разделы в next_targets)
          ↓
Итерация 2 - action: get_section_content(file="file.md", section="Сводная таблица устанавливаемого ПО")
          ↓
Итерация 2 - observation: "Получен полный список ПО: ..."
   next_targets: [] (данных достаточно)
          ↓
Итерация 2 - refine: needs_refinement=False
          ↓
final: формируем ответ с полными данными
```

НЕ ИСПОЛЬЗУЙ поля "actions" или "final_answer" на этом этапе!
