Будем здесь делать RAG (Retrieval-Augmented Generation) на основе python библиотек.

Цель - иметь вконсоли чат, который может обрабатывать запросы по списку документов, которые мы ему предоставим. И выдавать ответы на основе этих документов.
Инструкменты - ollama запущена на localhost:11434
Список моделей - команда ollama list
Предпочтительная модель - та которая умещается в 8 ГБ памяти GPU
Скачать можно любую модель из списка, например - ollama pull llama-2-7b-chat-hf, если будет обоснованно нужно

Можно использовать любые современные библиотеки для работы с текстом. Например, для векторизации текста и поиска по вектору можно использовать библиотеки как Faiss или Annoy. Для обработки текста и генерации ответов можно использовать Hugging Face Transformers.

Исходные файлы были в формате .docx, но были сконверитрованы в .md

Для начала нужно научиться работать с файлом "Z:\ES-Leasing\СОИБ КЦОИ\Общее описание системы-(ИИ-26).docx" 

Питон нужно использовать из .venv проекта, чтобы иметь все необходимые интсрументы

в docker-е запущено много чего, может быть запущено все, что нужно. Например chroma

нужен инструмент, который работает с .md-файлами из папки "Z:\ES-Leasing\СОИБ КЦОИ" и может векторизовать их для последующего поиска. Например, можно использовать библиотеку Hugging Face Transformers для векторизации текста и Faiss для поиска по вектору.

Агент должен обрабатывать запросы как на поиск точных терминов, так и на генерацию ответов на основе текста, логических связей между упоминаемыми документами, а так же искать по шаблонам regexp в документах, например - список всех IP и умень по контексту вокруг определять логику каждого IP.
