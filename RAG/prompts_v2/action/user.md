
## ЭТАП: ACTION (Выполнение)

Твоя задача: **вызвать инструменты** согласно плану.

### 🎯 ВОПРОС ПОЛЬЗОВАТЕЛЯ (РЕАЛЬНЫЙ):

```
{{ user_query }}
```

⚠️ **Это НАСТОЯЩИЙ вопрос!** Вызови инструменты чтобы найти ответ на ЭТОТ конкретный вопрос.

### План выполнения

**Полный план:**
{% for step in plan %}
{{ loop.index }}. {{ step }}{% if loop.index0 in completed_steps %} ✅ ВЫПОЛНЕНО{% endif %}
{% endfor %}

**Статус:**
- Завершено шагов: {{ completed_steps | length }}/{{ plan | length }}
- Доступно для выполнения: {{ available_steps | length }} шагов

{% if available_steps %}
**Доступные шаги (можно выполнить сейчас):**
{% for step_idx in available_steps %}

{% endif %}

### 💡 ВАЖНО: Можешь выполнять несколько шагов за раз!

Если видишь, что **несколько шагов подряд** можно выполнить одновременно - **делай это!**

Пример:
- Шаг 1: "Найти все СУБД"
- Шаг 2: "Найти IP-адреса"
- Шаг 3: "Получить описание"

Если шаги 1 и 2 можно выполнить параллельно (разные инструменты), выбери инструменты для обоих шагов!

### Что делать

1. **Посмотри на доступные шаги** - какие можно выполнить сейчас?
2. **Выбери инструменты** для одного или нескольких шагов подряд
3. **В поле `plan_steps`** укажи индексы шагов (0-based), например: `[0, 1]` для шагов #1 и #2
4. **Сформируй actions** с параметрами для каждого инструмента

### Formат ответа

```json
{{ schema_AgentAction }}
```

**Обязательно заполни `plan_steps`** - список индексов шагов (0-based), которые покрываются твоими действиями!

### Примеры

**Пример 1: Выполнение одного шага**
```json
{
  "status": "action",
  "thought": "Выполняю шаг 1: поиск СУБД",
  "plan_steps": [0],
  "actions": [
    {"tool": "semantic_search", "input": {"query": "СУБД базы данных"}}
  ]
}
```

**Пример 2: Выполнение нескольких шагов**
```json
{
  "status": "action",
  "thought": "Выполняю шаги 1 и 2: поиск СУБД и IP-адресов одновременно",
  "plan_steps": [0, 1],
  "actions": [
    {"tool": "semantic_search", "input": {"query": "СУБД базы данных"}},
    {"tool": "regex_search", "input": {"pattern": "\\\\d+\\\\.\\\\d+\\\\.\\\\d+\\\\.\\\\d+"}}
  ]
}
```

### Важно

- **Точность параметров**: 
  - `semantic_search` - семантический поиск по смыслу (параметр: `query` или `substring`)
  - `exact_search` - точный поиск по подстроке (параметр: `query` или `substring`)
  - 💡 Можешь использовать любое название - система автоматически конвертирует
- **Новое**: если это повторная итерация — ищи ДРУГУЮ информацию
- **plan_steps**: Обязательно указывай какие шаги выполняешь!
