# Режим отладки промптов (--steps)

## Описание

Режим отладки позволяет выполнить только первые N шагов (нод) графа агента, чтобы посмотреть промежуточные результаты и логи.

## Использование

```bash
# Выполнить только первые 2 шага
python rag_lg_agent.py --steps 2 "ваш вопрос"

# Выполнить первые 3 шага с подробным выводом
python rag_lg_agent.py --steps 3 --verbose "ваш вопрос"
```

## Последовательность нод графа

1. **plan** - Построение плана действий
2. **action** - Выполнение инструментов
3. **observation** - Обработка результатов
4. **refine** - Принятие решения (продолжить или финализировать)
5. **action** (повтор) - Второй цикл, если нужно
6. **observation** (повтор)
7. **refine** (повтор)
8. **final** - Формирование финального ответа

## Примеры

### Отладка планирования (1 шаг)
```bash
python rag_lg_agent.py --steps 1 "что такое СОИБ"
```
Выполнится только нода `plan`, можно посмотреть:
- Логи: `logs/001_llm_plan_request.log`, `logs/002_llm_plan_response.log`
- Сформированный план действий

### Отладка первого вызова инструментов (2 шага)
```bash
python rag_lg_agent.py --steps 2 "что такое СОИБ"
```
Выполнится `plan` → `action`, можно посмотреть:
- Какие инструменты вызываются
- Логи запросов: `logs/003_llm_action_request.log`, `logs/004_llm_action_response.log`

### Отладка обработки результатов (3 шага)
```bash
python rag_lg_agent.py --steps 3 "что такое СОИБ"
```
Выполнится `plan` → `action` → `observation`, можно посмотреть:
- Результаты выполнения инструментов
- Логи обработки: `logs/005_llm_observation_request.log`, `logs/006_llm_observation_response.log`

### Отладка решения о продолжении (4 шага)
```bash
python rag_lg_agent.py --steps 4 "что такое СОИБ"
```
Выполнится `plan` → `action` → `observation` → `refine`, можно посмотреть:
- Принимает ли агент решение продолжить или финализировать
- Логи: `logs/007_llm_refine_request.log`, `logs/008_llm_refine_response.log`

## Вывод в режиме отладки

При остановке до финальной ноды вы увидите:

```
================================================================================
Вопрос: что такое СОИБ
🔍 РЕЖИМ ОТЛАДКИ: лимит 2 шагов (нод графа)
Шагов: 1
Итераций: 1/2
Messages: 2
Tools executed: 0
================================================================================

================================================================================
🔍 ОТЛАДКА - СОСТОЯНИЕ НА МОМЕНТ ОСТАНОВКИ:
================================================================================

⚠️  Выполнение остановлено до формирования финального ответа

📊 Текущее состояние:
  - Plan: 3 шагов
  - Tool calls: 0
  - Observation: 0 символов
================================================================================
```

## Логи

Все промпты и ответы LLM сохраняются в `logs/` с последовательной нумерацией:
- `001_llm_plan_request.log` - System prompt + User prompt для plan ноды
- `002_llm_plan_response.log` - Ответ LLM для plan ноды
- `003_llm_action_request.log` - Промпт для action ноды
- `004_llm_action_response.log` - Ответ для action ноды
- И так далее...

## Польза для отладки

1. **Проверка промптов** - Можно посмотреть какие именно промпты отправляются в LLM
2. **Анализ ответов** - Как LLM интерпретирует инструкции
3. **Пошаговое выполнение** - Найти на каком шаге что-то идет не так
4. **Экономия времени** - Не ждать полного выполнения при отладке начальных шагов
5. **Тестирование изменений** - После правки промптов проверить первые шаги

## Отключение режима отладки

Просто не указывайте аргумент `--steps`:

```bash
python rag_lg_agent.py "ваш вопрос"
```

Граф выполнится полностью до формирования финального ответа.
